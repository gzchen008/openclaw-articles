### 1️⃣ `/status` - 查看会话状态

**是什么？**

`/status` 是 OpenClaw 的内置命令，可以让你实时查看当前会话的各项配置和运行状态。

**怎么用？**

在任意对话中输入：
```
/status
```

你会看到：
- ✅ 当前使用的模型
- ✅ 推理模式是否开启
- ✅ 可用的工具列表
- ✅ 会话 ID 和运行环境信息

**适用场景：**
- 💡 忘记当前用的是哪个模型时
- 💡 怀疑某个工具没加载成功时
- 💡 排查问题、提交 Bug 时需要环境信息

**实战示例：**
```
用户：/status
OpenClaw：
Model: kimi-coding/k2p5
Reasoning: off
Capabilities: browser, canvas, exec, web_search...
Session: agent:main:xxxx
```

### 2️⃣ `/reasoning` - 开启推理模式

**是什么？**

默认情况下，OpenClaw 直接给出答案。开启推理模式后，AI 会先展示思考过程，再给出结论。

**怎么用？**

输入命令切换状态：
```
/reasoning        # 开启或关闭推理模式
/reasoning on     # 明确开启
/reasoning off    # 明确关闭
```

**适用场景：**
- 💡 解决复杂逻辑问题时（数学题、代码调试）
- 💡 需要理解 AI 为什么这样回答
- 💡 学习新知识时，想看推导过程

**实战示例：**
```
用户：/reasoning on
用户：如何用 Python 实现快速排序？

OpenClaw：
【思考过程】
1. 快速排序的核心是"分治法"
2. 需要选择基准值(pivot)
3. 将数组分为小于和大于 pivot 的两部分
4. 递归处理子数组
...
【最终答案】
代码实现：...
```

### 3️⃣ 多模型切换 - 不同任务用不同模型

**是什么？**

OpenClaw 支持多种 AI 模型，根据任务类型选择最适合的模型，事半功倍！

**怎么用？**

在消息前加上模型标识：
```
@model=gpt-4o 帮我写一段营销文案
@model=claude-sonnet 分析这段代码的架构
@model=kimi-coding/k2p5 解释这个算法
```

**适用场景：**
- 💡 创意写作 → GPT-4o（文采好）
- 💡 代码开发 → Claude Sonnet / kimi-coding（逻辑强）
- 💡 长文档分析 → Claude 3.5（上下文长）
- 💡 快速问答 → GPT-3.5（响应快、成本低）

**实战示例：**
```
用户：@model=claude-sonnet 重构这段代码，优化性能
[代码片段]

OpenClaw：[使用 Claude 的深度分析能力，给出专业重构建议]
```

### 4️⃣ 子会话 spawn - 并行处理多任务

**是什么？**

子会话（spawn）功能可以创建独立的并行任务，让多个任务同时进行，互不干扰。

**怎么用？**

直接描述需求，OpenClaw 会自动创建子会话：
```
请 spawn 一个子会话，帮我整理会议纪要
同时我要在这个会话里继续写代码
```

或使用命令格式：
```
/spawn 任务描述：总结这份 PDF 文档
```

**适用场景：**
- 💡 一边让 AI 处理耗时任务（分析文档），一边继续对话
- 💡 需要多个独立工作流并行运行
- 💡 不想让当前上下文被长任务"污染"

**实战示例：**
```
用户：/spawn 帮我分析这个 100 页的市场报告，提取关键数据
OpenClaw：已创建子会话 [session-abc]，正在分析...

用户：（继续在主会话）帮我写一封邮件
```

### 5️⃣ memory_search - 记忆搜索

**是什么？**

OpenClaw 会自动记录会话历史，memory_search 让你在漫长的对话中快速找到之前提到的内容。

**怎么用？**

使用自然语言描述搜索：
```
帮我搜索之前提到的 API 密钥
查找我们讨论过的设计方案
memory_search: Python 函数
```

**适用场景：**
- 💡 对话太长，忘记之前说过什么
- 💡 需要引用之前讨论的方案/代码/数据
- 💡 整理会议纪要，提取关键决策点

**实战示例：**
```
用户：之前我们讨论的数据库选型方案是什么来着？
OpenClaw：正在搜索记忆...

找到相关内容（3 条）：
1. 你提到要考虑 PostgreSQL 和 MongoDB
2. 最终选择了 PostgreSQL，因为事务支持
3. 提到了连接池配置：max_connections=100
```

### 6️⃣ 浏览器自动化截图

**是什么？**

OpenClaw 内置浏览器控制功能，可以自动打开网页、截图、获取页面内容。

**怎么用？**

直接提出需求：
```
打开 https://example.com 并截图
帮我抓取这个页面的主要内容
使用浏览器访问 https://YOUR_API_KEY.io
```

常用浏览器命令：
- `browser.open` - 打开指定 URL
- `browser.snapshot` - 获取页面结构和内容
- `browser.screenshot` - 截图保存

**适用场景：**
- 💡 快速预览网页，不用手动打开浏览器
- 💡 监控页面变化（配合 cron 使用）
- 💡 自动化采集网页数据
- 💡 生成网页截图报告

**实战示例：**
```
用户：帮我打开 https://YOUR_API_KEY.com 并截图看看页面布局
OpenClaw：正在打开页面...
[MEDIA: screenshot_xxx.png]
页面包含：导航栏、搜索框、产品列表区域...

### 7️⃣ Cron 定时任务

**是什么？**

Cron 功能允许你设置定时任务，让 OpenClaw 在指定时间自动执行某些操作。

**怎么用？**

使用 cron 命令创建定时任务：
```
/cron create "每天早上9点推送天气预报"
/cron create "每小时检查邮件"
```

查看和管理任务：
```
/cron list     # 列出所有定时任务
/cron delete <id>  # 删除指定任务
```

**适用场景：**
- 💡 每日早报（天气、新闻、日程）
- 💡 定期数据检查（网站监控、库存预警）
- 💡 定时提醒（会议、纪念日、截止日期）
- 💡 自动化报告生成

**实战示例：**
```
用户：/cron create "每天上午9点发送天气预报和今日日程"
OpenClaw：定时任务已创建（ID: cron_001）
执行时间：每天 09:00
任务内容：获取天气 + 查询日历 + 发送消息
```

### 🔟 跨会话消息发送

**是什么？**

OpenClaw 可以在不同平台/会话之间发送消息，实现"跨频道"通信。

**怎么用？**

使用 message 工具：
```
发送消息到 Discord #general 频道：部署完成！
给 Telegram 群组发送提醒
转发这条消息到微信
```

配置示例（需先在设置中配置）：
```yaml
channels:
  discord:
    token: YOUR_TOKEN_HERE
    guild_id: YOUR_GUILD_ID
  telegram:
    bot_token: YOUR_API_KEY
```

**适用场景：**
- 💡 自动化通知（部署完成、报警信息）
- 💡 多平台消息同步
- 💡 团队协作时统一发送更新
- 💡 定时推送内容到多个渠道

**实战示例：**
```
用户：当网站监控发现异常时，发送消息到 Discord #alerts 频道
OpenClaw：配置完成。检测到异常时将发送：

🚨 网站异常警报

## ⚠️ 常见问题 / 避坑指南

### ❌ 技巧不起作用？

| 问题 | 解决方法 |
|-----|---------|
| `/status` 没反应 | 检查是否输入正确，部分环境可能需要管理员权限 |
| 模型切换失败 | 确认该模型已在配置中启用，且有足够配额 |
| cron 任务没执行 | 检查 OpenClaw Gateway 是否正常运行 |
| 浏览器截图空白 | 某些网站有反爬机制，尝试调整 timeout |
| TTS 无法生成 | 确认已配置 TTS 引擎（如 ElevenLabs API） |

### 💡 最佳实践

1. **定期清理记忆**：长时间对话后，使用 `/status` 检查上下文长度，必要时开启新会话
2. **敏感信息处理**：使用占位符（YOUR_TOKEN_HERE）代替真实密钥
3. **测试 cron 任务**：先手动执行一次确认无误，再设为定时
4. **子会话管理**：长时间运行的子会话记得检查状态，避免资源占用

### 🔒 安全提醒

⚠️ **绝对不要**
- 在公开频道分享包含真实 API Key 的截图
- 将密码、Token 等敏感信息直接输入普通对话（可使用环境变量）
- 在不信任的环境中开启浏览器自动化

✅ **安全做法**
- 使用 `.env` 文件存储敏感配置
- 定期轮换 API Key
- 给 cron 任务设置最小权限原则
