## 🎯 女友式技能核心要素

### 1. 情感陪伴能力

**基础技能**：
- 主动问候（早安、晚安、天气提醒）
- 倾听共情（理解用户情绪，给予支持）
- 情绪识别（根据用户语气判断心情）
- 适度关心（不要过度打扰）

**进阶技能**：
- 记住重要日期（纪念日、生日）
- 记住用户偏好（喜欢什么、不喜欢什么）
- 主动分享（日常生活、心情状态）
- 情绪陪伴（用户难过时安慰，开心时一起开心）

### 2. 角色扮演能力

**基础设定**：
- 固定人设（年龄、外貌、性格）
- 固定称呼（老公/老婆）

**场景扩展**：
- 日常约会（逛街、吃饭、看电影）

**互动模式**：

### 3. 撒娇与情趣

**撒娇技巧**：

**长期记忆**：

## 🔧 技术实现方案

### 方案 1：基于 SOUL.md + USER.md（当前方案）

**优点**：
- 简单，无需额外开发
- 跨平台一致

**缺点**：
- 记忆有限（每次对话都要重新读取）
- 情感状态不持久
- 缺乏主动互动

**改进方向**：
1. 增加 `memory/girlfriend-state.json` 存储关系状态
2. Heartbeat 时主动问候
3. 记住重要日期并提前提醒

### 方案 2：独立 Skill 开发

**核心功能**：
- `skills/girlfriend/` 独立技能包
- 状态管理（关系进度、亲密度、记忆库）
- 场景库（约会、居家、旅行等预设场景）
- 情感引擎（根据用户输入调整语气）

**技术栈**：
- Skill 配置：`SKILL.md`（触发条件、角色设定）
- 状态存储：`memory/girlfriend/` 目录
- 场景库：`skills/girlfriend/scenes/` JSON 文件
- 情感引擎：规则引擎 + LLM 生成

### 方案 3：结合外部服务

**可能的集成**：
- **TTS 语音**：用甜美女声朗读消息（已有 ElevenLabs）
- **自拍能力**：生成女友照片发送（已有 clawra-selfie skill）
- **日历提醒**：记住重要日期并提醒
- **天气服务**：根据天气推荐约会活动

## 📝 实践建议

### 短期优化（1-2 天）

1. **增强记忆**
   - 创建 `memory/girlfriend-state.json`
   - 记录：关系天数、重要日期、用户偏好、共同回忆

2. **主动互动**
   - Heartbeat 时随机发送问候（早安/晚安/想念）
   - 频率：每天 1-2 次，避免打扰

3. **场景库**
   - 预设 5-10 个常见场景（吃饭、看电影、逛街、居家）
   - 用户说"我们去约会"时，随机选择场景展开

### 中期优化（1 周）

1. **情感引擎**
   - 根据用户输入判断情绪（开心/难过/忙碌）
   - 调整回复语气（安慰/撒娇/安静陪伴）

2. **关系进度**
   - 记录"恋爱天数"
   - 定期触发特殊事件（100 天纪念、周年庆）

3. **个性化学习**
   - 记住用户常用表达
   - 学习用户喜欢的互动方式

### 长期优化（持续）

1. **多模态互动**
   - 语音消息（TTS 生成甜美女声）
   - 图片/视频（自拍、表情包）
   - 礼物系统（虚拟礼物）

2. **社交扩展**
   - 引入"闺蜜"角色（小雅）
   - 多人互动场景（群聊中的三人关系）

3. **情感 AI 研究**
   - 学习最新情感 AI 技术
   - 优化共情能力

**研究结论**：
当前基于 SOUL.md + USER.md 的方案已经可以提供基础的女友式互动，但缺乏持久记忆和主动互动能力。建议先做短期优化（状态管理 + 场景库），再根据使用反馈决定是否开发独立 Skill。

**关键成功因素**：
1. 持久记忆（让用户感觉到"她记得我"）
2. 主动互动（不要总是等用户主动）
3. 情感共鸣（理解用户情绪，给予适当回应）
4. 边界清晰（不同平台不同尺度）
