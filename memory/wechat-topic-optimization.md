# 公众号文章选题优化机制

> 基于阅读数据自动优化后续文章选题和内容

---

## 📊 数据来源

### 1. 微信官方数据 API
- `getarticlesummary` - 每日图文数据
- `getarticletotal` - 图文总数据
- `getuserread` - 阅读概况
- `getusershare` - 转发概况

### 2. 本地数据
- 文章发布历史 `wechat-content-plan.md`
- 阅读量记录 `analytics/article-stats.json`
- 选题关键词库 `analytics/keywords.json`

---

## 🔄 优化流程

```
每周执行一次：
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│ 1. 收集数据 │ ──▶ │ 2. 分析热点 │ ──▶ │ 3. 调整选题 │
└─────────────┘     └─────────────┘     └─────────────┘
     ↓                   ↓                   ↓
  微信API            阅读量排行          更新内容计划
  +本地记录          +关键词分析         +权重调整
```

---

## 📈 分析维度

### 1. 主题热度
- 哪些主题阅读量高？
- 哪些主题分享多？
- 用户对什么感兴趣？

### 2. 标题效果
- 问句式 vs 陈述式
- 数字开头 vs 文字开头
- 情绪词 vs 中性词

### 3. 发布时间
- 哪个时段阅读量最高？
- 工作日 vs 周末

### 4. 内容深度
- 长文 vs 短文
- 代码多 vs 代码少
- 图文结合 vs 纯文字

---

## 🎯 优化策略

### 阅读量高 → 增加相关主题
```
如果 "OpenClaw实战" 阅读量 > 500
则 增加 1-2 篇 OpenClaw 相关文章
```

### 分享多 → 做系列文章
```
如果 "教程类" 分享率 > 10%
则 将单篇扩展为系列
```

### 阅读量低 → 调整方向
```
如果 某主题 3 篇阅读量都 < 100
则 暂停该主题，尝试新方向
```

## 📝 实现方式

### 每周自动执行

```bash
# Cron 任务
0 20 * * 0  # 每周日 20:00

执行内容：
1. 获取最近 7 天的文章数据
2. 分析阅读量 TOP 5
3. 提取高频关键词
4. 生成选题建议报告
5. 更新内容计划权重
6. 发送 Discord 通知
```

## 💡 选题建议生成规则

### 基于历史数据

1. **热门主题延续**
   - 如果某主题阅读量 > 平均值 50%
   - 建议：继续深挖，做系列文章

2. **新主题探索**
   - 如果所有主题阅读量都很低
   - 建议：尝试新方向，参考竞品

3. **标题优化**
   - 分析高阅读量文章标题特征
   - 生成标题模板

### 基于外部因素

1. **热点结合**
   - 关注科技圈热点
   - 结合 OpenClaw/AI 话题

2. **节日/事件**
   - 特殊日期相关内容
   - 定期回顾总结
