## 💡 关键发现
### 1. OpenClaw 定时任务架构
- 定时任务在**远程 Gateway 服务器**运行,而非本地
- 远程服务器 IP 固定,适合调用需要 IP 白名单的 API(如微信)
- 本地环境 IP 不固定,不适合直接调用
### 2. 微信公众号 API 限制
- 需要在公众号后台配置 IP 白名单
- 只有白名单中的 IP 才能调用 API
- 本地环境 IP 变化频繁, 不适合直接调用
### 3. OpenClaw Memory 架构
- 支持 3 种后端: SQLite / Qdrant / Postgres+AGE
- 默认使用 SQLite(本地文件,无需额外服务)
- Qdrant 提高性能向量搜索(适合大规模数据)
- Postgres+AGE 提供图数据库能力(适合知识图谱)
