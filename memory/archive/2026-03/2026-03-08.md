## 飞书发送文件成功方法

### ✅ 成功方案

**关键点：文件必须放在 workspace 目录下**

```bash
# 1. 复制文件到 workspace
cp /tmp/file.mp4 ~/.openclaw/workspace/file.mp4

# 2. 使用 message tool 发送（path 参数用完整路径）
message action=send channel=feishu target=user:ou_xxx path=/Users/cgz/.openclaw/workspace/file.mp4
```

### ❌ 失败方案

直接发送 `/tmp/` 下的文件 - 飞书无法访问系统临时目录

### 原因分析

- OpenClaw 运行在沙盒环境中
- `/tmp/` 目录可能不在飞书上传服务的访问范围内
- `~/.openclaw/workspace/` 是 OpenClaw 的工作目录，有完整的读写权限

### 支持的文件格式

| 格式 | 测试状态 |
|------|----------|
| `.webm` | ✅ |
| `.mp4` | ✅ |
| `.png` | ✅ |

## agent-browser 技能安装

**时间：** 17:45

### 安装步骤
```bash
# 1. 安装技能
clawhub install agent-browser --force --workdir ~/.openclaw/workspace

# 2. 安装 CLI 依赖
npm install -g agent-browser
```

### 核心功能
- 📸 页面快照 + 元素引用（@e1, @e2...）
- 🖱️ 点击、填表、滚动、上传
- 🎬 录制视频（webm 格式）
- 🔐 保存登录状态
- 📱 模拟设备

### 录制视频流程
```bash
agent-browser open https://example.com
agent-browser record start ./demo.webm
# ... 执行操作 ...
agent-browser record stop

# 转换为 mp4（兼容性更好）
ffmpeg -i demo.webm -c:v libx264 -c:a aac demo.mp4
```

## agent-browser 登录态保存

**时间：** 19:05

### 保存登录态
```bash
# 登录后保存
agent-browser state save ~/.openclaw/workspace/taobao-auth.json
```

### 加载登录态
```bash
# 下次使用时加载
agent-browser state load ~/.openclaw/workspace/taobao-auth.json
agent-browser open https://www.taobao.com
# 此时已是登录状态
```

### 已保存的登录态
| 网站 | 文件 | 大小 |
|------|------|------|
| 淘宝 | `taobao-auth.json` | 1.8M |

### 注意事项
- 登录态有效期取决于网站（淘宝可能几周到几个月）
- 某些网站可能检测无头浏览器，需要额外处理
- 使用 `--headed` 模式登录更稳定
