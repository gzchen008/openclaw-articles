# 需求分级指南

## 分级标准

### 小需求 (Quick Dev 适用)

**量化指标**:
- 代码改动 < 200 行 (不含测试)
- 涉及文件 ≤ 5 个
- 预估工时 < 4 小时

**定性特征**:
- 单一职责变更
- 不涉及架构调整
- 不涉及数据模型变更
- 不涉及跨团队协调

**典型场景**:
| 场景 | 示例 |
|------|------|
| 枚举扩展 | 状态枚举增加值 |
| 字段新增 | DTO/VO 增加字段 |
| 逻辑调整 | 条件判断修改 |
| Bug 修复 | 空指针、边界处理 |
| 配置变更 | 超时、重试次数 |
| 文案修改 | 提示信息、错误码描述 |
| 简单查询 | 增加查询条件 |

### 中型需求 (简化 SDD)

**量化指标**:
- 代码改动 200-500 行
- 涉及文件 5-15 个
- 预估工时 4-16 小时

**定性特征**:
- 涉及 2-3 个模块协作
- 可能涉及接口变更
- 需要考虑兼容性

**典型场景**:
- 新增业务功能点
- 接口重构
- 性能优化
- 批量数据处理

### 大型需求 (完整 SDD)

**量化指标**:
- 代码改动 > 500 行
- 涉及文件 > 15 个
- 预估工时 > 16 小时

**定性特征**:
- 跨多个领域/模块
- 涉及数据模型变更
- 需要架构评审
- 跨团队协调

## 边界决策树

```
开始
  │
  ├─ 涉及数据模型变更？
  │   └─ 是 → 大型需求
  │
  ├─ 涉及公开 API 变更？
  │   └─ 是 → 至少中型需求
  │
  ├─ 跨 3 个以上模块？
  │   └─ 是 → 中型或大型需求
  │
  ├─ 预估改动 > 200 行？
  │   └─ 是 → 至少中型需求
  │
  └─ 以上都否 → 小需求 ✅
```

## 风险评估

即使代码量小，以下情况需要升级处理：

| 风险因素 | 说明 | 建议 |
|---------|------|------|
| 核心链路 | 涉及交易、支付、结算 | 增加测试覆盖 |
| 并发场景 | 多线程、分布式锁 | 需要设计评审 |
| 数据一致性 | 跨库、跨服务事务 | 升级为中型 |
| 外部依赖 | 第三方接口变更 | 增加集成测试 |
| 安全相关 | 权限、加密、脱敏 | 安全评审 |

## 拆分建议

当需求超出小需求范围时，可尝试拆分：

**拆分原则**:
1. 按功能点拆分: 每个子需求独立可交付
2. 按依赖链拆分: 先基础后上层
3. 按风险拆分: 高风险部分单独处理

**拆分示例**:

原需求: "实现订单退款功能"

拆分后:
1. [小] 订单状态枚举增加退款相关状态
2. [小] 退款 DTO/VO 定义
3. [中] 退款服务核心逻辑
4. [小] 退款接口暴露
5. [小] 退款消息通知

## 工时估算参考

| 任务类型 | 小 | 中 | 大 |
|---------|---|---|---|
| 需求分析 | 5min | 30min | 2h |
| 设计 | 0-10min | 1h | 4h |
| 编码 | 1h | 4h | 2d |
| 测试 | 30min | 2h | 1d |
| 文档 | 10min | 30min | 2h |
| **总计** | **< 2h** | **< 1d** | **> 1d** |

## 边界场景示例

以下是一些处于边界的场景，用于帮助准确判定需求级别：

| 场景描述 | 代码量 | 判定 | 判定理由 |
|---------|--------|------|---------|
| 单文件 180 行复杂业务逻辑 | 180 行 | 中型 | 虽然代码量少，但复杂度高，需要设计评审 |
| 5 个文件简单字段新增 | 50 行 | 小型 | 变更单一，无复杂性 |
| 核心交易链路 100 行改动 | 100 行 | 中型 | 涉及核心链路，风险高需额外测试 |
| 状态枚举新增 2 个值 | 10 行 | 小型 | 简单枚举扩展，影响可控 |
| 跨 2 个模块的接口重构 | 150 行 | 中型 | 涉及接口变更，需要兼容性评估 |
| 新增查询条件 + SQL 修改 | 80 行 | 小型 | 单一职责，无架构变更 |
| 批量数据处理脚本 | 250 行 | 中型 | 代码量超标，需拆分或提升级别 |
| 配置项新增 + 代码适配 | 30 行 | 小型 | 配置变更，无复杂逻辑 |
| 修复空指针异常 | 5 行 | 小型 | Bug 修复，影响范围明确 |
| 新增第三方接口调用 | 120 行 | 中型 | 涉及外部依赖，需要异常处理和测试 |
