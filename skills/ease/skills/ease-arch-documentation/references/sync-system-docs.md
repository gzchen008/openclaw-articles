## System Docs Sync (Opt-in)

本文件定义 **架构文档生成后** 对其他系统文档（如 `CLAUDE.md`、`README.md`）的“安全同步”规则。

### 核心原则

1. **默认关闭**：仅在用户明确要求“同步更新系统文档”时执行。
2. **只更新标记块**：禁止对未标记区域做任何改写/重排。
3. **输出可审查**：必须输出“同步报告”（修改了哪些文件/哪些块/摘要）。

### 标记块规范

使用成对注释包围自动更新区域：

```markdown
<!-- Updated by ease-arch-documentation:BEGIN -->
... auto generated content ...
<!-- Updated by ease-arch-documentation:END -->
```

规则：

- 仅允许替换 `BEGIN..END` 之间的内容。
- 若目标文件不存在标记块 → **跳过**（除非用户明确要求创建文件/插入块）。

### 同步报告（必须）

同步完成后输出：

- 更新的文件列表
- 每个文件：更新了哪些块（块标题/锚点/行号范围可选）
- 变更摘要（3-5 条）
- 跳过原因（未找到标记块/文件不存在/用户未 opt-in）
