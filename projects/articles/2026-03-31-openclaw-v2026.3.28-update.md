# OpenClaw v2026.3.28 重大更新：Grok 搜索、图像生成、审批机制全面升级！

OpenClaw 又双叒叕更新了！

这次带来了 **3 个重磅新功能** + **50+ 项优化修复**，让你的 AI 助手更强大、更稳定！

---

## 🚀 三大重磅新功能

### 1️⃣ xAI Grok 搜索正式上线

**终于来了！** OpenClaw 现在原生支持 Grok 的 `x_search` 功能：

✅ **自动启用**：配置 Grok API Key 后自动开启搜索能力
✅ **无缝集成**：无需手动切换插件，直接使用
✅ **模型选择器**：在 `openclaw onboard` 时可选择搜索模型

**使用场景**：
- 实时信息查询（新闻、事件、天气）
- 代码搜索（GitHub、Stack Overflow）
- 知识检索（维基百科、技术文档）

---

### 2️⃣ MiniMax 图像生成

**AI 绘画新选择！** MiniMax 现在支持图像生成：

✅ **generate**：从文本生成图像
✅ **image-to-image**：基于图片编辑生成新图像
✅ **aspect ratio 控制**：自由调整宽高比

**使用场景**：
- 公众号文章配图
- 社交媒体内容创作
- 产品原型设计

---

### 3️⃣ 插件审批机制（安全升级）

**更安全的工具执行！** 新增 `before_tool_call` hooks 异步审批：

✅ **暂停执行**：工具调用前暂停，等待用户确认
✅ **多渠道审批**：
   - `/approve` 命令（所有渠道）
   - Telegram 按钮
   - Discord 交互
   - 执行审批覆盖层

✅ **自动回退**：exec 和插件审批统一处理

**使用场景**：
- 高危操作（删除、修改配置）
- 外部 API 调用（发送消息、上传文件）
- 敏感数据访问（读取密钥、令牌）

## 🔥 其他重要更新

### ACP 通道绑定（Discord/BlueBubbles/iMessage）

现在可以在当前会话中直接绑定 ACP：

```bash
/acp spawn codex --bind here
```

**效果**：当前聊天变成 Codex 工作区，无需创建子线程！

### Gemini CLI 后端支持

OpenClaw 现在支持三种 CLI 后端：
- **Claude CLI**（已有）
- **Codex CLI**（已有）
- **Gemini CLI**（新增 ✨）

### Podman 容器化改进

- 简化 rootless 用户设置
- 安装 launch helper 到 `~/.local/bin`
- 新的 `openclaw --container <name>` 工作流

### Slack/Teams/Google Chat 文件上传

统一文件上传接口：
- Slack：`upload-file` action
- Microsoft Teams：原生文件上传
- Google Chat：原生文件上传
- BlueBubbles：通过 `upload-file` 发送

## 🐛 重要修复

### 1. Anthropic 敏感内容处理

**问题**：遇到敏感内容时程序崩溃
**修复**：结构化处理 `sensitive` 停止原因，不再崩溃

### 2. Gemini 3.1 模型支持

**问题**：Gemini 3.1 Pro/Flash/Lite 无法识别
**修复**：支持所有 Gemini 3.1 模型别名

### 3. Telegram 消息分割

**问题**：长消息在单词中间断开
**修复**：按单词边界分割，HTML 长度精确计算

### 4. WhatsApp 自聊循环

**问题**：机器人自己发的消息被重复处理
**修复**：过滤自己发送的消息，避免无限循环

### 5. Discord 网关重连

**问题**：Discord 断线后无法重连
**修复**：清理旧 socket，强制全新重连
