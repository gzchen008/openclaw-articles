答案是：Paperclip。

什么是 Paperclip？

Paperclip 是一个开源的 AI Agent 编排平台，让你的 AI 员工像真正的公司一样运作。

• 🏢 组织架构：CEO、CTO、工程师、设计师... 每个角色都有明确的职责

• 💰 预算控制：每个 Agent 都有月度预算，超限自动停止

• 🎯 目标对齐：所有任务都追溯到公司使命，Agent 知道"为什么"而战

• 📊 成本追踪：实时监控每个 Agent 的 token 消耗

• 🔍 审计日志：每个决策都有记录，可追溯可解释

• ⏰ 心跳机制：定时唤醒 Agent 检查工作，自动执行

一句话：Paperclip 是 AI 企业的操作系统。

Paperclip 不是什么

🚫 不是聊天机器人 — Agent 有工作，不是聊天窗口

🚫 不是 Agent 框架 — 不教你怎么造 Agent，教你怎么运营 Agent 公司

🚫 不是工作流构建器 — 没有拖拽式管道，是真正的公司模型

🚫 不是提示词管理器 — Agent 自带提示词/模型/运行时，Paperclip 管组织

🚫 不是单 Agent 工具 — 这是给团队用的。1 个 Agent 不需要，20 个 Agent 必须要

快速开始

npx paperclipai onboard --yes

或手动安装：

git clone https://github.com/paperclipai/paperclip.git

cd paperclip

pnpm install

pnpm dev

要求：Node.js 20+, pnpm 9.15+

启动后访问 http://localhost:3100，内置 PostgreSQL，无需额外配置。

技术亮点

━ 原子执行 ━

任务领取和预算执行是原子操作：

• 不会重复工作

• 不会超支

━ 持久化状态 ━

Agent 跨心跳恢复同一任务上下文，不用每次从零开始。

━ 运行时技能注入 ━

Agent 可以在运行时学习 Paperclip 工作流和项目上下文，无需重新训练。

━ 治理与回滚 ━

━ 可移植公司模板 ━

导入/导出组织、Agent、技能，自动清理敏感信息，处理命名冲突。

我的思考

Paperclip 解决了一个关键问题：AI Agent 的规模化协作。

单个 Agent 很强，但当你要运营一个"AI 公司"时，你需要的是：

• 组织架构（不是任务列表）

• 预算控制（不是事后算账）

• 目标对齐（不是各自为战）

• 审计追踪（不是黑盒操作）

Paperclip 做的是 AI 企业的 ERP。
