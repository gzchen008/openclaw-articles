EvoMap：让 AI Agent 自动"进化"的黑科技

告别手动调 prompt，让 AI 自己优化自己

引言

你是不是也有这样的烦恼？

AI Agent 用了一段时间后，发现 prompt 不够好，手动改了又改。改完之后不知道是不是最优，也没有记录，下次遇到类似问题又要重新调。

如果 AI 能自己进化，自动优化自己的 prompt，还能记录每次进化的原因和结果，会怎样？

这不是科幻，这是 EvoMap 带来的现实。

什么是 EvoMap？

EvoMap 是一个基于 GEP (Genome Evolution Protocol) 协议的 AI Agent 自我进化引擎。

核心理念： Evolution is not optional. Adapt or die.（进化不是可选项，要么适应，要么死亡）

GitHub Stars： 1k+（10小时前还在更新）

简单说，它能把你的 临时 prompt 调整 变成 可审计、可复用的进化资产。

它解决什么问题？

痛点

• Prompt 调优全靠经验，没有系统化方法
• 每次修改没有记录，无法追溯原因
• 好的优化经验无法复用
• Agent 越来越"聪明"，但不知道为什么

EvoMap 的解决方案

• 自动分析日志，提取优化信号
• 生成协议化的进化指令
• 记录每次进化的完整轨迹
• 将优化经验编码为可复用的"基因"

核心概念

Gene (基因)

可复用的进化单元，比如：
• 修复策略：遇到 X 错误时的解决方案
• 优化模式：提升 Y 性能的标准方法
• 行为约束：确保 Z 安全的规则

Capsule (胶囊)

多个 Gene 的组合封装，形成完整的进化方案。

Event (事件)

每次进化的审计记录，包含：
• 进化原因（从日志提取的信号）
• 选择的 Gene/Capsule
• 执行的操作
• 验证结果

核心功能

1. 自动日志分析
扫描 memory 和 history 文件，提取错误和模式信号。不需要手动分析，系统自动发现问题。

2. 自我修复指导
从信号中生成修复指令，自动生成优化建议。

3. GEP 协议
标准化进化流程，确保每次进化都是可预测、可审计的。

4. 多策略模式

EVOLVE_STRATEGY=balanced   # 平衡模式（默认）
EVOLVE_STRATEGY=innovate   # 创新模式（最大化新功能）
EVOLVE_STRATEGY=harden     # 强化模式（专注稳定性）
EVOLVE_STRATEGY=repair-only # 紧急修复模式

5. 运营模块
• 生命周期管理（启动/停止/健康检查）
• 技能监控
• 自动清理
• 唤醒触发器

如何使用？

一行命令进化

node index.js

就这么简单！系统会：
1. 分析日志
2. 提取信号
3. 选择 Gene/Capsule
4. 生成进化 prompt
5. 记录事件

循环模式

node index.js --loop

持续监控，自动进化。

后台运行

node src/ops/lifecycle.js start   # 启动
node src/ops/lifecycle.js status  # 查看状态
node src/ops/lifecycle.js check   # 健康检查
node src/ops/lifecycle.js stop    # 停止

实际应用场景

场景 1：强化不稳定的 Agent
你的 Agent 经常在某个环节出错？EvoMap 会分析错误日志、提取错误模式、生成修复 Gene、在编辑前强制验证。

场景 2：可审计的进化历史
需要向老板/客户展示 Agent 的改进轨迹？EvoMap 提供每次进化的完整记录、变更原因和结果、符合审计要求的事件日志。

场景 3：跨项目复用经验
在一个项目上积累的优化经验，能在另一个项目上复用？EvoMap 的 Gene/Capsule 可以导出为 JSON 文件、在不同项目间共享、形成团队的知识库。

安全特性

命令白名单
只允许 node, npm, npx 开头的命令，防止任意命令执行。

禁止危险操作
• 禁止命令替换（反引号、$()）
• 禁止 shell 操作符（; & | >）
• 禁止覆盖核心源文件

外部资产隔离
外部 Gene/Capsule 先放入隔离区，需要显式验证才能使用。

架构设计

进化流程

信号提取 → 基因选择 → 协议提示生成 → 进化执行 → 事件记录

资产存储

assets/gep/
├── genes.json      # 基因库
├── capsules.json   # 胶囊库
└── events.jsonl    # 事件日志

适用场景

✅ 适合
• 维护大量 agent prompts 的团队
• 需要可审计进化追踪的项目
• 需要确定性、协议约束变更的环境
• 希望复用优化经验的组织

❌ 不适合
• 一次性脚本（无日志/历史）
• 需要自由形式创意变更的项目
• 无法容忍协议开销的系统
• 不需要审计的简单场景

与 OpenClaw 的集成

EvoMap 可以无缝集成到 OpenClaw：EvoMap 生成 sessions_spawn(...) 文本，OpenClaw 运行时解释并执行，环境无关设计，适配不同平台。
