━ 🔥 又双叒叕更新了！ ━

3月8日，OpenClaw 发布了 v2026.3.7 版本。

这次更新堪称"史诗级"，带来了 5 个重磅新功能 + 100+ 项优化修复。

但最让我兴奋的是：Context Engine 插件接口。

这个功能让你的 AI 可以拥有无损记忆，再也不会因为上下文压缩而丢失重要信息了！

今天就来详细解读这次更新的核心亮点。

━ 🧠 1. Context Engine 插件接口（最重磅！） ━

它解决了什么痛点？

你有没有遇到过这种情况：

• ❌ 长对话后，AI 忘记了之前的约定

• ❌ 上下文压缩时，关键信息被丢弃

• ❌ 不同会话之间无法共享记忆

Context Engine 就是为了解决这个问题而生的。

它能做什么？

现在，第三方插件可以提供自定义的上下文管理策略。

例如：

• lossless-claw 插件：提供无损记忆，永不丢失信息

• smart-summarizer 插件：智能摘要，保留关键决策

• cross-session-memory 插件：跨会话记忆共享

完整生命周期钩子

Context Engine 提供了完整的生命周期钩子：

• 🔧 bootstrap - 启动时初始化

• 📥 ingest - 接收新消息

• 🧩 assemble - 组装上下文

• 🗜️ compact - 压缩上下文

• ✅ afterTurn - 对话后处理

• 🤖 prepareSubagentSpawn - 子代理启动前

• 🏁 onSubagentEnded - 子代理结束后

这意味着你可以完全控制 AI 的"记忆方式"。

怎么用？

安装支持 Context Engine 的插件：

clawhub install lossless-claw

然后在配置中启用：

{

"plugins": {

"contextEngine": "lossless-claw"

}

}

零行为改变：如果没有配置 Context Engine 插件，OpenClaw 会使用原有的压缩逻辑。

━ 🔗 2. ACP 持久化频道绑定 ━

它解决了什么痛点？

如果你用过 ACP（Agent Communication Protocol）在 Discord 或 Telegram 上绑定线程，你一定遇到过：

• ❌ 重启 Gateway 后，绑定丢失

• ❌ 每次都要重新绑定线程

• ❌ 多账号管理混乱

现在，这些绑定会持久化保存！

新功能

• ✅ Discord 频道绑定重启后保留

• ✅ Telegram 话题绑定重启后保留

• ✅ CLI 和文档完整支持

• ✅ 路由解析一致性保证

怎么用？

在 Telegram 话题中：

/acp spawn --thread here

绑定会自动保存，重启后仍然有效。

━ 📱 3. Telegram 话题路由增强 ━

它解决了什么痛点？

如果你在 Telegram 群组中使用话题（Topics）功能：

• ❌ 所有话题都用同一个 Agent

• ❌ 无法为不同话题设置不同的"人格"

• ❌ 原生命令授权混乱

现在，每个话题可以绑定不同的 Agent！

新功能

• ✅ 话题级 agentId 覆盖

• ✅ 支持 --thread here|auto 参数

• ✅ 原生命令 Mac Unicode 破折号兼容

• ✅ 可操作的审批按钮

• ✅ 成功绑定后自动置顶确认

使用场景

场景1：技术讨论区

• 话题：#技术问答

• Agent：coding-expert（专注代码）

场景2：闲聊区

• 话题：#闲聊

• Agent：friendly-assistant（轻松对话）

场景3：工作区

• 话题：#工作

• Agent：task-manager（任务管理）

每个话题都有专属的 AI 助手！

━ ⚠️ 重要：Breaking Change！ ━

Gateway 认证现在需要显式指定 gateway.auth.mode

如果你同时配置了 gateway.auth.token 和 gateway.auth.password，必须指定使用哪个：

{

"gateway": {

"auth": {

"mode": "token", // 或 "password"

"token": "YOUR_TOKEN"

}

}

}

不设置会导致启动/配对/TUI 失败！
