━ 🚀 优化 1：模型响应加速 ━

问题：AI 回复慢，等得心焦

解决方案

1️⃣ 选择更快的模型

不同模型响应速度差异很大：

• 最快：GLM-4-Flash、GPT-3.5-Turbo（1-2秒）

• 均衡：GLM-4、Claude Haiku（2-4秒）

• 最强：GPT-4、Claude Opus（5-10秒）

配置默认使用快速模型：

openclaw config.patch '{"agents": {"defaults": {"model": "zai/glm-4-flash"}}}'

2️⃣ 启用流式响应

流式响应让用户提前看到内容，不用等全部生成完：

openclaw config.patch '{"agents": {"defaults": {"stream": true}}}'

3️⃣ 减少上下文长度

历史消息太长会增加响应时间：

openclaw config.patch '{"agents": {"defaults": {"maxHistoryMessages": 20}}}'

4️⃣ 配置多模型 Fallback

主模型慢或失败时自动切换：

{

"agents": {

"defaults": {"model": "zai/glm-4-flash"},

"modelFallback": ["zai/glm-4", "kimi-coding/k2p5"]

}

}

━ 🚀 优化 2：内存占用优化 ━

问题：OpenClaw 占用太多内存

解决方案

1️⃣ 限制并发会话数

openclaw config.patch '{"gateway": {"maxConcurrentSessions": 10}}'

2️⃣ 定期清理旧会话

设置会话超时时间（默认 1 小时）：

openclaw config.patch '{"gateway": {"sessionTimeout": 3600}}'

3️⃣ 禁用不必要的插件

查看已启用插件：

openclaw config.get plugins.entries

禁用不用的：

openclaw config.patch '{"plugins": {"entries": {"slack": {"enabled": false}}}}'

━ 🚀 优化 3：日志管理 ━

问题：日志文件越来越大，占满磁盘

解决方案

1️⃣ 设置日志级别

生产环境用 info 或 warn：

openclaw config.patch '{"gateway": {"logLevel": "info"}}'

2️⃣ 启用日志轮转

自动清理旧日志：

openclaw config.patch '{"gateway": {"logRotation": {"maxFiles": 7, "maxSize": "10m"}}}'

3️⃣ 手动清理

rm ~/.openclaw/logs/*.log.old

truncate -s 0 ~/.openclaw/logs/gateway.log # 清空当前日志

━ 🚀 优化 4：网络请求优化 ━

问题：网络请求慢，经常超时

解决方案

1️⃣ 调整超时时间

openclaw config.patch '{"agents": {"requestTimeout": 60000}}' # 60秒

2️⃣ 启用请求重试

openclaw config.patch '{"agents": {"retry": {"maxRetries": 3, "retryDelay": 1000}}}'

3️⃣ 使用代理加速（海外模型）

openclaw config.patch '{"agents": {"proxy": "http://127.0.0.1:7890"}}'

━ 🚀 优化 5：定时任务优化 ━

问题：多个定时任务同时执行，系统卡顿

解决方案

1️⃣ 错开任务时间

避免多个任务在同一时间触发：

• 任务 A：9:00

• 任务 B：9:15

• 任务 C：9:30

2️⃣ 限制并发任务数

openclaw config.patch '{"gateway": {"maxConcurrentCrons": 2}}'

3️⃣ 使用轻量模型执行后台任务

后台任务不需要最强模型：

openclaw cron create --schedule "0 9 * * *" --model "zai/glm-4-flash" --prompt "..."

━ 🚀 优化 6：启动速度优化 ━

问题：Gateway 启动慢，等太久

解决方案

1️⃣ 延迟加载插件

openclaw config.patch '{"plugins": {"lazyLoad": true}}'

2️⃣ 禁用启动时检查

openclaw config.patch '{"gateway": {"skipStartupChecks": true}}'

━ 📊 性能监控 ━

优化后记得监控效果：

1️⃣ 查看资源占用

top -pid $(pgrep -f "openclaw gateway")

2️⃣ 查看响应时间

在日志中搜索响应时间：

grep "response time" ~/.openclaw/logs/gateway.log | tail -20

3️⃣ 查看会话统计

openclaw status
