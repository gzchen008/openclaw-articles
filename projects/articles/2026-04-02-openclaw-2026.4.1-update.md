# OpenClaw 2026.4.1 正式发布：重大功能更新与性能优化

🎉 **OpenClaw 2026.4.1** 已于 2026年4月1日 正式发布！本次更新带来了多项重磅功能改进和性能优化，为开发者们提供了更强大的 AI 编程助手体验。

## 🚀 核心功能更新

### 1. 任务管理革新：/tasks 聊天原生任务板
- **新增** `/tasks` 命令，提供聊天原生的后台任务展示板
- 实时显示当前会话的任务详情
- 当无关联任务可见时，显示 agent 本地备用计数
- 相关 Issue: #54226
- 贡献者：@vincentkoc

### 2. Web 搜索增强：集成 SearXNG 提供商
- **新增** 捆绑的 SearXNG 提供商插件
- 支持可配置的主机设置
- 提供更强大的本地化搜索能力
- 相关 Issue: #57317
- 贡献者：@cgdusek

### 3. Amazon Bedrock 集成：Guardrails 支持
- **新增** Amazon Bedrock Guardrails 支持
- 为企业级应用提供内容安全防护
- 相关 Issue: #58588
- 贡献者：@MikeORed

### 4. macOS 语音唤醒：Talk Mode 激活
- **新增** 语音唤醒选项，可触发 Talk Mode
- 提升macOS用户的交互体验
- 相关 Issue: #58490
- 贡献者：@SmoothExec

### 5. 飞书评论系统增强
- **新增** 飞书Drive评论事件流
- 支持评论线程上下文解析
- 内联回复功能
- feishu_drive 评论动作支持
- 完善的文档协作工作流
- 相关 Issue: #58497
- 贡献者：@wittam-01

### 6. 网页聊天配置优化
- **新增** `gateway.webchat.chatHistoryMaxChars` 配置
- 支持每个请求的 `maxChars` 设置
- 保持静默回复过滤和现有默认载荷限制

### 7. Agent 默认参数
- **新增** `agents.defaults.params` 全局默认提供商参数
- 统一配置管理
- 相关 Issue: #58548
- 贡献者：@lpender

### 8. 容错机制改进
- **新增** `auth.cooldowns.rateLimitedProfileRotations` 配置
- 限制提供商配置文件的轮换次数
- 改进失败回退行为文档
- 相关 Issue：#58707
- 贡献者：@Forgely3D

### 9. Cron 工具权限管理
- **新增** `openclaw cron --tools` 每个作业的工具权限列表
- 更精细化的权限控制
- 相关 Issue: #58504
- 贡献者：@andyk-ms

### 10. 会话路由优化
- 移动提供商特定的会话对话语法到插件拥有的会话键表面
- 保持 Telegram 主题路由和飞书范围继承

### 11. WhatsApp 反应支持
- **新增** `reactionLevel` 指导，用于 AI 反应
- 提供更自然的对话交互
- 贡献者：@mcaxtr

### 12. Telegram 错误处理
- **新增** 可配置的 `errorPolicy` 和 `errorCooldownMs` 控制
- 可按账户、聊天和主题抑制重复传递错误
- 相关 Issue: #51914
- 贡献者：@chinar-amrutkar

### 13. Z.AI 模型扩展
- **新增** `glm-5.1` 和 `glm-5v-turbo` 到捆绑 Z.AI 提供商目录
- 相关 Issue: #58793
- 贡献者：@tomsun28

### 14. 压缩模型一致性
- 确保 `/compact` 和其他上下文引擎压缩路径使用一致的配置覆盖模型
- 相关 Issue: #56710
- 贡献者：@oliviareid-svg

## 🔧 性能优化与问题修复

### 主要修复内容

#### 1. 聊天错误回复改进
- 停止将原始提供商/运行时失败泄露到外部聊天频道
- 返回友好的重试消息
- 为 Bedrock toolResult/toolUse 会话不匹配添加特定的 /new 提示
- 相关 Issue: #58831
- 贡献者：@ImLukeF

#### 2. 网关重载优化
- 在配置重载器中忽略启动配置写入，通过持久哈希
- 防止生成认证令牌和播种的 Control UI 原因触发重启循环
- 相关 Issue: #58678
- 贡献者：@yelog

#### 3. 任务网关性能
- 保持任务注册表维护清理不在同步 SQLite 压力下停滞网关事件循环
- 相关 Issue: #58670
- 贡献者：@openperf

#### 4. 任务状态显示
- 从 /status 和 session_status 隐藏过期的已完成后台任务
- 优先显示实时任务上下文
- 仅在没有活动工作时显示最近失败
- 相关 Issue: #58661
- 贡献者：@vincentkoc

#### 5. 执行批准改进
- 在内联或配置的工具策略未设置时，遵守 exec-approvals.json 安全默认值
- 保持 Slack 和 Discord 原生批准处理与推断批准者和实际频道启用一致
- 相关 Issue: #58679
- 贡献者：@scoootscooob 和 @vincentkoc

## 🛡️ 安全性增强

### 1. 执行批准持久化
- 将 `allow-always` 持久化为持久的用户批准信任
- 在无法持久化可执行允许列表条目的 shell 包装程序路径上重用确切的命令信任
- 保持静态允许列表条目不无声地绕过 ask:"always"
- 当 Windows 无法构建可执行允许列表执行计划时要求显式批准
- 贡献者：@scoootscooob 和 @vincentkoc

### 2. 网络沙盒改进
- 通过共享的沙盒策略清理 SSH 子进程环境变量
- 将市场存档下载和 Ollama 发现、认证和拉取请求通过受保护的 fetch 路由
- 确保沙盒执行和远程获取遵循仓库的信任边界

## 📊 用户体验提升

### 1. 会话模型切换
- 保持 /model 更改在繁忙运行后排队，而不是中断活动回合
- 重新定位排队的后续操作，以便稍后工作在当前回合完成后立即采用新模型

### 2. Web UI 开放响应
- 在网页聊天中保持重写的流式快照
- 当模型回溯早期输出时保持开放响应最终流式文本对齐
- 相关 Issue: #58641
- 贡献者：@neeravmakwana

### 3. Discord 入站媒体
- 将 Discord 附件和贴纸下载通过共享空闲超时和工作程序中止路径传递
- 防止缓慢或卡住的入站媒体获取挂起消息处理
- 相关 Issue: #58593
- 贡献者：@aquaright1

### 4. 记忆系统优化
- 保持完整的重新索引在会话启动或触发时不跳过会话转录
- 确保重新驱动的重新索引保留会话记忆
- 相关 Issue: #39732
- 贡献者：@upupc

### 5. QMD 收集模式
- 在创建 QMD 收集时优先使用 --mask 而不是 --glob
- 保持默认记忆收集的预期模式，防止在重启时冲突
- 相关 Issue: #58643
- 贡献者：@GitZhangChi

## 🎯 开发者工具改进

### 1. ACP 安全性
- 用语义批准类替换 ACP 的危险工具名称覆盖
- 只有窄只读读取/搜索可以自动批准，而间接执行能力和控制平面工具总是需要显式提示批准
- 贡献者：@vincentkoc

### 2. ACP 会话注册
- 为 ACP 子运行注册完成跟踪和生命周期清理
- 使注册失败清理明确为尽力而为，调用者不应假设已启动的 ACP 回合被完全中止
- 相关 Issue: #40885
- 贡献者：@xaeon2026 和 @vincentkoc

### 3. ACPX 运行时
- 从扩展包元数据派生捆绑的 ACPX 预期版本，而不是硬编码单独的字面量
- 防止插件本地 ACPX 安装在版本提升后失去健康检查对等
- 相关 Issue: #49089
- 贡献者：@jiejiesks 和 @vincentkoc

## 📋 更新指南

### 升级步骤
1. **备份数据**：升级前请确保备份重要配置和项目文件
2. **执行升级**：运行 `openclaw upgrade` 或下载新版本安装包
3. **检查配置**：升级后检查 `openclaw.json` 配置文件兼容性
4. **测试功能**：重点测试 `/tasks` 命令和新的搜索功能

### 配置更新
- 新增 `agents.defaults.params` 配置项
- 新增 `gateway.webchat.chatHistoryMaxChars` 配置项
- 新增 `auth.cooldowns.rateLimitedProfileRotations` 配置项

## 🔄 版本兼容性

- **向后兼容**：2026.4.1 向后兼容之前的配置文件格式
- **插件兼容**：所有现有插件继续正常工作
- **数据迁移**：自动处理配置文件升级和数据迁移
