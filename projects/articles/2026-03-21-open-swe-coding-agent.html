━ 什么是 Open SWE？ ━

Open SWE 是 LangChain 团队开源的异步编程代理框架，灵感来自 Stripe、Ramp、Coinbase 等顶级公司的内部 coding agent 架构。

核心数据：

1. ⭐ 7,657 Stars（今日新增 635 ⭐）

2. 🐍 Python 编写

3. 🏢 LangChain 官方维护

4. 📈 社区活跃，持续更新

一句话概括：让你用顶级公司的架构，打造自己的内部编程助手。

━ 为什么它这么火？ ━

1. 生产级架构，不是玩具

它采用了顶级公司验证过的设计模式：

1. 云沙箱隔离：每个任务在独立的云环境中运行，互不干扰

2. 精选工具集：15+ 工具，包括 shell 执行、网页抓取、Git 操作、PR 创建等

3. 中间件钩子：可以在任务执行的任何阶段插入自定义逻辑

这不是一个简单的脚本，而是一个可以投入生产使用的完整框架。

2. 异步并行，效率翻倍

传统 coding agent 是串行的：一次只能处理一个任务，后面的任务要排队。

Open SWE 是异步并行的：

1. ✅ 可以同时运行多个任务

2. ✅ 每个任务独立沙箱，互不影响

3. ✅ 任务执行中还能接收新消息

这意味着：你可以在 Slack 里连续发 10 个任务，它会同时处理，而不是一个一个来。

3. 多平台触发，随时随地用

支持多种触发方式：

1. 💬 Slack：在聊天中直接调用

2. 🎫 Linear：从任务管理工具触发

3. 💻 GitHub 评论：在代码审查时触发

4. 🔧 API：自定义集成

实际使用示例：

@openswe repo:owner/name 请帮我重构这个函数

就这么简单，不用切换工具，不用开 IDE。

4. 上下文感知，懂你的项目

它会读取项目根目录的 AGENTS.md 文件，自动理解你的项目规范：

# AGENTS.md

## 代码规范

- 使用 TypeScript

- 遵循 ESLint 规则

- 函数必须有单元测试

## Git 规范

- commit message 格式： type(scope): description

- PR 必须关联 issue

这样它写出的代码，天然符合你的团队风格。

5. 高度可定制，想改就改

Open SWE 的设计哲学：一切皆可替换。

你可以替换：

1. 🏗️ 沙箱提供商：Modal、Daytona、Runloop，或者自己的服务器

2. 🤖 模型：Claude、GPT-4、开源模型，随便换

3. 🔧 工具：添加你自己的工具

4. 🎯 触发器：自定义触发方式

5. 🔌 中间件：插入你的逻辑

创建基础 Agent 只需几行代码：

from open_swe import create_deep_agent

agent = create_deep_agent(

model="anthropic:claude-opus-4-6",

system_prompt="你的系统提示...",

tools=[http_request, fetch_url, commit_and_open_pr],

backend=sandbox_backend,

middleware=[ToolErrorMiddleware()],

)

━ 快速开始：5 步搭建你的编程助手 ━

第 1 步：克隆仓库

git clone https://github.com/langchain-ai/open-swe.git

cd open-swe

第 2 步:安装依赖

pip install -r requirements.txt

第 3 步:配置环境

需要配置 3 个东西：

1. GitHub App：用于 OAuth 和 PR 创建

2. LangSmith：用于沙箱和监控

3. 触发器：Slack / Linear / GitHub

第 4 步:创建 Agent

from open_swe import create_deep_agent

from open_swe.tools import http_request, fetch_url, commit_and_open_pr

from open_swe.middleware import ToolErrorMiddleware

agent = create_deep_agent(

model="anthropic:claude-opus-4-6",

system_prompt="你是一个专业的编程助手...",

tools=[http_request, fetch_url, commit_and_open_pr],

backend=sandbox_backend,

middleware=[ToolErrorMiddleware()],

)

第 5 步:在 Slack 中使用

@openswe repo:owner/name 请帮我重构这个函数

就这么简单！

━ 为什么选择 Open SWE？ ━

vs. GitHub Copilot

1. 触发方式：Open SWE 支持 Slack/Linear/GitHub 评论， Copilot 只能在 IDE 内

2. 任务类型：Open SWE 处理复杂多步骤任务， Copilot 只做单行/多行补全

3. 并行能力：Open SWE 异步并行 ✅， Copilot 串行 ❌

4. 定制性：Open SWE 高度可定制 ✅, Copilot 固定功能 ❌

5. 成本：Open SWE 自己部署（更灵活），Copilot 订阅制

结论：Copilot 是你的副驾驶，Open SWE 是你的团队助手。

vs. Cursor

1. 使用场景：Open SWE 适合团队协作， Cursor 适合个人开发

2. 触发方式：Open SWE 多平台触发， Cursor 只能在 IDE 内

3. 异步能力：Open SWE 支持 ✅， Cursor 不支持 ❌

4. 定制性：Open SWE 开源可定制 ✅, Cursor 商业产品 ❌

结论：Cursor 适合个人，Open SWE 适合团队。
