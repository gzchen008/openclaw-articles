# OpenClaw v2026.4.2 发布：重大更新与功能增强

**发布日期：** 2026年4月3日  
**版本号：** v2026.4.2  
**更新类型：** 重大功能更新

## 🎉 版本概览

OpenClaw v2026.4.2 是一个重要的版本更新，带来了大量功能增强、性能优化和重要的架构改进。本次更新重点关注插件系统重构、任务流管理、安全性增强以及多平台支持改进。

## 🚀 主要新特性

### 1. 插件系统重大重构

- **配置路径标准化**：将 x_search 和 web_fetch 配置从核心工具迁移到插件专属路径
- **配置迁移工具**：新增 `openclaw doctor --fix` 命令自动迁移旧配置
- **统一的认证机制**：标准化了 API 密钥和认证配置

### 2. 任务流管理 (Task Flow) 系统

- **核心任务流 substrate**：恢复并增强了核心任务流管理功能
- **托管与镜像同步模式**：支持灵活的任务流同步策略
- **持久化状态跟踪**：任务流状态和版本跟踪功能
- **`openclaw flows` 检查/恢复工具**：提供任务流管理界面

### 3. 安全性重大改进

- **传输策略集中化**：统一请求认证、代理、TLS 和头部处理
- **安全性验证**：阻止不安全的 TLS 和运行时传输覆盖
- **执行权限系统**：改进了 exec 批准流程和安全策略

### 4. 多平台增强

- **Android 助手集成**：新增 Google Assistant App Actions 支持
- **Android 入口点**：支持从 Assistant 启动 OpenClaw
- **多渠道支持改进**：Telegram、Feishu、Matrix 等渠道的功能增强

## 🔧 详细更新内容

### Breaking Changes

#### 插件配置迁移
- **xAI 插件**：将 `x_search` 设置从 `tools.web.x_search.*` 迁移到 `plugins.entries.xai.config.xSearch.*`
- **认证标准化**：统一使用 `plugins.entries.xai.config.webSearch.apiKey` / `XAI_API_KEY`
- **Web Fetch 插件**：将 Firecrawl 配置从 `tools.web.fetch.firecrawl.*` 迁移到 `plugins.entries.firecrawl.config.webFetch.*`

### 功能增强

#### 任务流管理
- **托管子任务生成**：支持外部协调器停止调度
- **粘性取消意图**：任务流取消机制改进
- **API 接口**：新增 `api.runtime.taskFlow` 供插件使用

#### Android 集成
- **助手角色入口**：新增 assistant-role entrypoints
- **App Actions 元数据**：支持从 Assistant 启动
- **提示传递**：支持将提示传递到聊天编辑器

#### 执行系统改进
- **默认安全模式**：网关/节点主机执行默认为 YOLO 模式
- **提供者回放钩子**：支持转录策略、回放清理和推理模式调度
- **插件钩子**：新增 `before_agent_reply` 钩子

### 平台支持改进

#### Feishu 平台
- **评论系统增强**：专门的 Drive 评论事件流
- **评论上下文解析**：支持话题回复
- **评论操作**：新增文档协作工作流

#### Matrix 平台
- **标准化提及**：符合规范的 `m.mentions` 元数据
- **客户端兼容**：Element 等客户端的可靠通知

#### 其他改进
- **差异查看器**：插件专属的 `viewerBaseUrl`
- **代理支持**：稳定的代理/公共源支持
- **压缩算法**：一致的模型配置解析

### Bug 修复

#### 安全性修复
- **传输策略**：集中化 HTTP、流和 WebSocket 路径
- **Copilot 提供**：分类本机 GitHub Copilot API 主机
- **流式传输头部**：统一 OpenAI websocket 和代理流路径

#### 执行系统修复
- **网关/节点回退**：恢复空配对设备令牌映射的传统角色回退
- **子agent 权限**：固定管理员子agent 网关调用
- **执行批准**：规范化无效的安全和询问值

#### 平台特定修复
- **Slack 格式化**：内置 Slack mrkdwn 指导
- **WhatsApp 状态**：自聊模式下发送不可用状态
- **Matrix 预览**：保持实时部分预览
- **Feishu 评论**：加强文档评论线程传递

## 🛠️ 升级建议

### 必需操作
1. **配置迁移**：运行 `openclaw doctor --fix` 迁移旧配置
2. **重启服务**：升级后重启 OpenClaw 服务
3. **权限检查**：验证执行策略和权限设置

### 兼容性注意事项
- **插件配置**：旧配置路径将被废弃，请及时迁移
- **API 变更**：部分 API 接口有所调整
- **依赖更新**：某些外部依赖可能需要更新版本

### 性能优化建议
- 启用新的任务流管理功能以获得更好的性能
- 考虑启用新的压缩算法以减少内存使用
- 利用改进的缓存机制提高响应速度
