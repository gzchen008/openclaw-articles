━ 📋 实战案例 ━

案例 1：服务器健康检查

场景：每天早上自动检查服务器状态，发现问题立即通知

实现步骤：

Step 1：配置 Cron 任务

openclaw cron add \
--name "daily-health-check" \
--schedule "0 9 * * *" \
--task "检查服务器状态：CPU、内存、磁盘、Docker 容器" \
--channel discord

Step 2：设置检查规则

如果 CPU > 80%，发送警告
如果磁盘剩余 < 10%，发送紧急通知
如果有容器停止，自动重启并通知

Step 3：测试

openclaw cron run daily-health-check

每天早上 9 点，你会收到一份服务器健康报告：

📊 服务器健康报告（2026-03-03 09:00）
✅ CPU: 35%
✅ 内存: 62%
⚠️ 磁盘: 89%（剩余 11GB）
✅ Docker: 5/5 容器运行中

案例 2：日志异常检测

场景：自动分析日志，发现异常模式

实现步骤：

Step 1：配置日志监控

每 30 分钟检查一次 nginx 日志
重点关注：5xx 错误、异常 IP、攻击特征

Step 2：设置告警规则

如果同一 IP 1 分钟内请求 > 100 次，标记为异常
如果 5xx 错误 > 10 次/分钟，发送紧急通知

效果：

• 自动识别 DDoS 攻击

• 发现爬虫异常行为

• 统计 API 错误率

案例 3：自动备份与恢复

场景：每天自动备份，失败立即通知

实现步骤：

Step 1：配置备份任务

openclaw cron add \
--name "daily-backup" \
--schedule "0 3 * * *" \
--task "备份 MySQL 数据库到 /backup/ 目录，保留最近 7 天" \
--channel discord

Step 2：添加失败处理

如果备份失败：
1. 发送紧急通知
2. 记录错误日志
3. 尝试重新备份（最多 3 次）

效果：

• 每天凌晨 3 点自动备份

• 备份失败立即知道

• 7 天后自动清理旧备份

━ 🚀 快速上手 ━

1. 安装 OpenClaw

npm install -g openclaw
openclaw gateway start

2. 配置通知渠道

# Discord
openclaw config set discord.webhook YOUR_WEBHOOK_URL

# Telegram
openclaw config set telegram.token YOUR_BOT_TOKEN

3. 创建第一个任务

openclaw cron add \
--name "my-first-task" \
--schedule "*/5 * * * *" \
--task "检查服务器 CPU 和内存使用率" \
--channel discord

4. 查看任务状态

openclaw cron list
openclaw cron logs my-first-task

━ 💡 实用技巧 ━

技巧 1：分层告警

不要所有问题都发紧急通知，设置优先级：

• 紧急：服务器宕机、数据库崩溃（电话 + Discord）

• 警告：CPU > 80%、磁盘 > 90%（Discord）

• 信息：日常备份成功、定期报告（仅记录）

技巧 2：定期报告

每周/每月自动生成报告：

每周一早上发送上周服务器运行报告
每月 1 号发送上月故障统计

技巧 3：知识库积累

把解决方案记录下来：

如果遇到 "Connection refused" 错误，检查：
1. 服务是否启动
2. 防火墙是否开放端口
3. 配置文件是否正确

下次遇到同样问题，AI 直接给你解决方案。

━ ⚠️ 注意事项 ━

1. 权限控制

OpenClaw 有服务器访问权限，务必：

• 配置授权发送者（只允许你的账号）

• 设置操作白名单（禁止危险命令）

• 定期审计日志

2. 成本控制

AI 调用有成本，建议：

• 非紧急任务使用便宜模型（如 GLM-4）

• 设置每日调用上限

• 监控 Token 消耗

3. 数据安全

敏感数据不要明文传输：

• API Key 存储在配置文件（不提交 Git）

• 日志中脱敏处理

• 使用 HTTPS 加密通信
