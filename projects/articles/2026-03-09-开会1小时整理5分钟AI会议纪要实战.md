### 方案二：语音转写 + AI 总结（推荐）

**步骤：**

1. 开会时用手机录音
2. 会后用语音转文字工具（如讯飞、通义听悟）
3. 把转写文本发给 OpenClaw
4. AI 自动生成结构化纪要

**示例 Prompt：**

```
请帮我整理以下会议记录，生成结构化纪要：

[粘贴转写文本]

要求：
1. 提炼核心议题（不超过 5 个）
2. 列出关键决议
3. 明确行动项（谁、做什么、截止时间）
4. 控制在 500 字以内
```

**优点：** 适合长会，准确度高
**缺点：** 需要额外录音工具

## 💡 实战技巧

### 技巧一：结构化 Prompt

不要只说"帮我整理会议纪要"，给 AI 更多上下文：

```
会议类型：周会 / 评审会 / 头脑风暴
参会角色：产品、开发、设计、运营
重点关注：决议 / 行动项 / 风险
输出格式：Markdown / 邮件 / Notion
```

### 技巧二：分批处理长会议

2 小时的会议转写可能有 1 万字，建议：

• 先让 AI 总结第一部分
• 确认方向正确
• 再处理剩余部分

### 技巧三：建立会议模板

在 OpenClaw 中保存常用模板：

```
# 会议纪要模板

📋 会议主题：
📅 会议时间：
👥 参会人员：
🎯 核心议题：
  1. ...
  2. ...
✅ 关键决议：
  • ...
📝 行动项：
  • [ ] 张三：完成 xxx，截止 3/15
  • [ ] 李四：准备 yyy，截止 3/20
```

每次开会直接套用，效率翻倍。
