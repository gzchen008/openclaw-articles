━ 🔍 问题 1：Gateway 启动失败 ━

症状

• openclaw gateway start 报错

• 端口被占用

• 权限不足

排查步骤

1️⃣ 检查端口占用

lsof -i :443

如果被占用，杀掉进程或改端口：

kill -9 <PID>

2️⃣ 检查配置文件

cat ~/.openclaw/openclaw.json

确认 JSON 格式正确（用 jq 验证）：

cat ~/.openclaw/openclaw.json | jq .

3️⃣ 查看详细日志

openclaw gateway start --verbose

━ 🔍 问题 2：AI 不回消息 ━

症状

• 消息发出去了，AI 没反应

• 群里 @机器人 不响应

• 私聊也没回复

排查步骤

1️⃣ 检查 Gateway 状态

openclaw gateway status

应该显示 running，如果是 stopped 就启动它。

2️⃣ 检查渠道配置

以飞书为例，确认配置正确：

openclaw config.get plugins.entries.lark

3️⃣ 检查模型配置

确认 API Key 有效：

openclaw config.get agents.defaults.model

openclaw config.get agents.authProfiles

4️⃣ 查看会话日志

tail -f ~/.openclaw/logs/gateway.log

看看有没有报错信息。

━ 🔍 问题 3：模型调用失败 ━

症状

• All models failed

• Provider in cooldown

• 401 Unauthorized

排查步骤

1️⃣ 检查 API Key

测试 API Key 是否有效（以智谱 AI 为例）：

curl -X POST "https://open.bigmodel.cn/api/paas/v4/chat/completions" \

-H "Authorization: Bearer YOUR_API_KEY" \

-H "Content-Type: application/json" \

-d '{"model": "glm-4", "messages": [{"role": "user", "content": "hi"}]}'

2️⃣ 检查余额

登录对应平台查看账户余额是否充足。

3️⃣ 处理限流

如果看到 cooldown，说明被限流了：

• 等待几分钟自动恢复

• 配置多个模型作为 fallback

• 降低调用频率

4️⃣ 配置多模型 fallback

在 openclaw.json 中配置：

{

"agents": {

"defaults": {

"model": "zai/glm-5"

},

"modelFallback": ["zai/glm-4.7", "kimi-coding/k2p5"]

}

}

━ 🔍 问题 4：飞书/Discord 配置不生效 ━

症状

• 配置改了，但行为没变

• 新频道没响应

• 权限配置错误

排查步骤

1️⃣ 重启 Gateway

配置修改后需要重启：

openclaw gateway restart

2️⃣ 检查插件状态

openclaw config.get plugins.entries.lark.enabled

应该是 true。

3️⃣ 检查频道白名单

如果配置了白名单，新频道需要手动添加：

openclaw config.patch '{"plugins": {"entries": {"lark": {"config": {"allowedChats": ["oc_xxx", "ou_xxx"]}}}}}'

4️⃣ 检查应用权限

飞书应用需要配置正确的权限范围，包括：

• 获取用户信息

• 接收消息

• 发送消息

• 读取群聊信息

━ 🔍 问题 5：定时任务不执行 ━

症状

• Cron 任务设置了但没触发

• 任务执行失败

• 时区不对

排查步骤

1️⃣ 查看任务列表

openclaw cron list

2️⃣ 检查时区设置

openclaw config.get gateway.timezone

确保是 Asia/Shanghai 或你的本地时区。

3️⃣ 检查任务状态

openclaw cron status <task-id>

4️⃣ 手动触发测试

openclaw cron trigger <task-id>

━ 🛠️ 万能救急命令 ━

遇到问题先试试这几个：

1️⃣ 重启大法

openclaw gateway restart

2️⃣ 查看完整状态

openclaw status

3️⃣ 查看日志

tail -100 ~/.openclaw/logs/gateway.log

4️⃣ 检查版本

openclaw --version

npm update -g openclaw # 更新到最新版
