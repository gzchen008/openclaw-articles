智能家居新玩法：用 OpenClaw 语音控制全屋设备

想象一下，躺在床上说一句「小J，关灯」，全屋灯光自动熄灭...

🏠 为什么需要 AI + 智能家居？

传统的智能家居有几个痛点：

1. 语音助手太笨 - 只能听懂固定指令，稍微换个说法就「我不明白」

2. 场景设置复杂 - 想配置一个「回家模式」，要在一堆 App 里点半天

3. 设备不互通 - 小米的灯、涂鸦的插座、HomeKit 的传感器...各玩各的

4. 缺乏「脑子」 - 智能设备只会按规则执行，不会主动思考

OpenClaw + Home Assistant 的组合，能解决这些问题：

✅ 自然语言控制：说什么都能听懂

✅ 智能场景推荐：AI 帮你设计自动化

✅ 统一管理：一个入口控制所有设备

✅ 主动建议：AI 会根据你的习惯提出优化方案

🛠️ 配置步骤

前置条件

• OpenClaw 已安装并正常运行

• Home Assistant 已部署（支持 Docker、树莓派、NAS 等）

• 智能设备已接入 Home Assistant

步骤 1：安装 Home Assistant

方式 A：Docker 部署（推荐）

docker run -d \
  --name homeassistant \
  --privileged \
  --restart=unless-stopped \
  -e TZ=Asia/Shanghai \
  -v /path/to/config:/config \
  -p 8123:8123 \
  homeassistant/home-assistant:stable

方式 B：树莓派

1. 下载 Home Assistant OS 镜像

2. 烧录到 SD 卡

3. 插卡启动，访问 http://homeassistant.local:8123

步骤 3：配置 OpenClaw 访问 Home Assistant

方式 A：通过 REST API

在 OpenClaw 中配置一个自定义命令：

{
  "name": "control_light",
  "description": "控制智能灯光",
  "execute": "curl -X POST http://homeassistant:8123/api/services/light/turn_on \
    -H 'Authorization: Bearer YOUR_TOKEN' \
    -H 'Content-Type: application/json' \
    -d '{\"entity_id\": \"light.living_room\", \"brightness\": 200}'"
}

方式 B：使用 Skills 开发

创建一个 homeassistant skill，封装常用操作：

# skills/homeassistant/scripts/ha_control.py
import requests

HA_URL = "http://homeassistant:8123"
HA_TOKEN = "YOUR_LONG_LIVED_ACCESS_TOKEN"

def turn_on_light(entity_id):
    """打开灯光"""
    url = f"{HA_URL}/api/services/light/turn_on"
    headers = {"Authorization": f"Bearer {HA_TOKEN}"}
    data = {"entity_id": entity_id}
    requests.post(url, headers=headers, json=data)
    return f"已打开 {entity_id}"

def set_temperature(entity_id, temp):
    """设置空调温度"""
    url = f"{HA_URL}/api/services/climate/set_temperature"
    headers = {"Authorization": f"Bearer {HA_TOKEN}"}
    data = {"entity_id": entity_id, "temperature": temp}
    requests.post(url, headers=headers, json=data)
    return f"已将 {entity_id} 设置为 {temp} 度"

步骤 4：创建智能场景

场景 1：回家模式

在 Home Assistant 中创建自动化：

# configuration.yaml
automation:
  - alias: "回家模式"
    trigger:
      - platform: state
        entity_id: device_tracker.your_phone
        to: "home"
    action:
      - service: light.turn_on
        entity_id: light.entrance
      - service: climate.set_temperature
        data:
          entity_id: climate.living_room
          temperature: 26

场景 2：睡眠模式

automation:
  - alias: "睡眠模式"
    trigger:
      - platform: time
        at: "23:30"
    action:
      - service: light.turn_off
        entity_id: all
      - service: climate.set_temperature
        data:
          entity_id: climate.bedroom
          temperature: 24

⚠️ 避坑指南

1. 网络稳定性

问题：设备经常离线，控制失灵

解决：

• 使用有线连接 Home Assistant

• 智能设备尽量接入 2.4G WiFi（穿墙更好）

• 配置 UPS 防止断电

2. 安全问题

问题：担心智能家居被黑客控制

解决：

• 不要把 Home Assistant 暴露在公网

• 使用 VPN 或 Tailscale 远程访问

• 定期更新 Home Assistant 版本

• 使用强密码 + 双因素认证

3. 设备兼容性

问题：有些设备 Home Assistant 不支持

解决：

• 查看官方集成列表：home-assistant.io/integrations

• 使用 MQTT 网关转换

• 考虑购买支持 Zigbee/Z-Wave 的设备（通用性更好）

4. 语音识别准确率

问题：方言或口音导致识别错误

解决：

• 训练 OpenClaw 理解你的说话习惯

• 使用简短清晰的指令

• 为常用场景设置别名
