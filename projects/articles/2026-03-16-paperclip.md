# 如果 OpenClaw 是员工，Paperclip 就是公司 🏢

## 你的 AI 员工需要一个 CEO

想象一下，你雇佣了 20 个 AI Agent：
- 3 个负责写代码
- 2 个做运营
- 5 个做内容
- 剩下的负责客服、财务、测试...

但问题来了：**谁管理这些 Agent？谁分配任务？谁控制预算？谁确保大家都在为同一个目标努力？**

答案是：**Paperclip**。

---

## 什么是 Paperclip？

**Paperclip** 是一个开源的 AI Agent 编排平台，让你的 AI 员工像真正的公司一样运作。

<p style="padding-left: 20px; margin: 4px 0;">• 🏢 组织架构：CEO、CTO、工程师、设计师... 每个角色都有明确的职责</p>
<p style="padding-left: 20px; margin: 4px 0;">• 💰 预算控制：每个 Agent 都有月度预算，超限自动停止</p>
<p style="padding-left: 20px; margin: 4px 0;">• 🎯 目标对齐：所有任务都追溯到公司使命，Agent 知道"为什么"而战</p>
<p style="padding-left: 20px; margin: 4px 0;">• 📊 成本追踪：实时监控每个 Agent 的 token 消耗</p>
<p style="padding-left: 20px; margin: 4px 0;">• 🔍 审计日志：每个决策都有记录，可追溯可解释</p>
<p style="padding-left: 20px; margin: 4px 0;">• ⏰ 心跳机制：定时唤醒 Agent 检查工作，自动执行</p>

**一句话：Paperclip 是 AI 企业的操作系统。**

## 核心特性

### 🏢 真正的组织架构

不是简单的任务列表，而是**真正的公司结构**：
- 层级关系、角色定义、汇报线
- Agent 有老板、有头衔、有职责
- 委派从上到下流动

### 🎯 目标对齐执行

每个任务都携带**完整的目标链路**：
- Agent 不仅能看到任务标题
- 还能看到"为什么"要做这件事
- 所有工作追溯到公司使命

### 💰 预算与成本控制

- 每个 Agent 都有月度预算
- 达到限制时自动停止
- 管理层可以随时调整优先级
- **再也不担心 Agent 疯狂烧钱**

### ⏰ 心跳机制

- Agent 定时唤醒，检查工作，自动执行
- 适合：客服、社媒、报告、监控等定期任务
- 不需要你手动触发

### 🔍 完整审计

- 每个对话可追溯
- 每个决策有解释
- 工具调用全记录
- 不可篡改的审计日志

### 🏢 多公司隔离

一次部署，运行多个公司：
- 完全的数据隔离
- 独立的审计轨迹
- 一个控制面板管理整个"帝国"

## Paperclip 不是什么

<p style="padding-left: 20px; margin: 4px 0;">🚫 不是聊天机器人 — Agent 有工作，不是聊天窗口</p>
<p style="padding-left: 20px; margin: 4px 0;">🚫 不是 Agent 框架 — 不教你怎么造 Agent，教你怎么运营 Agent 公司</p>
<p style="padding-left: 20px; margin: 4px 0;">🚫 不是工作流构建器 — 没有拖拽式管道，是真正的公司模型</p>
<p style="padding-left: 20px; margin: 4px 0;">🚫 不是提示词管理器 — Agent 自带提示词/模型/运行时，Paperclip 管组织</p>
<p style="padding-left: 20px; margin: 4px 0;">🚫 不是单 Agent 工具 — 这是给团队用的。1 个 Agent 不需要，20 个 Agent 必须要</p>

## 技术亮点

### 原子执行

任务领取和预算执行是原子操作：
- 不会重复工作
- 不会超支

### 持久化状态

Agent 跨心跳恢复同一任务上下文，不用每次从零开始。

### 运行时技能注入

Agent 可以在运行时学习 Paperclip 工作流和项目上下文，无需重新训练。

### 治理与回滚

审批门强制执行，配置变更版本化，坏变更可安全回滚。

### 可移植公司模板

导入/导出组织、Agent、技能，自动清理敏感信息，处理命名冲突。
