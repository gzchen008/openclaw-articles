## AI 会议纪要的 3 种方案

### 方案 1️⃣：OpenClaw + 语音转文字（推荐）

**适用场景**：线上会议（腾讯会议、Zoom、飞书）

**工作流程**：
1. 会议开始，开启录音/字幕
2. 会后导出文字记录
3. 发给 OpenClaw，自动生成纪要

**Prompt 模板**：
```
请根据以下会议记录，生成结构化纪要：

【会议记录】
{粘贴文字记录}

【输出格式】
1. 会议主题

4. 关键决策
5. 行动项（包含负责人和截止日期）
```

**优势**：
- ✅ 完全自动化，无需人工整理
- ✅ 格式统一，便于归档
- ✅ 支持多种会议平台

## 实战案例：周会纪要自动化

### 场景描述

每周一上午 10 点，团队有固定周会：
- 5-8 人参会
- 时长 30-60 分钟
- 使用飞书会议

### 实现方案

**Step 1：创建 OpenClaw 定时任务**

```bash
# 每周一 10:00 自动提醒
openclaw cron add \
  --schedule "0 10 * * 1" \
  --message "周会开始，请打开飞书字幕"
```

**Step 2：会议结束后，发送记录给 OpenClaw**

```
用户：@OpenClaw 请整理今天的周会记录
[粘贴飞书导出的字幕]
```

**Step 3：AI 自动生成纪要**

OpenClaw 输出：
```
## 📋 2026-03-08 周会纪要

### 参会人员
张三、李四、王五、赵六

### 主要议题
1. 项目 A 进度汇报
2. 项目 B 风险评估
3. 下周工作安排

### 关键决策
- 项目 A 延期至 3 月 15 日发布
- 项目 B 增加 1 名开发人员

### 行动项
| 任务 | 负责人 | 截止日期 |
|------|--------|----------|
| 修复登录 bug | 王五 | 3/10 |
| 更新项目文档 | 李四 | 3/12 |
| 客户需求确认 | 张三 | 3/11 |

### 待讨论事项
- 下周是否需要加班？
- 新人入职培训安排
```

**Step 4：自动发送到飞书群**

```bash
# OpenClaw 自动推送到飞书群
openclaw feishu send \
  --chat_id "oc_xxx" \
  --message "本周周会纪要已生成，请查收"
```

## 进阶技巧：让纪要更智能

### 1. 自动识别行动项

**Prompt**：
```
请从会议记录中提取所有行动项，格式：
- 任务描述
- 负责人
- 截止日期（如果没有提到，标注"待定"）
- 优先级（高/中/低）
```

### 2. 生成会议待办清单

**Prompt**：
```
根据会议纪要，为每个参会人生成个人待办清单：
- 张三的待办
- 李四的待办
- ...
```

### 3. 自动创建日历提醒

**Prompt**：
```
根据行动项的截止日期，生成 iCal 格式的日历事件
```

### 4. 多语言会议支持

**Prompt**：
```
会议记录包含中英文混合内容，请：
1. 生成中文纪要
2. 生成英文摘要
3. 翻译关键技术术语
```

## 常见问题

### Q: 飞书会议如何导出字幕？
A: 会议结束后，在飞书会议详情页 → 录制与字幕 → 导出文字记录

### Q: 腾讯会议支持吗？
A: 支持！腾讯会议也支持导出字幕，操作类似

### Q: 隐私安全吗？
A: 完全本地处理，数据不会上传到第三方（除了你选择的 AI 模型）

### Q: 可以自动识别发言人吗？
A: 取决于会议平台的字幕功能。飞书、Zoom 支持识别，腾讯会议需要手动标注
