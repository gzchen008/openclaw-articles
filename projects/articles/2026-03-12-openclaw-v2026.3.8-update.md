# OpenClaw 2026.3.8 发布：备份功能、LLM 搜索增强、ACP 来源追踪

> 2026年3月9日，OpenClaw 发布 v2026.3.8 版本，带来多项重要功能更新和修复。

---

## 🎯 核心亮点

### 📦 CLI 备份功能

新增 `openclaw backup create` 和 `openclaw backup verify` 命令：

```bash
# 创建本地状态备份
openclaw backup create

# 仅备份配置（不含工作区）
openclaw backup create --only-config

# 验证备份完整性
openclaw backup verify
```

- 支持 `--only-config` 仅备份配置
- 支持 `--no-include-workspace` 排除工作区
- 提供清单和负载验证
- 在破坏性操作流程中提供备份引导

### 🔍 Brave 搜索 LLM Context

新增 `web_search` 的 LLM Context 端点支持：

```json
// openclaw.json
{
  "tools": {
    "web": {
      "search": {
        "brave": {
          "mode": "llm-context"
        }
      }
    }
  }
}
```

- 返回提取的 grounding 片段和来源元数据
- 提供更精准的搜索结果上下文

### 🎙️ Talk 模式静默超时配置

新增 `talk.silenceTimeoutMs` 配置项，可自定义语音识别的静默等待时间：

```json
{
  "talk": {
    "silenceTimeoutMs": 2000  // 2秒静默后自动发送
  }
}
```

### 🔗 ACP 来源追踪

新增 ACP 入口来源元数据和可见收据注入：

```bash
openclaw acp --provenance off|meta|meta+receipt
```

- 保留并报告 ACP 来源上下文
- 包含会话追踪 ID

---

## 🖥️ macOS 增强

- **远程网关 Token 字段**：在 macOS 应用中新增远程模式 token 输入字段
- **Tailscale 网关发现**：保持 Tailscale Serve 探测活跃，优先使用直接传输
- **LaunchAgent 重启**：重新启用被禁用的 LaunchAgent 服务，支持从禁用状态恢复

---

## 🛠️ TUI 改进

- 自动从当前工作区推断活动 agent
- 新增亮色主题支持，通过 `COLORFGBG` 自动检测终端背景
- 支持通过 `OPENCLAW_THEME=light|dark` 手动切换

---

## 🌐 浏览器与网络

### Chrome 扩展中继

- 新增 `browser.relayBindHost` 配置，支持绑定到非回环地址
- 改进 WSL2 和跨命名空间的设置兼容性

### CDP 改进

- 规范化回环直接 WebSocket CDP URL
- 修复 Browserless 风格容器端点的通配符 URL 处理

---

## 📱 Android 精简

Android 应用移除以下权限和功能：
- 自更新功能
- 后台位置
- 屏幕录制
- 后台麦克风捕获

前台服务仅保留 `dataSync` 类型，更加轻量和隐私友好。

---

## 💬 消息平台修复

### Telegram
- **DM 路由**：按 agent 去重入站 DM，避免重复回复
- **轮播重启清理**：关闭时中止进行中的 API 获取
- **媒体下载**：仅对停滞的 body 读取超时，不中断慢速下载

### Matrix
- 更安全的 `m.direct` 回退检测
- 尊重显式房间绑定而非 DM 分类

### Feishu
- **插件引导**：在安装频道插件后清除短期插件发现缓存，避免重复提示下载

---

## 🐳 容器与部署

### Podman 改进

- 修复 `run_as_user` 中的权限拒绝问题
- **SELinux 自动检测**：自动检测 SELinux 强制/宽容模式，为绑定挂载添加 `:Z` 重新标记

### Docker 镜像优化

- 精简开发依赖
- 剥离仅构建的 dist 元数据，减小镜像体积

---

## 🔒 安全加固

- **浏览器 SSRF**：在严格浏览器导航流中阻止私有网络中间重定向跳转
- **MS Teams 授权**：即使配置了团队/频道路由允许列表，`groupPolicy: "allowlist"` 仍强制执行发送者允许列表
- **system.run**：将批准的 `bun` 和 `deno run` 脚本操作数绑定到磁盘文件快照，防止批准后脚本重写

---

## 📊 GPT-5.4 支持

- 使用 GPT-5.4 的 1,050,000 token 上下文窗口
- 支持 128,000 最大 token 数
- 规范化 `gpt-5.4` 回退传输
