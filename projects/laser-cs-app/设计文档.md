## 一、项目概述

### 1.1 项目背景

真人CS激光对抗是一款结合实体装备与移动应用的沉浸式战术竞技游戏。玩家使用配备激光发射器的仿真武器、激光接收头灯、智能地雷等设备，通过APP进行游戏组织、实时对战和成绩统计。

### 1.2 目标用户

- 真人CS俱乐部/场馆运营商
- 团建活动组织者
- 军事爱好者/战术竞技玩家

### 1.3 核心价值

- **沉浸式体验**：实体装备 + APP 实时反馈
- **战术深度**：兵种配合、区域占领、道具使用
- **数据驱动**：完整战绩记录、数据分析

### 3.3 设备连接

#### 3.3.1 蓝牙设备管理

```kotlin
data class BluetoothDevice(
    val deviceId: String,      // 设备唯一ID
    val deviceType: DeviceType, // GUN, HEADLIGHT, CAMERA

```

#### 3.3.2 连接流程

```
1. APP 扫描蓝牙设备
2. 用户选择设备 → 配对
3. 验证设备类型和固件版本
4. 连接成功 → 设备状态更新
5. 游戏中实时同步设备数据
```

#### 3.3.3 运动相机系统

**支持两种运动相机连接方式**：

| 连接方式 | 通信协议 | 适用场景 |
|----------|----------|----------|
| **蓝牙连接** | Bluetooth LE | 第三方运动相机（GoPro、Insta360 等） |
| **WiFi 连接** | TCP/UDP | 自研运动相机 |

##### 预录制功能

**工作流程**：

```
1. 游戏开始 → APP 发送"进入预录制模式"指令到运动相机
2. 运动相机开始预录制（循环缓存最近 N 秒视频）
3. APP 检测到击杀事件：
   ├── 服务器推送击杀确认
   └── APP 发送"快门指令"到运动相机
4. 运动相机收到快门指令 → 保存预录制片段 + 继续录制 M 秒
5. 录制完成 → 标记为"精彩镜头"
```

##### 第三方运动相机（蓝牙）

| 参数 | 说明 |
|------|------|
| 预录制时长 | 5-30 秒（可配置） |
| 触发延迟 | < 100ms |
| 保存时长 | 预录制 + 触发后 5-10 秒 |

**蓝牙指令示例**：
```json
{
  "type": "CAMERA_COMMAND",
  "device_id": "camera_001",
  "action": "PRE_RECORD_START",  // PRE_RECORD_START, PRE_RECORD_STOP, TRIGGER_SAVE
  "duration": 15,  // 预录制时长（秒）
  "post_trigger": 5  // 触发后继续录制时长（秒）
}
```

##### 自研运动相机（WiFi）

**技术参数**：

| 参数 | 说明 |
|------|------|
| 通信协议 | TCP Socket / UDP 广播 |
| 端口 | 8080（可配置） |
| 延迟 | < 50ms |
| 视频格式 | H.264 / H.265 |
| 分辨率 | 1080p / 4K 可选 |

**WiFi 指令协议**：

```kotlin
enum class CameraAction {
    PRE_RECORD_START,    // 开始预录制
    PRE_RECORD_STOP,     // 停止预录制
    TRIGGER_HIGHLIGHT,   // 触发精彩镜头保存
    SET_RESOLUTION,      // 设置分辨率
    GET_STATUS           // 获取相机状态
}

data class CameraCommand(
    val action: CameraAction,
    val params: Map<String, Any> = emptyMap()
)

// 示例：触发精彩镜头
val command = CameraCommand(
    action = CameraAction.TRIGGER_HIGHLIGHT,
    params = mapOf(
        "pre_duration" to 15,    // 预录制时长
        "post_duration" to 5,    // 触发后录制时长
        "event_type" to "KILL",  // 事件类型
        "player_id" to "user_123"
    )
)
```

**连接流程**：

```
1. APP 扫描局域网内的自研运动相机（UDP 广播）
2. 用户选择相机 → 建立 TCP 连接
3. 发送握手指令 → 验证设备身份
4. 连接成功 → 定期心跳保活
5. 游戏中：
   ├── 游戏开始 → 发送 PRE_RECORD_START
   ├── 检测击杀 → 发送 TRIGGER_HIGHLIGHT
   └── 游戏结束 → 发送 PRE_RECORD_STOP
```

##### 精彩镜头管理

| 字段 | 类型 | 说明 |
|------|------|------|
| clip_id | String | 片段唯一ID |
| user_id | String | 玩家ID |

| duration | Int | 片段时长（秒） |
| camera_type | Enum | BLUETOOTH / WIFI_CUSTOM |
| video_url | String | 视频文件URL（上传后） |

**触发条件**：

### 3.4 游戏核心

#### 3.4.1 兵种系统

| 兵种 | 血量 | 特殊能力 | 次数限制 |
|------|------|----------|----------|
| **突击手** | 120 | 无 | - |
| **医疗兵** | 100 | 复活队友 | 3次/局 |
| **弹药兵** | 90 | 弹药补给 | 5次/局 |

#### 3.4.2 武器系统

| 武器类型 | 威力 | 弹匣容量 | 射速 | CD时间 |
|----------|------|----------|------|--------|
| 突击步枪 | 15 | 30 | 中 | 0.1s |
| 冲锋枪 | 10 | 40 | 快 | 0.05s |
| 狙击枪 | 50 | 5 | 慢 | 2.0s |
| 霰弹枪 | 30 | 8 | 慢 | 1.5s |

#### 3.4.3 道具系统

| 道具 | 连接方式 | 激活方式 | 效果 |
|------|----------|----------|------|
| **防守地雷** | 4G | APP激活 | 区域内敌人全部阵亡 |
| **阔剑地雷** | 4G | 手动引爆 | 扇形区域内敌人阵亡 |

#### 3.4.4 击杀判定流程

```
1. 攻击者枪械发射激光信号（包含：设备ID、武器类型、时间戳）
2. 被攻击者头灯接收激光信号
3. 头灯通过蓝牙发送被击中事件到APP
4. APP 计算伤害：
   - 基础伤害 = 武器威力
   - 距离衰减（可选）
   - 护甲减免（可选）
5. 更新被攻击者血量
6. 血量 ≤ 0 → 阵亡
7. 上报服务器 → 更新击杀/死亡统计
```

### 3.5 地图系统

#### 3.5.1 区域类型

| 区域类型 | 功能 | 阵营限制 |
|----------|------|----------|
| **战场** | 主要交战区域 | 通用 |
| **复活区** | 阵亡后重生点 | 分阵营 |
| **基地** | 初始出生点 | 分阵营 |
| **补给区** | 弹药/血量补给 | 分阵营 |
| **占领区** | 占领得分区域 | 通用 |
| **毒气区** | 每5秒扣3点血 | 通用 |
| **辐射区** | 每5秒扣1点血 | 通用 |

#### 3.5.2 区域绘制

- 房主在地图上选择3个以上坐标点
- 系统自动绘制多边形区域
- 支持设置区域属性（CD时间、扣血频率等）

#### 3.5.3 占领区规则

```
1. 玩家进入占领区 → 开始计时
2. 区域内只有单阵营玩家 → 占领计时（120秒）
3. 区域内存在双方玩家 → 争夺状态（不计时）
4. 占领成功 → 该阵营获得积分加成
5. 占领期间被敌方全部击杀 → 恢复初始状态
```

### 3.6 游戏模式

#### 3.6.1 计时模式

- **规则**：规定时间内击杀数最高的队伍获胜
- **参数**：时长（最高10小时）
- **胜利条件**：时间结束 → 击杀数对比

#### 3.6.2 积分模式

- **规则**：达到目标积分的队伍获胜
- **参数**：目标积分（最高99999分）
- **积分来源**：
  - 击杀敌人：+100分
  - 占领区域：+50分/分钟
  - 完成任务：+200分

#### 3.6.3 自主结算模式

- **规则**：房主自主开局和结束
- **参数**：无
- **胜利条件**：房主手动结束 → 积分对比

### 3.7 实时态势

#### 3.7.1 地图展示

- 双方玩家实时位置（GPS）
- 玩家所持武器种类/型号
- 区域状态（占领方、争夺中）
- 道具位置

#### 3.7.2 数据更新频率

| 数据类型 | 更新频率 |
|----------|----------|
| 玩家位置 | 1秒 |
| 血量状态 | 实时（事件触发） |
| 区域状态 | 5秒 |
| 击杀事件 | 实时 |

### 3.8 历史战绩

#### 3.8.1 数据保存

- 服务器保存：30局（或1个月）
- 本地保存：所有历史战绩

#### 3.8.2 战绩内容

| 字段 | 说明 |
|------|------|
| 游戏时间 | 开始/结束时间 |
| 游戏模式 | 计时/积分/自主 |
| 地图 | 战场名称 |
| 阵营 | A/B |
| 兵种 | 突击/医疗/弹药 |
| 击杀数 | 本局击杀 |
| 死亡数 | 本局死亡 |

| 占领时长 | 占领区累计时间 |
| MVP | 是否为本局MVP |
