真人CS激光对抗APP系统设计文档

一、项目概述

1.1 项目背景

真人CS激光对抗是一款结合实体装备与移动应用的沉浸式战术竞技游戏。玩家使用配备激光发射器的仿真武器、激光接收头灯、智能地雷等设备，通过APP进行游戏组织、实时对战和成绩统计。

1.2 目标用户

  • 真人CS俱乐部/场馆运营商
  • 团建活动组织者
  • 军事爱好者/战术竞技玩家

1.3 核心价值

  • 沉浸式体验：实体装备 + APP 实时反馈
  • 战术深度：兵种配合、区域占领、道具使用
  • 数据驱动：完整战绩记录、数据分析

3.3.3 运动相机系统

支持两种运动相机连接方式：

连接方式通信协议适用场景
蓝牙连接Bluetooth LE第三方运动相机（GoPro、Insta360 等）
WiFi 连接TCP/UDP自研运动相机
预录制功能

工作流程：

  1. 游戏开始 → APP 发送「进入预录制模式」指令到运动相机
  2. 运动相机开始循环缓存最近 N 秒的视频
  3. APP 检测到玩家击杀事件
  4. APP 发送「快门指令」到运动相机
  5. 运动相机保存预录制片段 + 继续录制 M 秒
  6. 完成精彩镜头录制

技术参数：

参数说明
预录制时长5-30 秒（可配置）
触发延迟< 100ms
保存时长预录制 + 触发后 5-10 秒

蓝牙指令示例：

{
  "type": "CAMERA_COMMAND",
  "device_id": "camera_001",
  "action": "PRE_RECORD_START",
  "duration": 15,
  "post_trigger": 5
}
自研运动相机（WiFi）

技术参数：

参数说明
通信协议TCP Socket / UDP 广播
端口8080（可配置）
延迟< 50ms
视频格式H.264 / H.265
分辨率1080p / 4K 可选

WiFi 指令协议：

enum class CameraAction {
    PRE_RECORD_START,    // 开始预录制
    PRE_RECORD_STOP,     // 停止预录制
    TRIGGER_HIGHLIGHT,   // 触发精彩镜头保存
    SET_RESOLUTION,      // 设置分辨率
    GET_STATUS           // 获取相机状态
}

连接流程：

  1. APP 扫描局域网内的自研运动相机（UDP 广播）
  2. 用户选择相机 → 建立 TCP 连接
  3. 连接成功 → 定期心跳保活
  4. 游戏中：
    • 游戏开始 → 发送 PRE_RECORD_START
    • 检测击杀 → 发送 TRIGGER_HIGHLIGHT
    • 游戏结束 → 发送 PRE_RECORD_STOP

3.4.4 击杀判定流程

  1. 攻击者枪械发射激光信号（包含：设备ID、武器类型、时间戳）
  2. 被攻击者头灯接收激光信号
  3. 头灯通过蓝牙发送被击中事件到APP
  4. APP 计算伤害：
    • 基础伤害 = 武器威力
    • 距离衰减（可选）
    • 护甲减免（可选）
  5. 更新被攻击者血量
  6. 血量 ≤ 0 → 阵亡
  7. 上报服务器 → 更新击杀/死亡统计

3.5 地图系统

3.5.1 区域类型

区域类型功能阵营限制
战场主要交战区域通用
复活区阵亡后重生点分阵营
基地初始出生点分阵营
补给区弹药/血量补给分阵营
占领区占领得分区域通用
毒气区每5秒扣3点血通用
辐射区每5秒扣1点血通用

3.5.2 区域绘制

  • 房主在地图上选择3个以上坐标点
  • 系统自动绘制多边形区域
  • 支持设置区域属性（CD时间、扣血频率等）

3.5.3 占领区规则

  1. 玩家进入占领区 → 开始计时
  2. 区域内只有单阵营玩家 → 占领计时（120秒）
  3. 区域内存在双方玩家 → 争夺状态（不计时）
  4. 占领成功 → 该阵营获得积分加成
  5. 占领期间被敌方全部击杀 → 恢复初始状态

3.6 游戏模式

3.6.1 计时模式

  • 规则：规定时间内击杀数最高的队伍获胜
  • 参数：时长（最高10小时）
  • 胜利条件：时间结束 → 击杀数对比

3.6.2 积分模式

  • 规则：达到目标积分的队伍获胜
  • 参数：目标积分（最高99999分）
  • 积分来源：
    • 击杀敌人：+100分
    • 占领区域：+50分/分钟
    • 完成任务：+200分

3.6.3 自主结算模式

  • 规则：房主自主开局和结束
  • 参数：无
  • 胜利条件：房主手动结束 → 积分对比

6.2 设计原则

  • 操作便捷性：关键操作不超过3次点击

  • 低功耗：优化蓝牙和GPS使用，延长续航
