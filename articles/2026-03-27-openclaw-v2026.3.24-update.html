OpenClaw 2026.3.24 更新：OpenAI 兼容性增强、Teams 全面升级、技能管理更智能

OpenClaw 最新版本 2026.3.24 已发布！这次更新带来了多个重要改进，包括 OpenAI API 兼容性增强、Microsoft Teams 全面升级、技能管理系统优化等。让我们来看看这次更新的亮点。

🔧 OpenAI 兼容性增强

新增 /v1/models 和 /v1/embeddings 端点，并在 /v1/chat/completions 和 /v1/responses 中转发显式模型覆盖参数。这意味着更广泛的客户端和 RAG 系统兼容性！

💬 Microsoft Teams 全面升级

  • 官方 SDK 迁移：迁移至官方 Teams SDK，带来更稳定的表现
  • AI 代理最佳实践：支持流式 1:1 回复、欢迎卡片、反馈/反思、状态更新、输入指示器和原生 AI 标签
  • 消息编辑与删除：支持编辑和删除已发送的消息，包括线程中的回退处理

🎯 技能管理更智能

  • 一键安装：为 bundled skills 添加一键安装配置（coding-agent、gh-issues、openai-whisper-api 等），CLI 和 Control UI 可以在依赖缺失时提供安装选项
  • 状态过滤标签：Control UI 新增状态过滤标签（全部/就绪/需设置/已禁用），并显示计数
  • 详情对话框：点击技能卡片可查看详情，包括要求、开关、安装操作、API 密钥入口、源元数据和主页链接

🤖 Slack 交互式回复优化

恢复直接投递的富文本回复，自动将简单的 Options: 行渲染为按钮/选择器，改进 Slack 交互式设置默认值，并将回复控件与插件交互处理器隔离。

🐳 Docker 容器支持

新增 --container 和 OPENCLAW_CONTAINER 环境变量，可在运行的 Docker 或 Podman OpenClaw 容器内执行 openclaw 命令。

📋 Discord 自动线程命名

新增可选的 autoThreadName: "generated" 配置，让新创建的自动线程可以使用 LLM 生成的简洁标题进行异步重命名。

🔒 安全性修复

  • 关闭 mediaUrl/fileUrl 别名绕过漏洞，确保出站工具和消息操作无法逃逸媒体根目录限制
  • 对齐出站媒体访问与配置的 fs 策略，严格的工作空间代理保持沙盒化

🐛 其他修复

  • Gateway 重启：重启后通过心跳唤醒中断的代理会话，并重试出站投递
  • Docker 设置：避免启动前的共享网络命名空间循环，修复新安装失败问题
  • WhatsApp 群组：跟踪最近发送的消息 ID，正确抑制群组回声
  • Telegram 论坛主题：恢复 #General 主题 1 的路由
  • Discord 超时：在入站工作线程超时时发送可见的超时回复

📦 系统要求更新

将 Node 22 的最低支持版本降低到 22.14+（继续推荐 Node 24），同时 openclaw update 会在更新前检查目标包的 Node 版本要求，避免在不支持的运行时上安装。

📥 立即更新

运行以下命令更新到最新版本：

openclaw update
