## 什么是 Skills？

Skills 就是 AI 助手的"技能包"。每个 Skill 是一个文件夹，里面有一份说明书（SKILL.md），告诉 AI 助手在什么场景下该怎么做。

想象一下：你给新员工一本操作手册，他照着做就能完成任务。Skills 就是给 AI 的操作手册。

**核心特点：**

- 📄 **纯文本配置**：一个 Markdown 文件搞定，不用写代码
- 🔌 **即插即用**：放进 skills 文件夹就能用
- 🎯 **场景触发**：AI 自动判断什么时候用哪个 Skill
- 🔄 **可分享**：写好一个 Skill，别人也能用

## 一个 Skill 长什么样？

每个 Skill 就是一个文件夹，最简单的结构：

```
my-skill/
├── SKILL.md       # 核心说明书（必须）
└── scripts/       # 辅助脚本（可选）
    └── hello.sh
```

**SKILL.md 是灵魂。** 它告诉 AI 三件事：

1. **我是谁**（名称和描述）
2. **什么时候用我**（触发条件）
3. **怎么用我**（具体指令）

## 5 分钟写一个 Skill

我们来写一个实际的 Skill：**每日提醒助手**。

### 第一步：创建文件夹

```
~/.openclaw/workspace/skills/daily-reminder/
```

### 第二步：写 SKILL.md

```markdown
---
name: daily-reminder
description: 每日提醒技能。当用户说"提醒我"、"别忘了"、"明天要"等关键词时触发。
---

# 每日提醒助手

## 触发条件

当用户需要设置提醒或记事时使用此 Skill。

## 工作流程

1. 询问用户：提醒内容是什么？
2. 询问用户：什么时间提醒？
3. 使用 `openclaw cron add` 创建定时任务
4. 确认提醒已设置成功

## 输出格式

确认消息包含：
- ✅ 提醒内容
- 📅 提醒时间
- 🔔 提醒方式

## 注意事项

- 时间格式使用 ISO 8601
- 提醒内容不超过 200 字
- 如果时间已过，提示用户选择新的时间
```

**就这样！** 没有一行代码，一个 Skill 就完成了。

### 第三步：测试

重启 OpenClaw 或等待自动加载，然后对你的 AI 助手说：

> "提醒我明天下午3点开会"

AI 助手会自动匹配到这个 Skill，按照你写的流程执行。

## SKILL.md 写作技巧

写好 SKILL.md 的关键，不是技术，而是**表达清晰**。

### 1. 描述要精准

❌ 模糊的描述：
```
description: 帮助用户做事情
```

✅ 精准的描述：
```
description: 每日提醒技能。当用户说"提醒我"、"别忘了"时触发，使用 cron 创建定时提醒。
```

AI 根据描述判断要不要用这个 Skill，描述越精准，触发越准确。

### 2. 流程要具体

❌ 笼统的指令：
```
帮用户设置提醒
```

✅ 具体的步骤：
```
1. 解析用户输入的时间和内容
2. 转换为 cron 表达式
3. 执行 openclaw cron add --name "reminder" --schedule "0 15 * * *" --message "开会时间到了"
4. 返回确认信息
```

### 3. 给 AI 边界

告诉 AI 什么不该做，和告诉它该做什么一样重要：

```markdown
## 边界
- 不要删除其他用户的提醒
- 提醒时间不能超过一年
- 不要自动发送邮件通知（除非用户明确要求）
```

## 进阶：让 Skill 调用脚本

如果你的 Skill 需要做更复杂的事，可以写辅助脚本。

比如一个**股票查询 Skill**：

```
stock-checker/
├── SKILL.md
└── scripts/
    └── fetch_price.sh
```

SKILL.md 里这样引用：

```markdown
## 使用方法

执行以下命令获取股票价格：

\`\`\`bash
bash scripts/fetch_price.sh <股票代码>
\`\`\`

将返回结果格式化后告知用户。
```

AI 会读取 SKILL.md，在需要的时候调用脚本，再把结果整理好告诉你。

## 常见问题

**Q：Skills 和普通对话有什么区别？**

Skills 是预定义的"专家模式"。普通对话是通用能力，Skills 是针对特定场景的增强。AI 会根据你的描述自动选择用不用 Skill。

**Q：一个 Skill 可以调用另一个 Skill 吗？**

可以。SKILL.md 里可以引用其他 Skill 的功能，形成组合效果。

**Q：写错了怎么办？**

直接编辑 SKILL.md 就行。AI 下次加载时会读取最新版本。没有编译、没有重启，改了就生效。

**Q：Skill 会不会冲突？**

OpenClaw 会根据描述匹配最合适的 Skill。如果多个 Skill 匹配，选择描述最具体的那一个。
