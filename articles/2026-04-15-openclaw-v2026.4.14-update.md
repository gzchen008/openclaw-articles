# OpenClaw 2026.4.14 重磅更新：主动记忆、Codex 原生支持、安全大升级

> 从 v2026.4.11 升级到 v2026.4.14，跨越三个版本的重磅功能和安全修复

OpenClaw 又一次大规模更新来了！这次从 2026.4.11 到 2026.4.14，横跨三个版本，带来了**主动记忆系统**、**Codex 原生集成**、**LM Studio 本地模型支持**，以及一大波安全加固和稳定性修复。如果你还在用旧版本，这次真的值得升级。

## 🧠 主动记忆（Active Memory）：再也不用手动"记住这个"

这是本次更新最大的亮点。

以前用 OpenClaw，你得手动说"记住这个"或者"搜索记忆"才能调取历史信息。现在，新的 **Active Memory 插件**会在每次对话前自动运行一个记忆子代理，帮你拉取相关的偏好、上下文和历史细节。

**核心能力：**

- 支持三种上下文模式：消息级、最近对话级、全量上下文
- `/verbose` 实时查看记忆召回状态
- 高级提示词/思考模式覆盖，可精细调优
- 可选的转录持久化，方便调试
- 支持混合搜索（语义+词法），召回更精准

配置方式非常简单，在 `openclaw.json` 中启用 Active Memory 插件即可。官方文档：https://docs.openclaw.ai/concepts/active-memory

## 🤖 Codex 原生支持：一条命令接入 GPT-5.4

OpenClaw 现在内置了 **Codex 提供者**，`codex/gpt-*` 模型会自动使用 Codex 管理的认证、原生线程、模型发现和压缩，而 `openai/gpt-*` 继续走标准 OpenAI 路径。这意味着你可以直接在 OpenClaw 里用上最新的 Codex 能力，无需额外配置。

同时，嵌入式 GPT 风格的运行也得到了增强——推理专用轮次和空轮次现在有边界化续行机制，配带重放安全的重试门控。

## 🖥️ LM Studio 提供者：本地模型开箱即用

新增内置 **LM Studio 提供者**，支持：

- 一键接入本地 OpenAI 兼容模型
- 运行时模型发现
- 流式预加载
- 记忆搜索嵌入向量

本地跑模型又多了一个选择，配合之前的 Ollama 支持，OpenClaw 的本地化能力越来越强了。

## 🔒 安全加固：三个关键修复 + 全面防御

这个版本在安全方面投入巨大，值得特别关注：

**1. Shell 注入防御升级**

移除了 busybox/toybox 从安全解释器白名单，扩大了 shell 包装器检测范围，阻止 env-argv 注入攻击。这些修复由安全研究员贡献，AI 辅助发现。

**2. 审批授权修复**

空审批者列表不再被当作"已授权"，堵住了一个潜在的权限绕过漏洞。

**3. 浏览器安全全面加固**

浏览器沙箱导航防御跨 SSRF 默认值、主机名白名单、交互驱动的重定向、子框架、CDP 发现等全面收紧。浏览器控制的 SSRF 策略也得到了加强，本地 Chrome 管理和 CDP 控制现在有更严格的策略。

## 🛡️ 插件安全：加载范围精确化

新版本对插件加载做了重大改进——CLI、提供者和通道激活现在只加载清单声明的必要运行时，保持明确的作用域和信任边界。这不仅减少了启动时间，更重要的是缩小了攻击面。

## 🎙️ macOS Talk Mode：本地 MLX 语音

macOS 用户迎来实验性的 **本地 MLX 语音提供者**，支持：

- 明确的提供者选择
- 本地语音播放
- 打断处理
- 系统语音回退

这意味着 Talk Mode 现在可以在完全本地运行语音识别和合成，不需要外部 API。

## 🔧 其他重要改进

**执行策略命令**

新增 `openclaw exec-policy` 命令，支持 `show`、`preset`、`set` 子命令，用于同步执行审批配置。这对于管理多环境下的命令执行策略非常有用。

**网关命令发现**

新增 `commands.list` RPC，远程网关客户端可以发现运行时原生命令、文本命令、技能命令和插件命令，支持面向表面的命名和参数元数据。

**私有网络支持**

新增 `models.providers.*.request.allowPrivateNetwork` 配置，用于受信任的自托管 OpenAI 兼容端点。这对内网部署非常实用。

**Dreaming 改进**

- 梦境定时事件精确单次消费
- 短期信号完整置信度计算
- 日记时区使用主机本地时间
- 多轮梦境再访足以触发持久记忆晋升门槛

**QA 测试增强**

- 新增 Multipass 虚拟机运行器
- 新增 Telegram 实时 QA 通道
- 新增 Convex 支持的凭据租赁池

## 📋 全渠道稳定性修复

这个版本几乎给所有消息渠道都做了稳定性提升：

- **WhatsApp**：连接所有权集中化，重连和登录恢复更稳定；媒体发送不再静默丢弃附件
- **Telegram**：审批按钮回调独立序列化，不再阻塞代理轮次；话题消息正确路由
- **Discord**：重连前清除过期心跳定时器，防止僵尸回调
- **Matrix**：提及门控修复，支持非 OpenClaw 客户端
- **Slack**：独立的 doctor 契约侧车，配置读取不再崩溃
- **飞书**：allowlist 规范化，不再跨用户/聊天命名空间匹配
- **iMessage**：重试瞬态启动失败，不再立即弹回通道
- **Microsoft Teams**：全面修复媒体下载、SSO 回调、线程回复等

此外，消息投递去重标准化覆盖了 Telegram、Discord、Slack、WhatsApp、Matrix、LINE、飞书等几乎所有渠道。
