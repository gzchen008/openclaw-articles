# OpenClaw 技能系统详解：让你的 AI 助手无所不能

OpenClaw 的强大不仅在于核心功能，更在于其灵活的技能系统。通过技能（Skills），你可以让 AI 助手掌握各种专业能力——从代码审查到图片生成，从天气查询到智能家居控制。

今天我们来深入理解这个系统，并动手创建自己的技能。

## 🎯 什么是技能（Skill）？

技能是一组预定义的指令和能力扩展，让 AI 在特定场景下表现得更专业。一个技能通常包含：

- **SKILL.md** - 技能说明和使用指南
- **脚本文件** - 实现具体功能的代码
- **配置文件** - 参数和环境设置

当你的请求匹配某个技能时，OpenClaw 会自动加载该技能的指令，让 AI 知道如何处理这类任务。

## 📂 技能存放位置

OpenClaw 支持多个技能目录，按优先级加载：

1. `~/.openclaw/skills/` - 用户自定义技能（优先级最高）
2. `~/.npm-global/lib/node_modules/openclaw/skills/` - 内置技能

内置技能包括：coding-agent、github、weather、discord、imsg 等。你可以直接使用这些技能，也可以创建自己的技能覆盖它们。

## 🔧 实战：创建一个自定义技能

让我们创建一个简单的「每日日报」技能，用于生成工作日报。

### 第一步：创建技能目录

```bash
mkdir -p ~/.openclaw/skills/daily-report
```

### 第二步：编写 SKILL.md

创建文件 `~/.openclaw/skills/daily-report/SKILL.md`：

```markdown
# Daily Report Skill

用于生成每日工作日报的技能。

## 使用场景
当用户说「生成日报」、「今天做了什么」、「写日报」时触发

## 技能说明
1. 读取今天的 Git 提交记录
2. 汇总日历中的会议
3. 生成格式化的日报内容

## 输出格式
## 今日工作
- [工作项1]
- [工作项2]

## 明日计划
- [计划1]
- [计划2]
```

### 第三步：添加辅助脚本（可选）

如果需要复杂逻辑，可以添加脚本文件 `scripts/generate.sh`：

```bash
#!/bin/bash
# 获取今天的 Git 提交
git log --since="midnight" --oneline --author="$(git config user.name)"
```

### 第四步：使用技能

技能创建后，下次对话时直接说：

> 「帮我生成今天的日报」

OpenClaw 会自动加载这个技能，按照 SKILL.md 中的指令执行。

## 🌟 内置技能推荐


## 💡 技能编写最佳实践

### 清晰的触发条件

在 SKILL.md 开头说明什么情况下应该使用这个技能：

```markdown
## 触发条件
- 用户明确要求「生成日报」
- 用户问「今天做了什么」
- 用户说「写工作总结」
```

### 具体的执行步骤

给出清晰的步骤说明，AI 会按照这些步骤执行：

```markdown
## 执行步骤
1. 读取 ~/.openclaw/workspace/memory/ 下的今日笔记
2. 获取今天的 Git 提交记录
3. 汇总成日报格式
4. 输出到用户
```

### 安全边界

明确告诉 AI 什么不该做：

```markdown
## 安全限制
- 不要访问其他用户的数据
- 不要发送邮件给外部人员
```

## 🚀 高级技巧

### 技能继承

你可以在 SKILL.md 中引用其他文件：

```markdown
## 参考资料
参见 USER.md 中的个人信息
参见 TOOLS.md 中的工具配置
```

### 条件加载

使用前置条件判断是否加载技能：

```markdown
## 前置条件
- 需要存在 ~/.gitconfig
- 需要当前目录是 Git 仓库
```

### 技能组合

多个技能可以协同工作：

```
先用 weather 查天气，再用 imsg 发送给家人
```
