OpenClaw 2026.4.15：Claude Opus 4.7 默认登场 + Gemini 语音合成

你还在手动切换最新的 Claude 模型吗？OpenClaw 已经帮你搞定了。

一句话总结

OpenClaw 2026.4.15 正式发布，两大核心变化：Anthropic 全线默认升级到 Claude Opus 4.7，以及全新 Gemini TTS 语音合成支持。同时修复了超过 20 个稳定性与安全问题。

🚀 两大新功能

1. Claude Opus 4.7 成为默认模型

这次更新最重磅的变化——OpenClaw 中所有 Anthropic 相关的默认模型选择、opus 别名、Claude CLI 默认配置以及内置图片理解能力，全部升级到 Claude Opus 4.7。

这意味着什么？

• 不再需要手动配置最新模型名称

• opus 别名自动指向最新最强版本

• 图片理解开箱即用，无需额外配置

• Claude CLI 用户自动获得最新体验

如果你之前在配置文件里写了具体的模型版本号，现在可以简化为直接使用 opus 别名，以后更新模型时无需手动修改配置。

2. Gemini TTS 语音合成

Google Gemini 语音合成能力正式集成到 OpenClaw 的 google 插件中。这不是简单的文字转语音，而是一套完整的语音解决方案：

• 支持多种语音选择，找到最适合你场景的声音

• WAV 格式回复输出，适配各种播放场景

• PCM 电话语音输出，专门为语音通话场景优化

• 完整的配置文档和设置引导

对于使用 OpenClaw Talk Mode 的用户来说，这是又一个高质量本地/云端语音选择。

🔧 重要修复亮点

除了两个新功能，这次更新还包含大量修复，以下是最值得关注的：

稳定性修复

• Skills 列表排序稳定化：技能列表现在按名称排序，不再因为配置顺序不同导致 prompt 缓存频繁失效，响应更快

• 上下文和记忆预算优化：默认 prompt token 预算被缩减，长对话 session 拉取的上下文更精简

• CLI 升级不再卡死：npm 全局升级后，旧的打包文件会被自动清理

安全修复

• 本地 MEDIA 信任锚定：修复了自定义工具名称与内置工具冲突时可能继承本地媒体信任权限的问题

• 客户端工具名称冲突检测：冲突的工具名称会返回明确错误，避免混淆

Codex 用户必看

• Codex 模型传输层修复：旧版配置会自动修复为正确的传输路径，不再出现 HTML/Cloudflare 错误

BlueBubbles 用户必看

• 图片附件下载恢复：Node 22+ 环境下的入站图片附件下载已修复

🛠 升级指南

升级非常简单，一行命令搞定：

openclaw update

或者通过 npm：

npm install -g openclaw@latest

升级后建议运行一次健康检查：

openclaw doctor

升级后检查清单

• 如果使用了 Anthropic 模型，确认 opus 别名正常工作

• 如果使用了自定义工具，检查是否有名称冲突警告

• 运行 openclaw doctor 确认配置健康

💡 写在最后

2026.4.15 是一个"小版本大变化"的典型 OpenClaw 更新——功能列表不长，但每个变化都直接影响日常使用体验。Claude Opus 4.7 的默认集成让所有人自动获得最新模型能力，Gemini TTS 则为语音交互场景提供了更多选择。

如果你还在用旧版本，建议尽快升级。升级命令只有一行，但带来的体验提升是实实在在的。
