OpenClaw 2026.4.23 更新：图像生成全面升级、子代理上下文继承、安全加固

OpenClaw 2026.4.23 版本正式发布！本次更新带来了多项重量级功能：OpenAI 和 OpenRouter 的图像生成与编辑能力全面接入、子代理可选的上下文继承机制、生成工具的超时控制、本地嵌入上下文可配置，以及大量安全修复和稳定性改进。

这是一个质量优先的版本，超过 120 项变更和修复，覆盖图像生成、代理架构、安全防护、平台适配等多个维度。

🎨 图像生成能力全面升级

本次更新对图像生成做了系统性增强：

• OpenAI 图像生成免 Key 可用：通过 Codex OAuth 认证，openai/gpt-image-2 无需额外配置 OPENAI_API_KEY 即可使用

• OpenRouter 图像生成支持：持有 OPENROUTER_API_KEY 即可通过 OpenRouter 使用各类图像模型

• 参考图编辑修复：OpenAI 多参考图编辑通过 multipart 上传恢复正常

• Azure OpenAI 图像支持：自动识别 Azure 风格的图像端点，支持 api-key 认证和部署 URL

• Google Gemini 图像生成 SSRF 修复：私有网络代理配置下的 image_generate 现在可以正常工作

• 图片质量与格式控制：代理可以请求 provider 支持的质量和输出格式提示，支持 OpenAI 的背景、压缩等高级参数

🤖 子代理上下文继承

新增可选的"分叉上下文"机制，允许 sessions_spawn 的子代理继承父会话的对话记录，同时保持默认的隔离模式不变。

这意味着你可以按需让子代理"了解上下文"再执行任务，而不是每次都从零开始。适用于多步骤编排、任务委托等复杂场景。

⏱️ 生成工具超时控制

图像、视频、音乐和 TTS 生成工具现在支持按调用设置 timeoutMs，对于需要长时间生成的任务，你可以单独延长超时，而不是受全局限制的约束。

配合网关层面的 undici 流超时修复（不再被短 header 超时覆盖），慢速生成任务（如 Gemini 图像）现在可以稳定完成。

🧠 本地嵌入上下文可调

新增 memorySearch.local.contextSize 配置项（默认 4096），让本地嵌入上下文大小可以根据硬件条件灵活调整，在资源受限的设备上也能流畅使用向量检索。

🔒 安全修复亮点

安全是本次更新的核心主题之一，大量来自社区的安全加固贡献：

• 网关配置编辑白名单化：agent 驱动的配置修改改为显式白名单机制，只允许修改提示词、模型等安全路径

• MCP 工具权限隔离：修复 ACPX OpenClaw 工具桥接暴露 cron 等仅限 owner 工具的权限提升漏洞

• Teams 跨 Bot Token 防护：要求 Bot Framework token 必须包含已配置的 app ID，阻止跨 Bot 重放

• Android 安全加固：要求回环地址才能使用明文网关连接；外部 App 意图不再自动发送提示词

• Secret 热重载：Webhook 路由密钥支持即时刷新，无需重启网关

• 权限审批显式启用：不再自动启用审批客户端，需要显式配置

• Discord 斜杠命令策略：原生斜杠命令不再绕过频道策略限制

• WhatsApp 安全：联系人/vCard 信息不再内联到消息体，防止隐藏的提示注入

• 群聊安全：频道来源的群名和参与者标签不再直接注入系统提示

💻 平台适配改进

Windows：

• 修复 codex.cmd shim 解析问题，无需手动创建 .exe shim

• 修复插件 SDK 别名更新时的 ENOTEMPTY/EPERM 竞争条件

• 文件备份密钥支持 UTF-8 BOM 剥离

Linux：

• 网关子进程自动提升 oom_score_adj，内存压力下优先终止临时进程

WhatsApp：

• 修复并发重连导致消息重复发送 7-12 倍的问题

• 统一出站媒体归一化逻辑

🔧 其他重要修复

• 记忆系统：MEMORY.md 大小写规范化，openclaw doctor --fix 可自动合并分裂的记忆文件

• 记忆 dreaming：从心跳中解耦为独立的轻量代理任务，不再被心跳禁用影响

• Codex 驱动：固定会话级别的 harness 选择，防止配置变更导致热切换

• 块流式传输：修复部分传输中止后的重复回复问题

• 模型状态：/status 新增 Runner 字段，清晰显示当前运行器类型

• 依赖升级：Pi 升级到 0.70.0，支持上游 gpt-5.5 目录元数据

• Slack 群聊：MPIM 群组 DM 正确归类为群聊上下文，不再泄露工具进度

• WebChat：生成图片持久化为认证媒体，历史重载不再丢失图片

📦 如何升级

一行命令完成升级：

npm install -g openclaw@latest && openclaw gateway restart

升级后建议运行健康检查：

openclaw doctor --fix

Doctor 会自动处理记忆文件规范化、Codex 驗证清理、dreaming cron 迁移等兼容性问题。
