## 🔧 三种方案，总有一款适合你

### 方案一：纯文字模式（最简单）

适合：文字交流为主的会议、即时通讯记录

**操作步骤：**

1. 开会时，在 OpenClaw 所在的群里正常讨论
2. 会后，发送：`帮我整理今天的会议纪要`
3. OpenClaw 自动提取：
   - 讨论的主题
   - 做出的决定
   - 待办事项 + 负责人
   - 下次会议时间（如果有）

**示例对话：**

```
你：帮我整理今天的会议纪要

OpenClaw：
📝 会议纪要 - 2026-02-28

【讨论主题】
• Q1 营销方案最终定稿
• 新产品上线时间确认

【关键决定】
✅ 3月15日正式上线
✅ 预算追加 20%
✅ 小王负责技术对接

【待办事项】
• 小王：3月10日前完成接口联调
• 小李：3月8日前输出宣传素材
• 小张：联系供应商确认库存

【下次会议】
3月5日 14:00，会议室B
```

---

### 方案二：语音转文字 + AI 整理（推荐）

适合：有录音的正式会议、跨时区会议

**操作步骤：**

1. 用手机/电脑录下会议内容
2. 用任意语音转文字工具（如飞书、讯飞、Whisper）生成文本
3. 把文本发给 OpenClaw：`这是今天的会议记录，帮我整理成纪要`
4. 等待 30 秒，收获完美纪要

**进阶技巧：**

如果会议很长（1小时+），可以分段处理：

```
你：这是会议前30分钟的记录，先帮我整理这部分
[粘贴文本]

你：这是后半段，继续整理
[粘贴文本]

你：把两段合并成完整的纪要
```

### 方案三：自动化工作流（高级玩家）

适合：固定频率的例会、对效率有极致追求的人

**思路：**

1. 用 OpenClaw 的 Cron 功能，定时触发整理任务
2. 用浏览器自动化，定期拉取飞书/钉钉的会议记录
3. 整理完自动发送到指定群/邮箱

**简单示例：**

每天下午 6 点，自动提醒你整理当天的会议：

```json
{
  "schedule": "0 18 * * *",
  "action": "send",
  "message": "⏰ 该整理今天的会议纪要啦！把记录发给我，30秒搞定~"
}
```

## ⚠️ 常见问题

**Q: OpenClaw 能识别多人对话吗？**

A: 可以！只要文本中有说话人的标识（如「张三：」「[李四]」），OpenClaw 就能准确区分。

**Q: 敏感信息怎么办？**

A: OpenClaw 支持本地部署，数据不出你的服务器。如果会议涉及机密，建议用本地模型。

**Q: 支持英文会议吗？**

A: 完全支持！OpenClaw 可以处理中英混合内容，还能帮你翻译成另一种语言。

**Q: 纪要格式能自定义吗？**

A: 当然！你可以说「用 Markdown 格式整理」或「用表格形式展示待办事项」。

## 🎁 彩蛋：会议纪要模板

如果你想让纪要更规范，可以用这个模板：

```
# 会议纪要

**日期**：YYYY-MM-DD
**时间**：HH:MM - HH:MM
**地点**：会议室/线上
**参会人**：姓名1、姓名2、姓名3

## 一、议题

1. 议题一
2. 议题二

## 二、讨论内容

### 议题一
• 要点1
• 要点2

### 议题二
• 要点1
• 要点2

## 三、决议事项

| 序号 | 决议内容 | 负责人 | 完成时间 |
|------|----------|--------|----------|
| 1 | xxx | xxx | xxx |

## 四、下次会议

**时间**：YYYY-MM-DD HH:MM
**议题**：xxx
```

把模板发给 OpenClaw，说「以后用这个格式整理」，它就会按这个格式输出。
