> 你有没有经历过这种痛苦？开了一个小时的会，会后花半小时整理纪要，还要反复确认谁说了什么、哪个任务是给谁的……

**录制会议音频**

- 转写音频为文字

- 生成待办事项清单

## 📋 前置条件

在开始之前，确保你已经：

1. 安装好 OpenClaw（不会的话看之前的教程）
2. 有一个 OpenAI API Key（用于语音转文字）
3. 手机或电脑能录制音频
4. 有一个接收纪要的渠道（Discord/Telegram/微信）

## 🔧 方案一：实时语音转文字

如果你开会时能用电脑，这个方案最方便。

### 步骤 1：配置语音识别

在 OpenClaw 中配置语音转文字服务：

```json
{
  "speechToText": {
    "provider": "openai",
    "model": "whisper-1"
  }
}
```

### 步骤 2：开启录音

会议开始时，发送指令：

```
开始记录会议：产品需求评审会
参会人员：张三、李四、王五
```

AI 会自动：
- 记录开始时间
- 保存参会人员名单
- 准备接收语音输入

### 步骤 3：实时转写

会议进行中，你可以：
- 直接对着麦克风说话，AI 实时转写
- 或者在关键节点发送简短笔记

### 步骤 4：生成纪要

会议结束时发送：

```
结束会议，生成纪要
```

AI 会自动整理出：
- 会议主题和时间
- 参会人员
- 关键讨论点（带时间戳）
- 决策结果
- 待办事项（带负责人和截止日期）

## 🎙️ 方案二：录音 + 后处理

如果会议不方便用电脑，用手机录音，会后让 AI 处理。

### 步骤 1：录制音频

用手机自带的录音 App，全程录制会议。

**小技巧：**
- 把手机放在会议桌中央，收音效果最好
- 如果是线上会议，可以用电脑录屏软件同时录音
- 录音格式建议用 m4a 或 mp3（兼容性好）

### 步骤 2：上传音频

会议结束后，把音频文件发给 OpenClaw：

```
@OpenClaw 请帮我整理这个会议录音，生成会议纪要
```

（附带音频文件）

### 步骤 3：AI 自动处理

OpenClaw 会自动：
1. 调用 Whisper API 转写音频
2. 用 LLM 分析内容
3. 提取关键信息
4. 生成结构化纪要

### 步骤 4：确认和发布

AI 会先发一个草稿给你确认：

```
【会议纪要草稿】

📌 会议主题：产品需求评审会
⏰ 时长：45 分钟
👥 参会：张三、李四、王五

📋 关键讨论：
1. 新功能上线时间确认
2. 用户反馈的处理优先级
3. 技术方案的选型

✅ 决策：
- 上线时间定在下周三
- 用户反馈优先处理高优问题
- 技术方案采用 B 方案

📝 待办：
- 张三：完成技术文档（周五前）
- 李四：准备测试用例（下周一前）
- 王五：通知用户上线时间（周二前）

确认无误吗？如果需要修改，请告诉我。
```

你只需要回复"确认"或提出修改意见，AI 就会把最终版发送到指定群组。

## 💡 方案三：集成日历 + 自动提醒

这个方案更进一步，实现 **全自动化**。

### 配置日历集成

首先，让 OpenClaw 能读取你的日历：

```json
{
  "calendar": {
    "provider": "google",
    "credentials": "你的 Google Calendar 凭证"
  }
}
```

### 设置自动提醒

创建一个 Cron 任务：

```json
{
  "schedule": "every 1 hour",
  "action": "check_calendar",
  "params": {
    "lookahead": "30 minutes"
  }
}
```

AI 会每小时检查一次日历，如果有会议要开始，自动发送提醒：

```
📅 即将开会！

会议：产品周会
时间：10:00-11:00
地点：3 号会议室

需要我开始录音吗？
```

### 自动生成并发送纪要

会议结束后，AI 自动：
1. 生成会议纪要
2. 发送到工作群（Discord/Slack）
3. 创建对应的待办事项
4. 同步到项目管理工具（如 Notion）

**全程零操作，真正解放双手。**

## 🔍 常见问题

### Q: 录音文件太大怎么办？

A: Whisper API 支持最大 25MB 文件。如果超过，可以：
- 用 ffmpeg 压缩音频
- 切割成多个文件分段处理
- 降低录音质量（单声道、32kbps 就够用）

### Q: 识别不准确怎么办？

A: Whisper 对中文支持很好，但如果遇到：
- 专业术语识别错误：可以在提示词中加"领域术语表"
- 方言口音重：可以用"提示文本"告诉 AI 大概在讨论什么
- 多人说话分不清：让 AI 在转写时标注说话人

### Q: 隐私安全怎么办？

A: OpenClaw 支持本地部署，你可以：
- 在自己的服务器上运行
- 使用本地 Whisper 模型（无需联网）
- 数据完全不离开你的控制

## 🎁 实战技巧

### 技巧 1：定制纪要模板

不同类型的会议，纪要格式可以不同：

<strong>产品评审会</strong> - 重点记录需求变更、优先级调整
<strong>技术方案会</strong> - 重点记录方案对比、技术决策
<strong>周会/日站会</strong> - 重点记录进度同步、阻塞问题

### 技巧 2：自动创建任务

让 AI 识别"待办事项"，自动创建任务：

```json
{
  "todoIntegration": {
    "provider": "notion",
    "database": "任务数据库ID"
  }
}
```

AI 会把"张三：完成文档（周五前）"自动变成 Notion 任务，并分配给对应的人。

### 技巧 3：历史纪要检索

所有纪要保存到 Notion 或数据库，之后可以随时搜索：

```
@OpenClaw 帮我查一下上个月关于"价格调整"的会议决策
```

AI 会搜索历史纪要，找到相关内容并总结。

## 🚀 进阶玩法

### 玩法 1：会议洞察分析

积累一定数量的纪要后，让 AI 分析：

```
分析过去 10 次产品会议，找出：
1. 最常讨论的话题
2. 决策效率趋势
3. 待办事项完成率
```

AI 会生成可视化报告，帮你发现会议效率问题。

### 玩法 2：自动生成 PPT

会议纪要可以一键转成 PPT：

```
@OpenClaw 把这次会议纪要转成 5 页 PPT
```

AI 会自动：
- 提取关键标题
- 生成要点列表
- 匹配模板样式
- 输出 PPT 文件

### 玩法 3：多语言纪要

跨国会议，可以生成双语纪要：

```
@OpenClaw 用中英双语生成纪要
```

AI 会自动翻译，生成两个版本，方便不同语言的同学阅读。
