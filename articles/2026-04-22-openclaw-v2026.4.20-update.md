# OpenClaw 2026.4.20 更新：安全加固、Kimi K2.6 默认上线、Cron 全面进化

你还在为 AI 助手频繁掉线、模型切换莫名报错、定时任务经常"静默失败"而头疼吗？

OpenClaw 2026.4.20 版本来了一次大规模更新，横跨安全、模型、调度、协议四大板块。如果你还在跑 2026.4.15 或更早版本，这篇文章帮你快速抓住重点。

## 📌 版本速览

从 2026.4.15 升级到 2026.4.20，主要涵盖以下版本：2026.4.18、2026.4.19-beta、2026.4.20。涉及变更 200+ 项，修复 150+ 项。

## 🔒 安全：全方位加固

安全永远是第一优先级。这次更新堵上了多个重要漏洞：

**设备配对权限收窄**：非管理员配对设备只能查看和管理自己的配对记录，无法枚举其他设备或审批他人的配对请求。

**Config 变异防护**：Agent 无法通过 gateway tool 篡改沙箱配置、插件信任、网关认证/TLS、Hook 路由、SSRF 策略等关键路径。模型驱动的 `config.patch` 和 `config.apply` 现在被严格限制。

**WebSocket 广播权限**：配对设备和 Node 角色的会话无法再被动接收聊天内容，需要 `operator.read` 权限才能看到 chat 和 tool-result 事件。

**环境变量隔离**：工作区 `.env` 文件中所有 `OPENCLAW_*` 键被自动屏蔽，防止不受信任的工作区配置影响运行时行为。

**MCP 环境键过滤**：MCP stdio 服务启动时自动屏蔽 `NODE_OPTIONS` 等解释器环境键，同时保留正常的凭证和代理变量。

**Host Key 强制校验**：macOS 远程 SSH 连接从 `accept-new` 改为 `yes`，首次连接不再自动接受未知主机密钥，必须提前在 `~/.ssh/known_hosts` 中信任。

## 🧠 模型：Kimi K2.6 上位、Codex 全面修复

**Moonshot Kimi K2.6 成为默认模型**：绑定的 Moonshot 设置、网页搜索、媒体理解默认使用 `kimi-k2.6`，同时保留 `kimi-k2.5` 兼容。K2.6 支持 `thinking.keep = "all"`，但其他 Moonshot 模型或使用 `tool_choice` 时会自动剥离。

**Claude Opus 4.7 xhigh 推理**：Anthropic 的 Opus 4.7 新增 `xhigh` 推理力度档位，与自适应 thinking 独立控制。

**OpenAI Codex 大修**：
- ChatGPT/Codex OAuth 请求改走 `/backend-api/codex` 端点
- Codex 子进程异常退出不再导致 Gateway 崩溃
- OAuth 导入不再污染 `.codex` 本地文件，OpenClaw 成为凭证的权威来源
- App Server 审批默认改为 `on-request`，避免过度宽松

**GPT 推理模式修复**：`/think off` 不再发送 `reasoning.effort: "none"` 这种无效载荷，各模型的 reasoning effort 映射终于对齐了。

## ⏰ Cron：从"能用"到"好用"

Cron 模块几乎是重写级别的改进：

**状态分离**：运行时状态拆到独立的 `jobs-state.json`，`jobs.json` 只存定义，方便 Git 追踪不冲突。

**投递可靠性**：Telegram 定时投递的去重键改为每次执行独立，不再因为复用 session id 导致后续发送被静默跳过。

**delivery:none 正确处理**：明确配置不投递的 Cron 任务不再产生虚假的"投递失败"记录。

**投递预览**：`cron list/show` 现在能看到投递目标预览，不用再猜任务会发到哪里。

## 🌐 Telegram：长连接治理

Telegram 的连接管理做了彻底清理：

**连接池泄漏修复**：之前每个网络错误都会静默替换 transport，导致长时间运行的 Gateway 累积数百个到 `api.telegram.org` 的连接，耗尽代理配额。现在每个 origin 有严格连接数上限。

**轮询超时放宽**：默认轮询看门狗从 90s 提到 120s，长任务不再被误判为卡死。

**流式回复去重**：自动 compaction 重试时，预览消息不再重复出现。

## 💬 BlueBubbles（iMessage）：更稳更智能

**发送超时扩展**：默认从 10s 提到 30s，macOS 26 的 Private API 发送不再超时丢消息。

**智能降级**：Agent 使用非标准 emoji 反应时（如 👀），自动降级到最接近的 iMessage tapback（如 `love`），不再整体失败。

**优先 iMessage**：同一联系人同时有 iMessage 和 SMS 聊天时，优先走 iMessage，除非明确指定 `sms:`。

## 🛠 其他重要改进

**Session 维护**：内置条目上限和过期清理默认启用，过期的 cron/executor session 不会再 OOM。

**Compaction 通知**：上下文压缩时发送可选的开始/完成通知，让你知道 Agent 在干什么。

**Mattermost 流式回复**：thinking、工具活动和部分回复现在在同一个草稿预览中流式展示。

**插件启动修复**：绑定的 Discord、WhatsApp、Slack、Telegram 等插件的运行时依赖在导入前自动修复，不再需要手动处理。

## ⚡ 升级建议

这是一个安全+稳定性双重加固的大版本，强烈建议所有用户升级。

```bash
npm install -g openclaw@latest
openclaw gateway restart
```

升级后建议检查：
- Cron 任务是否正常投递（`openclaw cron list`）
- Telegram 连接数是否正常
- 设备配对状态是否正常（`openclaw devices`）

完整 changelog 见：https://docs.openclaw.ai
