OpenClaw v2026.3.11 发布：安全加固 + Ollama 一流支持 + iOS 大改版

这个版本包含一个重要的安全修复，以及 Ollama、iOS、macOS 等多项重大更新。

⚠️ 重要安全修复

WebSocket 跨站攻击漏洞（GHSA-5wcw-8jjv-m286）

影响：在 trusted-proxy 模式下，未受信任的来源可能获得 operator.admin 访问权限。

修复：Gateway/WebSocket 现在强制对所有浏览器来源连接进行 origin 验证，无论是否存在代理头。

建议：立即升级到此版本。

🚀 重大新功能

1. Ollama 一流支持

现在 OpenClaw 内置了 Ollama 设置向导：

• Local 模式：本地模型，无需联网
• Cloud + Local 模式：云端 + 本地混合
• 浏览器云登录
• 精选模型建议
• 云模型处理（跳过不必要的本地拉取）

2. iOS Home Canvas 大改版

• 内置欢迎屏幕，实时 Agent 概览
• 连接、重连、前台返回时刷新
• 紧凑连接指示器移到左上角
• 浮动控件改为停靠工具栏
• 适配更小的手机屏幕
• 在解析的主会话中打开聊天

3. macOS 聊天界面改进

• 新增聊天模型选择器
• 重启后保留显式 thinking-level 选择
• 强化 provider-aware 会话模型同步

4. 多模态记忆索引（实验性）

可选启用图片和音频的记忆索引：

{
"memorySearch": {
"extraPaths": ["images", "audio"],
"embedding": {
"model": "gemini-embedding-2-preview"
}
}
}

• 使用 Gemini gemini-embedding-2-preview
• 支持可配置输出维度
• 维度变化时自动重建索引

5. OpenCode Go Provider

新增 OpenCode Go provider：

• 与 Zen 共享一个 OpenCode 密钥
• 在向导/文档中将 Zen 和 Go 视为一个 OpenCode 设置
• 保持运行时 provider 分离

6. Hunter Alpha / Healer Alpha 模型

OpenRouter 用户可以试用新的免费 stealth 模型：

• Hunter Alpha：免费 stealth 模型
• Healer Alpha：免费 stealth 模型
• 可用窗口约一周

7. Discord 自动线程归档

可配置自动创建线程的归档时间：

{
"channels": {
"discord": {
"autoArchiveDuration": "1h" // 1h, 1d, 3d, 1w
}
}
}

8. ACP Session 恢复

runtime: "acp" 的 sessions_spawn 支持 resumeSessionId：

{
"runtime": "acp",
"resumeSessionId": "existing-acp-session-id"
}

可以恢复现有的 ACPX/Codex 对话，而不是总是从头开始。

🔧 其他改进

Gateway/Node 待处理工作

新增内存待处理队列原语：

• node.pending.enqueue - 入队待处理工作
• node.pending.drain - 排空待处理工作

模型 Fallback 改进

• GLM-5 / DeepSeek：从用户可见文本中剥离模型控制标记
• Gemini MALFORMED_RESPONSE：视为可重试超时
• Venice / Poe 402：识别余额不足错误，触发 fallback
• HTTP 499：视为临时错误，触发 fallback

平台相关

• iOS 前台恢复：前台返回时立即重连，不再等待后续唤醒
• macOS LaunchAgent 重启：保持 LaunchAgent 在显式重启期间注册
• Discord 回复分块：解析有效的 maxLinesPerMessage 配置
• Telegram HTML 发送：分块长 HTML 消息，保留纯文本回退
• 飞书本地图片：mediaLocalRoots 通过 sendText 本地图片 shim

ACP 相关

• Session 恢复：loadSession 重放存储的用户和助手文本
• 工具流：tool_call 和 tool_call_update 事件包含文本内容和文件位置提示
• 图片附件：将规范化入站图片附件转发到 ACP 运行时
