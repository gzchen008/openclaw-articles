# OpenClaw 2026.4.15 更新：Claude Opus 4.7 默认登场 + Gemini 语音合成

> 你还在手动切换最新的 Claude 模型吗？OpenClaw 已经帮你搞定了。

## 一句话总结

OpenClaw 2026.4.15 正式发布，两大核心变化：**Anthropic 全线默认升级到 Claude Opus 4.7**，以及**全新 Gemini TTS 语音合成支持**。同时修复了超过 20 个稳定性与安全问题。

---

## 🚀 两大新功能

### 1. Claude Opus 4.7 成为默认模型

这次更新最重磅的变化——OpenClaw 中所有 Anthropic 相关的默认模型选择、`opus` 别名、Claude CLI 默认配置以及内置图片理解能力，全部升级到 **Claude Opus 4.7**。

这意味着什么？

- 不再需要手动配置最新模型名称
- `opus` 别名自动指向最新最强版本
- 图片理解开箱即用，无需额外配置
- Claude CLI 用户自动获得最新体验

如果你之前在配置文件里写了具体的模型版本号，现在可以简化为直接使用 `opus` 别名，以后更新模型时无需手动修改配置。

### 2. Gemini TTS 语音合成

Google Gemini 语音合成能力正式集成到 OpenClaw 的 `google` 插件中。这不是简单的文字转语音，而是一套完整的语音解决方案：

- 支持多种语音选择，找到最适合你场景的声音
- WAV 格式回复输出，适配各种播放场景
- PCM 电话语音输出，专门为语音通话场景优化
- 完整的配置文档和设置引导

对于使用 OpenClaw Talk Mode 的用户来说，这是又一个高质量本地/云端语音选择。

## 🔧 重要修复亮点

除了两个新功能，这次更新还包含大量修复，以下是最值得关注的：

### 稳定性修复

- **Skills 列表排序稳定化**：技能列表现在按名称排序，不再因为 `skills.load.extraDirs` 配置顺序不同而导致 prompt 缓存频繁失效。这意味着更少的 API 重复调用，更快的响应速度
- **上下文和记忆预算优化**：默认启动和技能加载的 prompt token 预算被缩减，`memory_get` 摘要默认有长度限制。长对话 session 拉取的上下文更精简，不会因为记忆文件过大而撑爆 token 窗口
- **CLI 升级不再卡死**：npm 全局升级后，旧的打包文件（dist chunks）会被自动清理，不再因为残留文件导致升级失败

### 安全修复

- **本地 MEDIA 信任锚定**：修复了一个安全问题——自定义工具定义如果名称与内置工具冲突，可能继承本地媒体文件的信任权限。现在这种情况会被明确拒绝
- **客户端工具名称冲突检测**：当客户端定义的工具名称与内置工具或其他客户端工具归一化后冲突时，会返回 `400` 错误，避免混淆

### Codex 用户必看

- **Codex 模型传输层修复**：旧版 `openai-codex` 配置中缺少 `api` 字段或指向 `chatgpt.com` 的情况会自动修复为正确的 Codex 传输路径，不再出现 HTML/Cloudflare 错误页面

### BlueBubbles 用户必看

- **图片附件下载恢复**：Node 22+ 环境下 BlueBubbles 的入站图片附件下载已修复，同时支持 `updated-message` 类型的附件 webhook 和自动重试

## 🛠 升级指南

升级非常简单，一行命令搞定：

```
openclaw update
```

或者通过 npm：

```
npm install -g openclaw@latest
```

升级后建议运行一次健康检查：

```
openclaw doctor
```

### 升级后检查清单

- 如果使用了 Anthropic 模型，确认 `opus` 别名正常工作
- 如果使用了自定义工具，检查是否有名称冲突警告
- 运行 `openclaw doctor` 确认配置健康
