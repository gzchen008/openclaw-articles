# OpenClaw v2026.3.11 发布：安全加固 + Ollama 一流支持 + iOS 大改版

> 这个版本包含一个重要的安全修复，以及 Ollama、iOS、macOS 等多项重大更新。

---

## ⚠️ 重要安全修复

### WebSocket 跨站攻击漏洞（GHSA-5wcw-8jjv-m286）

**影响**：在 `trusted-proxy` 模式下，未受信任的来源可能获得 `operator.admin` 访问权限。

**修复**：Gateway/WebSocket 现在强制对所有浏览器来源连接进行 origin 验证，无论是否存在代理头。

**建议**：立即升级到此版本。

---

## 🚀 重大新功能

### 1. Ollama 一流支持

现在 OpenClaw 内置了 Ollama 设置向导：

- **Local 模式**：本地模型，无需联网
- **Cloud + Local 模式**：云端 + 本地混合
- **浏览器云登录**：直接在浏览器中完成 Ollama Cloud 登录
- **精选模型推荐**：内置模型建议列表
- **智能云端模型处理**：跳过不必要的本地拉取

```bash
openclaw onboarding
# 选择 Ollama → 选择 Local 或 Cloud + Local
```

### 2. iOS Home Canvas 大改版

- **欢迎屏幕**：新增实时 Agent 概览，连接/重连/前台返回时自动刷新
- **停靠工具栏**：浮动控件改为停靠工具栏，更适合小屏手机
- **聊天优化**：在主会话中打开聊天，而不是合成 `ios` 会话

### 3. macOS 聊天界面增强

- **模型选择器**：聊天界面直接切换模型
- **Thinking 级别持久化**：显式选择的 thinking 级别在重启后保留
- **Provider 感知同步**：共享聊天组合器的会话模型同步更稳定

### 4. 多模态记忆索引（实验性）

可选启用图片和音频的记忆索引：

```json
{
  "memorySearch": {
    "extraPaths": ["images", "audio"],
    "embedding": {
      "model": "gemini-embedding-2-preview"
    }
  }
}
```

- 使用 Gemini `gemini-embedding-2-preview`
- 支持可配置输出维度
- 维度变化时自动重建索引

### 5. OpenCode Go Provider

新增 OpenCode Go provider：

- 与 Zen 共享一个 OpenCode 密钥
- 在向导/文档中将 Zen 和 Go 视为一个 OpenCode 设置
- 保持运行时 provider 分离

### 6. Hunter Alpha / Healer Alpha 模型

OpenRouter 用户可以试用新的免费 stealth 模型：

- **Hunter Alpha**：免费 stealth 模型
- **Healer Alpha**：免费 stealth 模型
- 可用窗口约一周

### 7. Discord 自动线程归档

可配置自动创建线程的归档时间：

```json
{
  "channels": {
    "discord": {
      "autoArchiveDuration": "1h"  // 1h, 1d, 3d, 1w
    }
  }
}
```

### 8. ACP Session 恢复

`runtime: "acp"` 的 `sessions_spawn` 支持 `resumeSessionId`：

```json
{
  "runtime": "acp",
  "resumeSessionId": "existing-acp-session-id"
}
```

可以恢复现有的 ACPX/Codex 对话，而不是总是从头开始。

## 🔧 其他改进

### Gateway/Node 待处理工作

新增内存待处理队列原语：

- `node.pending.enqueue` - 入队待处理工作
- `node.pending.drain` - 排空待处理工作

为休眠节点工作交付奠定基础。

### Git 运行时状态

自动忽略 gateway 生成的 `.dev-state` 文件，本地运行时状态不再显示为未跟踪的仓库噪音。

### Exec 子命令标记

子命令环境现在标记为 `OPENCLAW_CLI`，子进程可以检测是否从 OpenClaw CLI 启动。

## 💥 Breaking Changes

### Cron 通知收紧

隔离的 cron 任务不能再通过临时 agent 发送或主会话摘要进行通知。

**迁移**：运行 `openclaw doctor --fix` 自动迁移旧的 cron 存储和通知/webhook 元数据。

---

## 🐛 重要 Bug 修复

### 模型相关

- **GLM-5 / DeepSeek 文本清理**：移除泄漏的模型控制标记（`<|...|>` 和全角变体），防止内部分隔符到达用户
- **Kimi Coding 工具调用**：恢复 `anthropic-messages` 格式工具调用，停止降级为 XML/纯文本
- **Gemini MALFORMED_RESPONSE**：视为可重试的超时，预览模型枚举漂移会干净地回退

### 平台相关

- **iOS 前台恢复**：前台返回时立即重连，不再等待后续唤醒
- **macOS LaunchAgent 重启**：保持 LaunchAgent 在显式重启期间注册，通过分离的 launchd helper 处理自重启
- **Discord 回复分块**：解析有效的 `maxLinesPerMessage` 配置，长回复不再意外分割
- **Telegram HTML 发送**：分块长 HTML 消息，保留纯文本回退和静默参数
- **飞书本地图片**：`mediaLocalRoots` 通过 `sendText` 本地图片 shim，允许的本地图片路径再次上传为飞书图片

### ACP 相关

- **Session 恢复**：`loadSession` 重放存储的用户和助手文本
- **工具流**：`tool_call` 和 `tool_call_update` 事件包含文本内容和文件位置提示
- **图片附件**：将规范化入站图片附件转发到 ACP 运行时
