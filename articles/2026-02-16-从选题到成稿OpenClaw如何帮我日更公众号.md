## 🛠️ 第一步：用 OpenClaw 做选题库

日更最大的敌人是"不知道写什么"。

我让 OpenClaw 帮我维护一个**选题库**，每周自动更新：

### 操作方法

1. 告诉 OpenClaw 你的公众号定位：
```
我的公众号是关于 [你的领域] 的，目标读者是 [你的读者画像]，

1. 符合近期热点

```

2. OpenClaw 会生成选题列表，我选 3-5 个作为本周计划

3. 把选题保存到 `content-plan.md`，每天自动提醒

## ✍️ 第二步：从大纲到初稿（10分钟搞定）

有了选题，下一步就是写文章。

我的流程是这样的：

### 1. 让 OpenClaw 生成大纲（1分钟）

```
请为《[选题]》这篇文章生成一个详细大纲，要求：
1. 开头有钩子，3秒内抓住注意力
2. 中间结构清晰，有小标题
3. 结尾有行动号召
```

OpenClaw 会给出类似这样的大纲：

```markdown
## 为什么XXX很重要？（痛点）
## 常见的3个误区
## 我的解决方案（核心干货）
## 具体操作步骤
## 总结 + 行动号召
```

### 2. 逐段扩写成初稿（5分钟）

大纲确定后，我不会让 AI 一次性写完——那样出来的文章太"机器味"。

我选择**分段写**：

```
请根据大纲的"为什么XXX很重要"部分，写200字开头，要求：
1. 用一个痛点场景切入
2. 语言口语化，像在聊天
3. 不要用"首先、其次、最后"这种词
```

然后继续下一段。

### 3. 加入个人案例（2分钟）

这是让文章"有灵魂"的关键。

在核心干货部分，我会告诉 OpenClaw：

```
请在第三段加入一个真实案例：
"上周我遇到一个读者，他的情况是..."
```

这样文章就有了**真实感**，读者会觉得"这是活生生的人在写"。

## 💡 5 个进阶技巧

### 技巧 1：建立你的"写作风格库"

创建一个 `writing-style.md`，记录你的写作习惯：

```markdown
# 我的写作风格

## 常用开场白
- "你有没有这样的困扰？"
- "想象一下..."

## 常用结尾
- "现在就去试试吧"
- "明天见 👋"

## 禁用词
- 不要用"首先、其次、最后"
- 不要用"综上所述"
```

每次写文章时，让 OpenClaw 参考这个文件。

### 技巧 2：用子会话并行写多篇文章

如果要批量生产内容，可以用 OpenClaw 的**子会话功能**：

```
请启动3个子任务，同时写3篇文章：
1. 《...》
2. 《...》
3. 《...》
```

3篇文章同时生成，10分钟后全部完成。

### 技巧 3：素材库自动整理

写文章时用到的案例、数据、金句，让 OpenClaw 自动整理：

```
请把这篇文章中的金句提取出来，保存到 quotes.md
```

下次写文章时，可以直接引用：

```
请从 quotes.md 中选一句相关的金句，加到这篇文章的开头
```

### 技巧 4：A/B 测试标题

让 OpenClaw 同时生成多个标题版本：

```
请为这篇文章生成5个标题：
- 2个痛点型
- 2个数字型
- 1个悬念型
```

然后选一个最吸引人的。

### 技巧 5：定期复盘

每周让 OpenClaw 分析你的文章数据：

```
请分析本周3篇文章的共同点：
1. 哪些标题点击率高？
2. 哪些开场白效果好？
3. 下周可以优化的点？
```

持续迭代，越写越好。
