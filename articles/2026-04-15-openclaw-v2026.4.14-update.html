🧠 主动记忆：再也不用手动"记住这个"

以前用 OpenClaw，你得手动说"记住这个"或者"搜索记忆"才能调取历史信息。现在，新的 Active Memory 插件会在每次对话前自动运行一个记忆子代理，帮你拉取相关的偏好、上下文和历史细节。

核心能力：

• 支持三种上下文模式：消息级、最近对话级、全量上下文

• /verbose 实时查看记忆召回状态

• 高级提示词/思考模式覆盖，可精细调优

• 可选的转录持久化，方便调试

• 支持混合搜索（语义+词法），召回更精准

配置方式非常简单，在 openclaw.json 中启用 Active Memory 插件即可。官方文档：https://docs.openclaw.ai/concepts/active-memory

🤖 Codex 原生支持 + LM Studio 本地模型

OpenClaw 现在内置了 Codex 提供者，codex/gpt-* 模型会自动使用 Codex 管理的认证、原生线程、模型发现和压缩，而 openai/gpt-* 继续走标准 OpenAI 路径。一条命令就能接入最新的 GPT-5.4 Codex 能力。

同时新增内置 LM Studio 提供者，支持一键接入本地 OpenAI 兼容模型、运行时模型发现、流式预加载和记忆搜索嵌入向量。配合之前的 Ollama 支持，OpenClaw 的本地化能力越来越强。

🔒 安全加固：三个关键修复

这个版本在安全方面投入巨大：

• Shell 注入防御升级：移除 busybox/toybox 安全白名单，扩大 shell 包装器检测，阻止 env-argv 注入

• 审批授权修复：空审批者列表不再被当作"已授权"

• 浏览器安全全面加固：SSRF、主机名白名单、CDP 发现等全面收紧

• 插件加载精确化：只加载清单声明的必要运行时，缩小攻击面

🎙️ macOS Talk Mode 本地 MLX 语音

macOS 用户迎来实验性的本地 MLX 语音提供者，支持明确提供者选择、本地语音播放、打断处理和系统语音回退。Talk Mode 现在可以完全本地运行，不需要外部 API。

🔧 其他重要改进

• 执行策略命令：新增 openclaw exec-policy 命令，管理命令执行审批配置

• 网关命令发现：新增 commands.list RPC，远程客户端可发现所有可用命令

📋 全渠道稳定性修复
