OpenClaw v2026.3.12 发布：Dashboard v2 + Fast Mode + 安全加固

今天刚发布的大版本，重点是全新仪表盘和快速模式。

🚀 主要新特性

1. Control UI Dashboard v2（全新仪表盘）

模块化视图：
• 概览、聊天、配置、Agent、Session 五大模块
• 命令面板 - 快速访问所有功能
• 移动端底部标签 - 手机上更好用
• 增强聊天工具 - 斜杠命令、搜索、导出、置顶消息

2. GPT-5.4 Fast Mode（快速模式）

可配置的会话级快速切换：

• /fast 命令支持
• OpenAI/Codex 请求优化
• 更快的响应速度
• 每个模型可配置默认值

3. Provider Plugin 架构

迁移到插件架构：
• Ollama
• vLLM
• SGLang

好处：核心代码更模块化，更容易添加新 Provider

4. Kubernetes 部署支持

新增 K8s 安装路径：
• 原始 manifests
• Kind 集群设置
• 完整部署文档

5. Slack Block Kit 支持

Agent 可以发送 Block Kit 消息：
• 通过标准 Slack 出站路径
• 支持 channelData.slack.blocks
• 更丰富的消息格式

🔒 安全修复（重要！）

设备配对安全

问题：之前 /pair 和 openclaw qr 会在聊天或 QR 中嵌入共享凭证。

修复：切换到 短期 bootstrap token，不再嵌入共享凭证。

插件安全

问题：克隆的仓库可能执行 workspace plugin 代码。

修复：禁用隐式自动加载，需要显式信任才能执行。

Exec 审批安全

问题：Unicode 隐藏字符可能绕过检测。

修复：
• 零宽度和全角命令技巧无法绕过
• POSIX 大小写敏感保持
• Ruby 审批流程加固

Webhook 安全

平台 修复
飞书 要求 encryptKey + verificationToken
LINE 空事件 POST 也需要签名
Zalo 无效密钥猜测限流

🐛 重要 Bug 修复

Kimi Coding

问题：工具调用降级为 XML/纯文本。
修复：恢复 anthropic-messages 工具原生格式。

Telegram

问题：模型选择器不持久化。
修复：内联模型按钮正确工作。

Mattermost

问题：重复消息（一个线程、一个顶层）。
修复：修复 block streaming 去重逻辑。

📊 升级命令

openclaw update

升级后建议运行：

openclaw doctor --fix

🎯 总结

v2026.3.12 的核心更新：

1. Dashboard v2 - 全新模块化仪表盘
2. Fast Mode - GPT-5.4 和 Claude 快速模式
3. Provider Plugin - Ollama/vLLM/SGLang 插件化
4. K8s 支持 - Kubernetes 部署路径
5. 安全加固 - 配对、插件、审批、Webhook 全面加固
