🔥 OpenClaw 2026.4.11 更新：ChatGPT 对话导入、视频生成大升级

OpenClaw 又双叒叕更新了！这次带来了一大波实用功能，尤其是记忆系统和视频生成方面的重大升级。

如果你一直在纠结"怎么把 ChatGPT 里的优质对话迁移过来"，这次终于有答案了。

💭 ChatGPT 对话导入 Dreaming 记忆系统

这是本次更新我最喜欢的新功能。

以前你在 ChatGPT 里积累的那些高质量对话——技术讨论、创意灵感、学习笔记——全部散落在 OpenAI 的服务器上，没法和你的 AI Agent 共享。

现在，一键导入。

新增了 Imported Insights 和 Memory Palace 日记两个子标签页，你可以在 UI 中直接查看：

• 导入的原始对话内容

• 编译后的 Wiki 页面（自动整理）

• 完整的源页面追溯

这意味着你可以把所有 AI 对话精华汇聚到一处，构建真正的个人知识图谱。不管你之前用的是 ChatGPT、Claude 还是其他什么，知识都可以跟着你走。

🎬 视频生成工具大幅增强

video_generate 工具这次迎来了五项重要改进：

• URL 直传交付：生成的视频不再需要载入内存，直接通过 URL 投递，大文件也能秒级返回

• 参考音频输入：给视频配音乐/旁白，音视频联合创作不再是梦

• 自适应宽高比：横屏竖屏方屏随便选，不用手动计算尺寸

• 更高图片输入上限：多图参考生成更流畅，支持更多素材同时输入

• 类型化 providerOptions：各视频提供商可以暴露更丰富的生成模式，专业用户有更多控制权

特别感谢社区贡献者 @xieyongliang 的出色工作！

💬 飞书文档评论会话升级

飞书集成继续打磨。文档评论会话现在支持：

• 更丰富的上下文解析：理解评论的完整语境

• 评论表情回应：可以用 emoji 回应评论了

• 打字反馈：对方能看到你正在输入，对话体验更接近即时聊天

如果你在飞书文档里和 AI 讨论问题，体验会流畅很多。

🟦 Microsoft Teams：表情回应支持

Teams 集成新增表情回应功能：

• 支持回应列表查询

• Graph 分页支持

• 委托 OAuth 设置（应用身份读取 + 用户身份发送）

Teams 用户终于可以用 emoji 和 AI 互动了。

🔌 插件系统：声明式激活与设置

插件清单现在可以声明激活条件和设置描述符，这意味着：

• 插件认证、配对、配置流程不再需要核心代码硬编码

• 插件开发更规范、更独立

• 第三方插件作者可以更自由地扩展功能

对插件生态的发展是个重要基础。

🔧 重要修复

• Codex OAuth：修复新登录时 scope 被重写导致的 invalid_scope 错误

• 音频转录：修复 OpenAI/Groq/Mistral 转录失败的问题

• macOS Talk Mode：首次授权麦克风后无需二次切换

• WhatsApp：修复默认账户配置识别问题

• Google Veo：移除不支持的 numberOfVideos 字段

• ACP 子代理：子代理进度信息不再泄露到父会话

• 代理故障转移：修复跨提供商切换时继承失败状态的问题

• Telegram Topics：修复话题会话路径交替问题

📦 一键升级

npm update -g openclaw && openclaw gateway restart

升级完重启就好，所有配置自动保留。
