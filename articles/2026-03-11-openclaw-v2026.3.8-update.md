# OpenClaw 2026.3.8 发布：备份功能、Talk 模式增强、LLM 搜索模式

> 发布日期：2026年3月9日

OpenClaw 2026.3.8 版本正式发布！本次更新带来了多项重要新功能和改进，包括全新的备份系统、Talk 模式静默超时配置、Brave 搜索 LLM Context 模式等。同时修复了大量问题，提升了稳定性和安全性。

---

## 🎉 新功能亮点

### 📦 CLI 备份功能

新增 `openclaw backup create` 和 `openclaw backup verify` 命令，支持本地状态归档：

- `--only-config`：仅备份配置
- `--no-include-workspace`：排除工作区
- 清单和负载验证
- 破坏性操作时的备份引导

```bash
# 创建完整备份
openclaw backup create

# 仅备份配置
openclaw backup create --only-config

# 验证备份
openclaw backup verify backup.tar.gz
```

### 🎙️ Talk 模式增强

新增 `talk.silenceTimeoutMs` 配置项，可自定义语音识别的静默等待时间：

```json
{
  "talk": {
    "silenceTimeoutMs": 2000
  }
}
```

### 🔍 Brave 搜索 LLM Context 模式

新增 `tools.web.search.brave.mode: "llm-context"`，启用 Brave 的 LLM Context 端点：

- 返回提取的上下文片段
- 包含来源元数据
- 更适合 AI 直接使用

```json
{
  "tools": {
    "web": {
      "search": {
        "brave": {
          "mode": "llm-context"
        }
      }
    }
  }
}
```

### 🖥️ TUI 自动 Agent 推断

TUI 现在可以自动推断当前工作区的活跃 agent，无需手动指定：

- 从工作区配置中检测
- 保留显式 `agent:` 目标指定

### 📱 macOS 远程网关配置

macOS 应用新增远程网关 token 字段：

- 支持远程模式配置
- 保留现有的加密 token 配置
- 对不兼容的 token 格式发出警告

### 🔗 ACP 来源元数据

新增可选的 ACP 来源追踪：

```bash
openclaw acp --provenance off|meta|meta+receipt
```

### 📝 版本输出增强

`openclaw --version` 现在包含 git commit hash（如有）：

```bash
openclaw --version
# 2026.3.8 (abc1234)
```

## 🐛 重要修复

### 稳定性改进

- **macOS 更新恢复**：重新启用被禁用的 LaunchAgent 服务
- **Telegram DM 路由**：修复同一消息触发重复回复的问题
- **Cron 交付确认**：修复 Telegram 消息虚假成功报告
- **Matrix DM 路由**：更安全的 homeserver 兼容性处理

### 安全修复

- **浏览器 SSRF**：阻止严格模式下的私有网络重定向
- **MS Teams 权限**：修复 allowlist 绕过问题
- **system.run**：绑定批准的脚本快照，防止执行时篡改
- **Skills 下载**：固定工具目录，防止路径重定向攻击

### 平台特定修复

- **Android**：移除自更新、后台定位等权限
- **Podman/SELinux**：自动检测并添加 `:Z` 重标签
- **Docker 镜像**：精简开发依赖，减小镜像体积
