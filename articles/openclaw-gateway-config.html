⚙️ OpenClaw Gateway 配置全攻略 | 从入门到精通

OpenClaw Gateway 是整个系统的大脑中枢，所有模型、工具、Agent、通道的配置都在这里。

今天带你逐一解锁每个配置模块，看完你就能完全掌控自己的 AI 助手！

📁 配置文件在哪里？

配置文件路径：

~/.openclaw/openclaw.json

查看当前配置：

openclaw gateway config

🗺️ 配置结构总览

OpenClaw Gateway 配置包含11 大模块：

模块 功能 重要性
models AI 模型配置（Provider、API） 核心
agents Agent 行为、记忆、工具权限 核心
auth 认证方式（API Key / OAuth） 核心
tools 工具权限、沙箱、安全策略 核心
bindings 通道 → Agent 绑定关系 核心
browser 浏览器自动化配置 高级
env 环境变量（密钥、API Secret） 高级
messages 消息队列、防抖、TTS 高级
logging 日志级别、输出格式 调试
diagnostics 诊断追踪、OpenTelemetry 调试
ui UI 主题、助手名称 外观

🤖 1. models - 模型配置

作用：配置 AI 模型 Provider（OpenAI、Claude、GLM 等）

常用配置项：

• providers：定义模型提供商

• baseUrl：API 端点

• apiKey：API 密钥（建议用 env.vars 代替）

• models：模型列表（id、name、contextWindow）

示例配置：

{

"models": {

"providers": {

"zai": {

"baseUrl": "https://api.zukijourney.com/v2",

"apiKey": "${ZAI_API_KEY}",

"models": [

{ "id": "glm-5", "name": "GLM-5" }

]

}

}

}

}

🧠 2. agents - Agent 配置

作用：定义 AI 助手的行为、记忆、权限

核心配置项：

• defaults.model：默认模型（primary + fallbacks）

• defaults.workspace：工作目录

• defaults.memorySearch：记忆检索配置

• defaults.thinkingDefault：思考深度（off/minimal/low/medium/high）

• defaults.heartbeat：主动轮询配置

• list：Agent 列表（id、name、workspace）

示例：配置默认模型

{

"agents": {

"defaults": {

"model": {

"primary": "zai/glm-5",

"fallbacks": ["zai/glm-4.7", "kimi-coding/k2p5"]

},

"workspace": "~/.openclaw/workspace",

"thinkingDefault": "low"

}

}

}

🔐 3. auth - 认证配置

作用：配置 API 认证方式

认证方式：

• api_key：直接用 API Key

• oauth：OAuth 授权（如 GitHub Copilot）

• token：Bearer Token

示例：配置认证

{

"auth": {

"profiles": {

"my-openai": {

"provider": "openai",

"mode": "api_key"

}

}

}

}

🛠️ 4. tools - 工具权限配置

作用：控制 AI 能用哪些工具

预设 Profile：

• minimal：最小权限（只读）

• coding：编码权限（读写文件）

• messaging：消息权限（发消息）

• full：全部权限

关键配置：

• allow：允许的工具列表

• deny：禁止的工具列表

• exec：Shell 命令权限

• web.search：网络搜索配置

示例：配置工具权限

{

"tools": {

"profile": "full",

"exec": {

"host": "gateway",

"security": "allowlist"

},

"web": {

"search": { "enabled": true }

}

}

}

🔗 5. bindings - 通道绑定

作用：把消息通道（Discord、Telegram）绑定到 Agent

示例：绑定 Discord 通道

{

"bindings": [

{

"agentId": "main",

"match": {

"channel": "discord",

"peer": { "kind": "direct", "id": "YOUR_USER_ID" }

}

}

]

}

🌐 6. browser - 浏览器配置

作用：配置浏览器自动化

关键配置：

• enabled：是否启用

• profiles：浏览器配置文件

• cdpUrl：Chrome DevTools Protocol 地址

• headless：无头模式

🔐 7. env - 环境变量

作用：存储密钥、API Secret（不要硬编码在配置里！）

示例：配置环境变量

{

"env": {

"vars": {

"OPENAI_API_KEY": "sk-xxx",

"ZAI_API_KEY": "zai-xxx"

}

}

}

💬 8. messages - 消息配置

作用：控制消息队列、防抖、TTS

消息队列模式：

• steer：新消息打断旧回复

• queue：排队处理

• collect：收集后批量处理

TTS 配置：

• tts.enabled：启用语音

• tts.provider：elevenlabs / openai / edge

• tts.auto：自动朗读（off / always / inbound）

📝 9. logging - 日志配置

作用：控制日志输出

日志级别：

silent → fatal → error → warn → info → debug → trace

{

"logging": {

"level": "info",

"consoleLevel": "warn",

"consoleStyle": "pretty"

}

}

🎨 10. ui - UI 配置

作用：自定义助手外观

{

"ui": {

"assistant": {

"name": "小J",

"avatar": "https://example.com/avatar.png"

}

}

}

🔄 11. diagnostics - 诊断配置

作用：调试、追踪、OpenTelemetry

关键配置：

• cacheTrace：缓存追踪日志

• otel：OpenTelemetry 集成

💡 实用技巧

1. 使用 config.patch 而非 config.apply

错误做法：每次都重写整个配置文件

正确做法：只更新需要的字段

openclaw gateway config.patch '{ "agents": { "defaults": { "thinkingDefault": "high" } } }'

2. 模型 Fallback 机制

配置多个备用模型，主模型失败时自动切换：

"model": {

"primary": "zai/glm-5",

"fallbacks": ["kimi-coding/k2p5", "openai/gpt-4o"]

}

3. 安全存储密钥

不要在 models.providers.apiKey 里直接写密钥！

应该用 env.vars 存储密钥，然后用 ${VAR_NAME} 引用。

4. 查看配置 Schema

openclaw gateway config.schema

⚠️ 常见问题

Q: 配置修改后没生效？

A: 配置修改后会自动重启 Gateway，等待几秒后再试。

Q: 模型调用失败？

A: 检查 auth.profiles 和 env.vars 是否配置正确。

Q: 工具执行被拒绝？

A: 检查 tools.profile 和 tools.exec.security 配置。
