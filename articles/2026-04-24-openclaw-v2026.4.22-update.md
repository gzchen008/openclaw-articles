# OpenClaw 2026.4.22 更新：xAI 全家桶来了，还有本地终端模式和诊断神器

> 从图片生成到语音转录，从无网关聊天到一键诊断导出，这次更新分量十足。

---

## 一句话概括

OpenClaw 2026.4.22 新增了 xAI 图片生成、语音合成与转录的全链路支持，加入了无需 Gateway 的本地终端模式，并带来了大量开发者体验提升——插件加载速度提升近 90%，doctor 诊断提速 74%，还有腾讯云、Amazon Bedrock Mantle 等新 Provider 接入。

---

## 一、xAI 全家桶：图片、语音、转录一站搞定

这可能是本次更新最重磅的部分。OpenClaw 现在完整支持 xAI 的 Grok 系列能力：

- **图片生成**：支持 `grok-imagine-image` 和 `grok-imagine-image-pro` 两个模型，还支持参考图编辑模式
- **语音合成（TTS）**：内置 6 种 xAI 原生语音，支持 MP3、WAV、PCM、G.711 四种音频格式输出
- **语音转录（STT）**：`grok-stt` 支持音频文件转文字，还支持 Voice Call 场景的实时流式转录

如果你已经在用 xAI 的 Grok 模型做对话，现在可以直接把图片和语音能力也接上了，不用额外配置其他 Provider。

## 二、语音通话转录扩展：Deepgram、ElevenLabs、Mistral

Voice Call 的实时流式转录不再只有 OpenAI 和 xAI 两个选择。本次更新新增了三个 Provider：

- **Deepgram**：高速语音识别，适合实时场景
- **ElevenLabs**：除实时转录外，还新增了 Scribe v2 批量音频转录，可以直接处理收到的语音消息
- **Mistral**：欧洲用户的好选择

这意味着你的语音助手方案更灵活了——可以根据延迟、成本、语言需求自由搭配。

## 三、本地终端模式：不需要 Gateway 也能聊

新增的 TUI 本地嵌入模式（Local Embedded Mode）是一个很实用的功能：

- **不需要启动 Gateway 守护进程**，直接在终端里跑 OpenClaw 聊天
- **插件审批门仍然生效**，不会因为本地运行就跳过安全检查
- 适合快速调试、离线开发、或资源受限的环境

简单理解：以前要 `openclaw gateway start` 再聊，现在直接终端里就能对话了。

## 四、新 Provider 接入：腾讯云和 Amazon Bedrock Mantle

### 腾讯云

OpenClaw 正式内置腾讯云 Provider 插件：

- 支持 TokenHub 快速接入配置
- 包含 `hy3-preview` 模型目录
- 分层定价元数据自动适配

### Amazon Bedrock Mantle

新增 Claude Opus 4.7 通过 Mantle 的 Anthropic Messages 路由调用：

- 使用 Provider 自有的 Bearer Auth 流式传输
- 不再把 AWS Bearer Token 当 Anthropic API Key 用
- 终于可以在 Bedrock 上正确调用 Opus 4.7 了

## 五、开发者体验大升级

### `/models add` 命令

以前要手动改配置文件才能加新模型。现在直接在聊天里输入 `/models add`，注册模型后立即使用，不用重启 Gateway。

### 插件加载提速 82-90%

内置插件现在优先使用原生 Jiti 加载已打包的 dist 模块，实测加载时间减少了 82% 到 90%。启动更快，重启也不心疼了。

### Doctor 诊断提速 74%

`openclaw doctor --non-interactive` 的运行时间减少了约 74%——通过懒加载插件路径和优先使用已编译产物实现。

### Gateway 一键诊断导出

新增支持就绪的诊断导出功能：

- 自动脱敏的日志和配置
- 包含状态、健康检查、稳定性快照
- 直接用于 Bug 报告，不用手动收集信息

### Tokenjuice 支持

新增可选的 Tokenjuice 插件，专门为 Pi 嵌入式运行设计，能自动压缩冗长的 exec 和 bash 工具输出，节省 Token 消耗。

## 六、其他值得关注的更新

### OpenAI 原生 Web Search

使用 OpenAI Responses 模型时，如果没有固定其他搜索 Provider（如 Brave），OpenClaw 会自动使用 OpenAI 原生的 web_search 工具。配置更简单了。

### WhatsApp 增强

- 支持配置原生回复引用模式（`replyToMode`）
- 群组和私聊支持独立 System Prompt 配置，支持通配符和账号级覆盖

### Claude CLI 改进

- 默认使用热 stdio 会话，响应更快
- Gateway 重启后自动恢复之前的 Claude 会话

### GPT-5 Prompt 统一

GPT-5 的 Prompt 覆盖层移到了共享 Provider 运行时，无论是通过 OpenAI、OpenRouter 还是 Codex 调用，行为都一致了。

### Pi 更新到 0.68.1

新增模型包括 `opencode-go/kimi-k2.6`、Qwen、GLM、MiMo、MiniMax 等，模型目录从 pi 直接获取，不再依赖插件维护的别名。

### 本地模型 Token 统计修复

vLLM、SGLang、llama.cpp、LM Studio 等本地推理后端现在被标记为兼容流式用量统计，Token 计数不再显示 unknown。

## 七、升级建议

```bash
npm update -g openclaw
openclaw gateway restart
```

如果你在用 xAI、腾讯云、或 Amazon Bedrock，这次升级值得立刻安排。本地终端模式和诊断导出也是实打实的效率工具。
