🚀 OpenClaw 新版本发布

版本 v2026.3.31

📝 更新概述

OpenClaw v2026.3.31 已正式发布！本次更新带来了大量新功能、改进和重要修复，进一步提升了系统的稳定性和用户体验。该版本主要聚焦于安全性增强、性能优化和新增的平台支持。

📅 发布时间：2026年3月31日
🔧 类型：功能更新 + 安全增强
⚡ 兼容性：向后兼容（部分配置需注意迁移）

⚠️ 重要变更

节点执行系统优化

  • 移除了重复的 nodes.run shell 包装器，确保节点 shell 执行始终通过 exec host=node 进行
  • 保持节点特定的能力在 nodes invoke 和专用 media/location/notify actions 中

插件 SDK 更新

  • 弃用传统的 provider 兼容子路径和捆绑的 provider 设置
  • 迁移警告会自动发出，引导用户使用新的 documented entrypoints
  • 保持 openclaw/plugin-sdk/* 作为向前路径

安全性增强

  • 插件安装和技能依赖安装现在默认失败关闭，需要显式 override
  • trusted-proxy 配置现在拒绝混合共享令牌配置
  • 节点命令在节点配对批准前保持禁用状态

✨ 新功能

🔄 背景任务管理

  • 将任务转换为真正的共享后台运行控制平面，统一 ACP、subagent、cron 和后台 CLI 执行
  • SQLite 支持的账本系统，改善任务生命周期管理和恢复
  • 添加了线性任务流程控制 surface，支持任务的取消和状态查看

🤖 ACP/Plugins 增强

  • 添加默认关闭的 ACPX plugin-tools MCP bridge 配置
  • 支持可选的 streamable-http 传输选择和每服务器连接超时
  • 改进内置桥接打包/日志路径，确保全局安装正常工作

📱 新平台支持

  • QQ Bot：添加为捆绑的 channel plugin，支持多账户设置、slash 命令、提醒和媒体收发
  • LINE 增强：支持图片、视频和音频外发，包括显式预览/跟踪处理
  • WhatsApp：支持对传入消息进行 emoji 反应，提供更自然的对话交互

🔍 搜索与历史

  • Matrix 添加可选的房间历史上下文支持
  • Matrix 网络支持显式的 HTTP(S) 代理配置
  • Matrix 支持草稿流式回复，部分回复可以在同一消息中更新

🐛 重要修复

🔐 安全性修复

  • 修复 ACP 危险工具名称覆盖问题，引入语义审批类
  • 改进 SecretRef 反序列化的往返恢复
  • 增强 HTTP 工具调用的授权检查
  • 修复 Android 设备信息从包管理器读取版本元数据

🔄 运行时修复

  • 修复 ACP child runs 的注册和生命周期清理
  • 改进 Agents/Anthropic failover 机制，正确处理 api_error
  • 修复 Agents/system prompt 中的 agent.name 插值
  • 改善 Config/SecretRef 在 Control UI 中的验证

📱 移动端修复

  • 修复 Android 配对时的重复 push receiver 条目问题
  • 改善 iOS Live Activities 在 Swift 6 构建中的兼容性
  • 修复 LINE markdown 中下划线的保留逻辑

🚀 性能优化

  • 改善 Diffs 的 SSR 预加载工作，减少不必要的渲染
  • 优化 Agents/LLM 的 idle-stream timeout 配置
  • 改进 MCP 工具的 materialize 和名称处理
  • 增强执行审批的性能和用户体验

📋 更新建议

  • 安全用户：请仔细检查 breaking changes 部分，某些功能配置可能需要调整
  • 插件开发者：注意插件 SDK 的迁移路径，建议尽快更新到新的 entrypoints
  • 企业用户：新的安全性改进可能需要更新相关的安全策略和审批流程
  • 移动端用户：享受更好的 Android/iOS 体验和新的平台支持
