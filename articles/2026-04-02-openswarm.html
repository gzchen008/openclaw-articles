OpenSwarm：让 AI Agent 团队自主协作的开源神器

你有没有想过，如果 AI Agent 能像团队一样自主协作，会是什么样？

今天给大家介绍一个超酷的开源项目：OpenSwarm ⭐ 316 stars

它是一个自主 AI Agent 编排器，基于 Claude Code CLI，能让多个 AI Agent 像团队一样协作完成复杂任务！

🤔 OpenSwarm 是什么？

简单来说，OpenSwarm 是一个AI 开发团队自动化系统：

• 从 Linear（项目管理工具）自动拉取任务

• 分配给 Worker/Reviewer/Tester/Documenter 四个角色

• 自动执行代码生成、审查、测试、文档编写

• 通过 Discord 实时汇报进度

• 拥有长期记忆（LanceDB 向量存储）

• 自动改进 PR（监控 CI、修复冲突）

核心亮点：这不是简单的 AI 聊天，而是真正的团队协作 —— Worker 写代码，Reviewer 审查，Tester 测试，Documenter 写文档，全自动化！

🏗️ 核心架构

OpenSwarm 的架构设计非常精妙：

1️⃣ PairPipeline（配对流水线）

• Worker：生成代码（基于 Linear issue）

• Reviewer：审查代码质量（APPROVE/REVISE/REJECT）

• Tester：运行测试（可选）

• Documenter：生成文档（可选）

• 循环迭代：最多 N 轮直到通过

2️⃣ 认知记忆系统（Cognitive Memory）

• LanceDB：向量数据库，存储长期记忆

• Xenova E5：768 维向量嵌入

• 混合检索：相似度 + 重要性 + 新近度 + 频率

• 记忆类型：信念、策略、用户模型、系统模式、约束

3️⃣ 知识图谱（Knowledge Graph）

• 静态代码分析

• 依赖关系映射

• 影响分析（改动文件会影响哪些模块）

• 冲突检测（并发任务的文件级冲突）

4️⃣ Discord 控制

• !auto start：启动自动模式

• !pair run：手动执行配对任务

• !provider codex：切换到 Codex

• !schedule：管理定时任务

🎯 适用场景

• 自动化开发：从 Linear issue 自动生成代码、测试、文档

• PR 自动改进：监控 CI 失败，自动修复，解决冲突

• 长期项目维护：记忆系统记住项目历史，避免重复犯错

• 团队协作增强：AI Agent 作为"虚拟团队成员"

🚀 快速开始

前置要求

• Node.js >= 22

• Claude Code CLI（claude -p）或 OpenAI Codex CLI

• Discord Bot Token

• Linear API Key

安装步骤

# 克隆项目
git clone https://github.com/unohee/OpenSwarm.git
cd OpenSwarm

# 安装依赖
npm install

# 配置
cp config.example.yaml config.yaml

# 创建 .env 文件
DISCORD_TOKEN=your-token
DISCORD_CHANNEL_ID=your-channel-id
LINEAR_API_KEY=your-key
LINEAR_TEAM_ID=your-team-id

# 启动
npm run start

启动自动模式

在 Discord 中输入：

!auto start --pair

系统会自动开始从 Linear 拉取任务并执行！

• 运行时切换：!provider codex

3️⃣ Rich TUI 聊天界面

• Claude Code 风格暗色主题

• 5 个标签页：Chat、Projects、Tasks、Stuck、Logs

• 流式响应 + 极客主题加载消息

• 快捷键：Tab 切换、Enter 发送、Esc 退出

4️⃣ PR 自动改进

• 监控开放 PR

• 自动修复 CI 失败

• 自动解决合并冲突

• 持续重试直到所有检查通过

📊 技术栈

• Runtime：Node.js 22+ (ESM)

• Language：TypeScript（严格模式）

• Agent Execution：Claude Code CLI / OpenAI Codex CLI

• Task Management：Linear SDK

• Communication：Discord.js 14

• Vector DB：LanceDB + Apache Arrow

• Embeddings：Xenova/transformers（multilingual-e5-base）

• Scheduling：Croner

• Config：YAML + Zod 验证
