🏠 为什么选择 Home Assistant？

Home Assistant 是目前最强大的开源智能家居平台，支持超过 1000 种设备：

• 品牌中立：支持 Apple HomeKit、Google Home、小米、飞利浦等

• 隐私优先：本地运行，数据不上传云端

• 社区活跃：丰富的集成和自动化模板

但：Home Assistant 的自动化配置需要写 YAML，学习曲线陡峭。

OpenClaw 的加入改变了这一切 —— 让你用自然语言控制智能家居。

🔧 前置准备

在开始之前，你需要：

1. 一台运行 OpenClaw 的设备（Mac/Linux/树莓派）

2. Home Assistant 实例（建议使用 HAOS 或 Docker 部署）

3. 确保两者在同一网络，或通过内网穿透互通

安装 OpenClaw（开源免费）

# 使用 npm 安装（推荐）

npm install -g @jackwener/openclaw

# 或从源码构建

git clone https://github.com/openclaw/openclaw.git

cd openclaw && npm install && npm run build

获取 Home Assistant 信息

登录 Home Assistant，进入 设置 → 关于，记录：

• 访问地址（如 http://192.168.1.100:8123）

• 长期访问令牌（用户 → 安全 → 创建令牌）

⚙️ 配置 OpenClaw 连接

方法一：使用 OpenClaw 节点功能

如果你有树莓派或其他设备，可以通过 nodes 工具连接：

# ~/.openclaw/openclaw.json

{

"nodes": {

"home-pi": {

"host": "192.168.1.100",

"type": "raspberry-pi"

}

}

}

方法二：通过 HTTP API 集成

OpenClaw 可以通过 Home Assistant 的 REST API 控制设备：

# 在 OpenClaw 中配置

{

"homeAssistant": {

"url": "http://192.168.1.100:8123",

"token": "YOUR_LONG_LIVED_ACCESS_TOKEN"

}

}

场景二：离家模式

我出门了

AI 自动执行：

• 启动安防摄像头

• 调低空调温度

场景三：定时自动化

通过 OpenClaw 的 cron 功能设置定时任务：

# 每天早上 7:00 自动开窗帘

{

"name": "morning-routine",

"schedule": {"kind": "cron", "expr": "0 7 * * *"},

"payload": {

"kind": "http",

"url": "http://HA地址:8123/api/services/cover/open_cover",

"body": {"entity_id": "cover.living_room_curtain"}

}

}

1. 不要将 Home Assistant 暴露在公网，使用 VPN 或内网穿透

4. 启用 Home Assistant 的双因素认证
