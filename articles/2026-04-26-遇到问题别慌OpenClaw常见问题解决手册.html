一、安装阶段常见问题

1️⃣ npm install -g openclaw 报权限错误

症状：EACCES permission denied 或 npm ERR! code EPERM

原因：npm 全局安装目录权限不足

解决方案：

方案 A（推荐）——修改 npm 全局路径：

• mkdir -p ~/.npm-global

• npm config set prefix '~/.npm-global'

• echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.zshrc

• source ~/.zshrc

• npm install -g openclaw

方案 B——使用 nvm 管理 Node.js 版本，安装后自动解决权限问题。

2️⃣ Node.js 版本过低

症状：SyntaxError: Unexpected token 或 ERR_MODULE_NOT_FOUND

原因：OpenClaw 要求 Node.js 20+，建议 v22+

解决方案：

• node -v 先检查当前版本

• 如果低于 v20，用 nvm 升级：nvm install 22 && nvm use 22

3️⃣ 安装成功但 openclaw 命令找不到

症状：command not found: openclaw

原因：npm 全局 bin 目录不在 PATH 中

解决方案：

• npm config get prefix 查看 npm 全局路径

• 把输出的 bin 目录加到 PATH：echo 'export PATH=$(npm config get prefix)/bin:$PATH' >> ~/.zshrc

• source ~/.zshrc

二、Gateway 启动问题

4️⃣ Gateway 启动失败：端口被占用

症状：Error: listen EADDRINUSE: address already in use :::5800

原因：另一个 OpenClaw 实例或其他程序占用了端口

解决方案：

• lsof -i :5800 查找占用进程

• kill -9 <PID> 根据输出 kill 掉对应进程

• openclaw gateway start 重新启动

5️⃣ Gateway 启动后立即退出

症状：openclaw gateway start 后没有报错但立刻退出

原因：配置文件有误或缺少必要字段

解决方案：

• openclaw gateway start --verbose 查看详细日志

• cat ~/.openclaw/openclaw.json | head -50 检查配置文件

常见配置问题：

• API Key 格式错误（多了空格或引号）

• JSON 语法错误（多了逗号、少了括号）

• model 名称拼写错误

6️⃣ Gateway 频繁重启或内存占用过高

症状：Gateway 自动重启，或者内存使用超过 1GB

原因：会话历史过长、日志堆积

解决方案：

• rm -rf ~/.openclaw/logs/*.log.*.gz 清理旧日志

• openclaw gateway restart 重启

• openclaw sessions list 检查活跃会话数

三、消息通道问题

7️⃣ Discord Bot 不回复消息

症状：Gateway 正常运行，但 Discord 消息无响应

排查步骤：

第一步，检查 Bot 是否在线：在 Discord 服务器成员列表中，Bot 头像是否有绿色在线标识。

第二步，检查 Token 配置：openclaw configure --show | grep discord

第三步，检查权限：确保 Bot 有「发送消息」和「读取消息历史」权限。

第四步，检查频道映射：确认 openclaw.json 中 discord.channels 配置了正确的频道 ID。

8️⃣ 微信/飞书消息延迟严重

症状：消息要等几十秒甚至几分钟才收到回复

原因：模型 API 响应慢或网络延迟

解决方案：

• 切换到更快的模型（如 gpt-4o-mini 或 claude-3.5-haiku）

• 检查网络到 API 端点的延迟

• 如果使用代理，检查代理服务器性能

9️⃣ 飞书 Bot 发送失败：权限不足

症状：日志显示 permission denied 或 no authority

原因：飞书应用权限配置不完整

解决方案：

登录飞书开放平台 → 你的应用 → 权限管理，确保已开通：

• im:message — 发送消息

• im:message.receive_v1 — 接收消息

• im:resource — 获取资源

然后重新发布应用版本。

四、模型与 API 问题

🔟 API Key 无效

症状：401 Unauthorized 或 invalid_api_key

解决方案：

• 确认 Key 没有多余的空格或换行符

• 确认 Key 没有过期或被撤销

• 确认 Key 对应的模型有访问权限

重新配置：openclaw configure --section model

1️⃣1️⃣ 回复内容被截断

症状：AI 回复到一半突然中断

原因：达到了模型的最大输出 token 限制

解决方案：

在 openclaw.json 中调整 maxTokens 参数：

• "maxTokens": 8192

如果内容确实很长，可以在对话中告诉 AI「请继续」。

1️⃣2️⃣ 模型响应速度不稳定

症状：有时秒回，有时要等很久

原因：API 服务商负载波动

解决方案：

配置备用模型（fallback），OpenClaw 会自动降级：

• 主模型：anthropic/claude-sonnet-4

• 备用：openai/gpt-4o、openai/gpt-4o-mini

五、文件与权限问题

1️⃣3️⃣ AI 无法读取本地文件

症状：回复「找不到文件」或「权限不足」

原因：文件路径不在工作区内

解决方案：

OpenClaw 默认工作目录是 ~/.openclaw/workspace。确保文件放在这个目录下，或者使用绝对路径。

1️⃣4️⃣ Git 操作失败

症状：git push 报 Permission denied (publickey)

解决方案：

• ssh -T git@github.com 检查 SSH Key 是否配置

• 如果没有 Key：ssh-keygen -t ed25519

• cat ~/.ssh/id_ed25519.pub 复制到 GitHub SSH Keys 设置

六、性能问题

1️⃣5️⃣ Gateway 占用 CPU 过高

症状：openclaw 进程 CPU 占用持续超过 50%

排查步骤：

• openclaw sessions list 检查是否有大量并发请求

• openclaw cron list 检查是否有死循环的 Cron 任务

• openclaw gateway restart 重启释放资源

1️⃣6️⃣ 磁盘空间不足

症状：日志报 ENOSPC: no space left on device

解决方案：

• du -sh ~/.openclaw/* 查看目录占用

• find ~/.openclaw/logs -name "*.gz" -mtime +30 -delete 清理旧日志

• find ~/.openclaw/sessions -mtime +60 -exec rm -rf {} + 清理旧会话

七、终极排查法

如果上面的方案都没解决你的问题，试试这个万能排查流程：

第一步：查看日志

• openclaw gateway logs --lines 100

日志里通常写着真正的错误原因。

第二步：更新到最新版

• npm update -g openclaw

• openclaw gateway restart

很多问题在新版本中已经修复。

第三步：重置配置（谨慎）

• cp ~/.openclaw/openclaw.json ~/.openclaw/openclaw.json.bak 先备份

• openclaw configure 重新配置

第四步：求助社区

如果以上都没用，可以到以下渠道寻求帮助：

• GitHub Issues：搜索是否有相同问题

• Discord 社区：实时交流

• 官方文档：查看 FAQ 和排障指南
