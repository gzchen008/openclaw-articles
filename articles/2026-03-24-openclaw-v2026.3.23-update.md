# OpenClaw 2026.3.23 更新：修复 30+ 问题，稳定性大幅提升

> 发布时间：2026年3月24日
> 版本：2026.3.23
> 仓库：https://github.com/openclaw/openclaw

## 🎯 更新概览

OpenClaw 2026.3.23 版本带来了 **30+ 项重要修复**，涵盖浏览器控制、插件系统、模型集成、安全防护等核心模块，显著提升了系统稳定性和用户体验。

---

## 🔧 核心修复亮点

### 浏览器控制优化

**Chrome MCP 附加流程改进**
- 修复了 macOS Chrome 附加流程中的超时问题
- 等待现有会话浏览器标签页完全可用后再处理，减少重复授权提示
- 解决了慢速 Linux 环境下的浏览器启动回归问题

**影响**：浏览器自动化更加稳定，减少用户等待时间和授权干扰

---

### 插件系统增强

**消息工具修复**
- Discord `components` 和 Slack `blocks` 参数恢复可选
- 飞书 `message(..., media=...)` 现在正确路由媒体发送路径
- 修复了 pin/unpin/react 操作的 schema 验证错误

**ClawHub 认证**
- macOS 环境正确读取 ClawHub 登录凭证
- 支持 XDG 配置路径，技能浏览保持登录状态
- 解决了未认证 429 错误和空列表问题

**影响**：消息发送和技能管理更加可靠

---

### 模型集成改进

**OpenAI Codex OAuth**
- 支持 HTTP/HTTPS 代理配置
- 修复了代理环境下 Token 刷新失败的问题
- 确保 MiniMax 和 OpenAI Codex 登录流程在代理环境中正常工作

**Mistral 模型**
- 降低最大 token 默认值到安全输出预算
- `openclaw doctor --fix` 可修复旧的配置文件
- 避免 Mistral 422 拒绝错误

**OpenRouter 自动路由**
- 修复定价刷新无限递归问题
- `usage.cost` 恢复正常计算

**影响**：模型调用更稳定，成本追踪更准确

---

### 安全防护加强

**Shell 命令审批**
- 拒绝单引号包裹的 `$0`/`$n` 位置参数
- 禁止换行符分隔的 `exec` 命令
- 保持 `exec --` 载体形式的合法性

**Gateway 认证**
- Canvas 路由需要认证
- Agent 会话重置需要管理员权限
- 匿名访问和非管理员重置请求被拒绝

**影响**：防止命令注入，提升系统安全性

---

### 网关和进程管理

**网关探测**
- 修复成功握手后超时问题
- 慢速设备正确报告 RPC 失败而非假阴性

**进程监管**
- 防止 launchd 和 systemd 下的锁冲突崩溃
- 重复进程保持重试等待而非退出失败

**影响**：网关更加稳定，减少意外重启

---

## 📊 其他改进

- **Telegram 自动回复**：保持同聊天入站去重顺序
- **Discord 命令**：特权斜杠命令返回明确的未授权回复
- **Matrix 插件**：修复 Jiti 环境下的重复导出崩溃
- **DeepSeek 插件**：重构为共享单提供商插件入口
- **内存插件**：LanceDB 首次使用时自动引导安装
- **发布安装**：保留之前版本的捆绑插件和 UI 资产

---

## 🚀 如何升级

```bash
# 全局安装
npm install -g openclaw@latest

# 或使用 OpenClaw CLI
openclaw update
```

升级后运行 `openclaw doctor --fix` 检查并修复配置问题。
