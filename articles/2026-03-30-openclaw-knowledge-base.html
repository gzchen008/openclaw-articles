如何构建 OpenClaw 知识库？从零到精通的完整指南

你的 AI 助手会"失忆"吗？

每次重启会话，它就忘了你的偏好、项目进展、重要决策...这谁顶得住？

今天教你用 OpenClaw 构建一个永不失忆的知识库，让你的 AI 越用越聪明！

一、OpenClaw 记忆架构长啥样？

OpenClaw 的记忆系统很简单：全部用 Markdown 文件存储。

~/.openclaw/workspace/
├── MEMORY.md              # 长期记忆（决策、偏好、事实）
├── memory/
│   ├── 2026-03-30.md      # 每日笔记
│   ├── 2026-03-29.md
│   └── weekly/            # 周度汇总（可选）

加载规则：

MEMORY.md：每次私聊启动时自动加载

memory/*.md：今天 + 昨天的文件自动加载

群聊不加载 MEMORY.md（保护隐私）

核心理念：文件即记忆。只要写进文件，AI 就能记住。

二、三种 Memory Search 后端对比

1️⃣ Builtin（默认推荐）

特点：SQLite + FTS5 + 向量搜索，零配置

自动启用条件（有任一 API Key）：

OpenAI (OPENAI_API_KEY)

Gemini (GEMINI_API_KEY)

Voyage (VOYAGE_API_KEY)

Mistral (MISTRAL_API_KEY)

适合人群：大多数用户

2️⃣ QMD（高级搜索）

特点：Reranking + 查询扩展 + 额外目录索引

安装：

bun install -g https://github.com/tobi/qmd
brew install sqlite  # macOS

适合人群：需要高级搜索、索引外部目录、回忆历史对话

3️⃣ Honcho（知识图谱）

特点：AI 原生跨会话记忆 + 用户建模 + 多代理感知

适合人群：多代理系统、需要知识图谱

三、快速上手：零配置方案

如果你有 OpenAI 或 Gemini 的 API Key，什么都不用配置，直接就能用！

步骤 1：配置 API Key

export OPENAI_API_KEY="sk-YOUR_KEY_HERE"
# 或
export GEMINI_API_KEY="YOUR_KEY_HERE"

步骤 2：手动写入记忆

直接告诉你的 AI 助手：

"记住，我喜欢用 TypeScript"

"记住，项目 A 正在迁移到 Kubernetes"

"记住，每周五下午开例会"

它会自动写入 MEMORY.md 或 memory/YYYY-MM-DD.md。

步骤 3：搜索记忆

你的 AI 会自动使用 memory_search 查找相关内容。

就这么简单！ 🎉

四、进阶：QMD 高级搜索

需要更多功能？QMD 是你的选择。

安装 QMD

bun install -g https://github.com/tobi/qmd

配置 OpenClaw

{
  "memory": {
    "backend": "qmd",
    "qmd": {
      "paths": [
        { "name": "docs", "path": "~/notes", "pattern": "**/*.md" },
        { "name": "projects", "path": "~/project" }
      ],
      "sessions": { 
        "enabled": true,
        "retentionDays": 30 
      },
      "update": { "interval": "5m" },
      "limits": { "maxResults": 10, "timeoutMs": 10000 }
    }
  }
}

QMD 的强大功能

✅ Reranking：重排序，提高搜索质量

✅ Query Expansion：查询扩展，找得更准

✅ 索引额外目录：项目文档、团队笔记都能搜

✅ 索引会话记录：能回忆历史对话

✅ 完全本地运行：数据不离开你的机器

五、最佳实践：如何写记忆？

写入规则

内容类型 写入位置 示例
决策、偏好、事实 MEMORY.md "用户喜欢 TypeScript"
每日笔记、运行上下文 memory/YYYY-MM-DD.md "今天完成了 X 功能"
临时观察 memory/YYYY-MM-DD.md "项目 A 正在迁移到 K8s"

搜索优化：Hybrid Search 权重调优

{
  "memorySearch": {
    "query": {
      "hybrid": {
        "vectorWeight": 0.7,
        "textWeight": 0.3,
        "mmr": { "enabled": true, "lambda": 0.7 },
        "temporalDecay": { "enabled": true, "halfLifeDays": 30 }
      }
    }
  }
}

参数说明：

vectorWeight 高：语义相似更重要（"部署"能找到 "CDN 发布"）

textWeight 高：精确匹配更重要（"OpenClaw" 只找 "OpenClaw"）

mmr.lambda：0 = 最大多样性，1 = 最大相关性

halfLifeDays：每 N 天分数减半（越新越重要）

索引额外目录

有项目文档在外面？配置一下：

{
  "memorySearch": {
    "extraPaths": ["../team-docs", "/srv/shared-notes"]
  }
}

六、常用 CLI 命令

# 查看状态
openclaw memory status

# 搜索记忆
openclaw memory search "项目部署"

# 强制重建索引
openclaw memory index --force

# 查看 QMD 状态
openclaw memory status --agent main --index

七、总结：如何选择方案？

需求 推荐方案
快速上手 Builtin + API Key
需要搜索项目文档 QMD + paths 配置
需要回忆历史对话 QMD + sessions.enabled
完全自动化记忆 三层 Cron 方案
多代理知识图谱 Honcho

关键点：

✅ 文件即记忆，只要写进文件就能记住

✅ 有 API Key 就能启用向量搜索

✅ 需要高级功能就用 QMD

✅ 想完全自动化就用三层 Cron

你的 AI 助手，从此拥有"长期记忆"！
