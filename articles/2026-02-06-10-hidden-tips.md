## 🔟 十大隐藏技巧详解

### 1️⃣ `/status` - 一键查看会话状态

**是什么：** 查看当前会话的详细信息，包括使用的模型、token 消耗、会话类型等。

**怎么用：**
直接在对话框输入：
```
/status
```

你会看到：
- 🎯 当前模型（比如 kimi-coding/k2p5）
- 💰 本次会话的 token 消耗估算
- 🌐 当前频道类型（Discord/Telegram/Slack 等）
- ⚙️ 是否启用了推理模式

**适用场景：**
- 感觉回复变慢了，检查是不是用了大模型
- 想确认当前配置是否正确
- 排查问题时先看基础信息

---

### 2️⃣ `/reasoning` - 开启深度思考模式

**是什么：** 让 AI 在回答前展示完整的思考过程，适合复杂问题分析。

**怎么用：**
```
/reasoning on   # 开启
/reasoning off  # 关闭
```

开启后，AI 会在回答前显示一个「💭 Thinking」区域，你可以看到：
- 问题拆解过程
- 思考路径
- 决策依据

**适用场景：**
- 编程 Debug 时，看 AI 如何定位问题
- 写方案时，看 AI 的逻辑推理过程
- 学习新知识点时，理解 AI 的思考方式

**💡 小技巧：** 推理模式会消耗更多 token，简单问题建议关闭。

---

### 3️⃣ 多模型切换 - 不同任务用不同武器

**是什么：** OpenClaw 支持多个 AI 模型，你可以根据任务类型自由切换。

**怎么用：**
在任意消息中加入模型别名：
```
@kimi-coding 帮我写这段 Python 代码
@claude 帮我润色这封邮件
@gpt4 帮我分析这个商业案例
```

**常用模型推荐：**

| 模型 | 适用场景 | 特点 |
|------|----------|------|
| kimi-coding | 编程、技术问题 | 代码能力强，响应快 |
| claude | 写作、分析、长文本 | 逻辑清晰，表达自然 |
| gpt4 | 通用任务、创意生成 | 综合能力最强 |
| glm | 中文内容、翻译 | 中文理解好 |

**适用场景：**
- 写代码用 kimi-coding，写文案用 claude
- 复杂分析用 gpt4，日常问答用轻量模型
- 根据响应速度和成本灵活选择

---

### 4️⃣ 子会话 spawn - 并行处理多任务

**是什么：** 在后台启动独立的子会话，同时处理多个任务，互不干扰。

**怎么用：**
```
 spawn: 帮我分析这份数据，晚上给我结果
```

或者详细配置：
```
 spawn:
任务：分析附件中的销售数据，生成周报
模型：kimi-coding
超时：300秒
标签：周报分析
```

**适用场景：**
- 让 AI 在后台处理耗时任务，你继续做其他事
- 同时处理多个独立任务（数据分析 + 文案创作 + 邮件整理）
- 避免主会话历史被长任务污染

**💡 小技巧：** 用 `sessions_list` 查看所有子会话状态，用 `sessions_send` 给子会话发送消息。

---

### 5️⃣ `memory_search` - 秒搜历史记忆

**是什么：** 语义搜索你之前和 AI 的对话记录，快速找到需要的信息。

**怎么用：**
```
帮我搜索之前讨论过的 Docker 配置方法
```

或者直接调用工具：
```
memory_search: 查询关于 "数据库备份" 的记忆
```

**适用场景：**
- "我之前问过这个问题，答案是什么来着？"
- 跨会话查找之前的方案
- 整理知识点，避免重复提问

**💡 小技巧：** OpenClaw 会自动搜索记忆文件，无需手动指定。重要的结论让 AI 写入 MEMORY.md，方便长期保存。

---

### 6️⃣ 浏览器自动化 - 让 AI 帮你「逛」网页

**是什么：** OpenClaw 可以控制浏览器，截图、点击、填写表单，真正「看懂」网页。

**怎么用：**
```
帮我打开 https://github.com/trending，截图给我看看今天的趋势
```

复杂操作：
```
打开 https://example.com，点击登录按钮，截图给我看登录页面
```

**适用场景：**
- 监控网页变化（价格、库存、数据）
- 自动化填写表单
- 截图验证页面效果
- 数据采集和竞品分析

**💡 小技巧：** 浏览器操作需要目标网页可访问，部分网站有反爬虫机制可能无法使用。

---

### 7️⃣ Cron 定时任务 - 设置一次，每天自动执行

**是什么：** 让 AI 在指定时间自动执行任务，比如每天早报、定时提醒、自动报告。

**怎么用：**
创建一个定时任务：
```
每天早上 9 点帮我生成今日待办事项
```

系统会帮你创建 cron 任务，你也可以手动配置：
```json
{
  "name": "每日早报",
  "schedule": {
    "kind": "cron",
    "expr": "0 9 * * *",
    "tz": "Asia/Shanghai"
  },
  "payload": {
    "kind": "agentTurn",
    "message": "生成今日早报"
  }
}
```

**适用场景：**
- 每天早上自动生成工作日报
- 定时检查邮件/日历并提醒
- 定期备份数据
- 每日 GitHub 项目推荐

**💡 小技巧：** 一次性提醒用 `at` 类型，循环任务用 `cron` 表达式。

---

### 8️⃣ 代码执行与文件操作 - AI 直接动手

**是什么：** OpenClaw 可以直接执行代码、读写文件、运行命令，不只是「说说而已」。

**怎么用：**

**执行命令：**
```
帮我查看当前目录下的所有文件
```

**读写文件：**
```
把这段代码保存到 test.py 文件
读取 /Users/cgz/.openclaw/workspace/config.json 给我看看
```

**代码执行：**
```
运行这段 Python 代码，看看输出结果
```

**适用场景：**
- 让 AI 直接修改配置文件
- 批量处理文件（重命名、格式转换）
- 运行脚本获取系统信息
- 自动化部署和运维

**💡 小技巧：** OpenClaw 会询问是否执行敏感操作，确认安全后再允许。

---

### 9️⃣ TTS 语音输出 - 让 AI 开口说话

**是什么：** 将文字转换为语音，支持多种声音和输出格式。

**怎么用：**
```
把这段话转成语音："欢迎使用 OpenClaw，你的私人 AI 助手"
```

或者在代码中调用：
```python
# 生成语音文件
tts(text="你好世界", channel="telegram")
```

**适用场景：**
- 把长文章转成音频，通勤时听
- 为视频配音
- 生成播客内容
- 视障用户辅助阅读

**💡 小技巧：** 不同频道支持不同格式的音频输出，Telegram 支持直接发送语音消息。

---

### 🔟 跨会话消息发送 - 多群组协同

**是什么：** 在不同会话/频道之间发送消息，实现跨群通知、多频道同步。

**怎么用：**
```
发送消息到工作群：今天的会议取消了
```

或者直接指定会话：
```
sessions_send(sessionKey="xxx", message="任务完成")
```

**适用场景：**
- 在私聊中让 AI 转发消息到工作群
- 跨 Discord 频道同步重要通知
- 主会话监控多个子会话的状态

**💡 小技巧：** 使用 `sessions_list` 查看所有活跃会话的 key。

## ❓ 常见问题

**Q1: 这些技巧会消耗很多 token 吗？**
- 大部分不会。只有 `/reasoning` 和浏览器自动化会额外消耗，建议按需使用。

**Q2: 可以同时使用多个技巧吗？**
- 完全可以！比如在子会话中使用浏览器自动化，同时主会话继续对话。

**Q3: 技巧记不住怎么办？**
- 收藏这篇文章，或者在 MEMORY.md 中记录你常用的技巧。

**Q4: 某些功能提示权限不足？**
- 部分功能需要特定配置或权限，检查你的 OpenClaw 配置或咨询管理员。
