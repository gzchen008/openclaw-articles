# OpenClaw 记忆系统全解析：让 AI 助手真正记住你

你有没有遇到过这种情况：每次打开 AI 助手，它都像失忆了一样，之前聊过什么、你的偏好是什么、上次的任务进展到哪了——统统不记得。你不得不一遍又一遍地重复背景信息，效率极低。

OpenClaw 的记忆系统就是为了彻底解决这个问题而生的。它不是简单的"上下文窗口"，而是一套完整的、分层的记忆架构，让 AI 助手真正拥有短期、中期和长期记忆能力。

今天这篇文章，我会从底层原理到实际配置，带你完整掌握 OpenClaw 的记忆系统。

## 一、为什么 AI 助手需要"记忆"？

大多数 AI 聊天工具的痛点非常一致：

**上下文窗口有长度限制。** GPT-5.4 的 128K 看起来很长，但连续聊几天就满了。一旦超出，最早的对话就会被截断，助手就"忘了"之前说过的话。

**跨会话完全断联。** 昨天聊的和今天聊的，之间没有任何联系。每次开新对话都是一张白纸。

**个性化无从谈起。** 没有记忆，助手就无法学习你的偏好、习惯和工作方式，永远只能给出通用回答。

OpenClaw 用三层记忆架构解决了这些问题。

## 二、三层记忆架构

OpenClaw 的记忆系统分三层，各有分工：

### 2.1 工作记忆（Session Context）

这就是当前对话的上下文窗口。你在本次会话中说的每一句话、AI 的每一次回复，都在这里。

**特点：** 速度快、精确、但容量有限
**生命周期：** 随会话结束而消失
**最佳用途：** 当前任务的即时上下文

### 2.2 日记记忆（Daily Notes）

路径：`memory/YYYY-MM-DD.md`

这是 OpenClaw 的"日记本"。每天的对话要点、决策、重要事件都会被记录下来。下次会话启动时，助手会自动读取今天和昨天的日记，快速恢复上下文。

**配置方式：** 在工作区目录下创建 `memory/` 文件夹即可自动启用。

**日记内容示例：**

```markdown
# 2026-04-14

## 今日要事
- 完成了用户认证模块的代码审查
- 确定下周三与客户视频会议
- 偏好更新：代码风格改用 ESLint Airbnb 规范

## 待跟进
- 部署测试环境需要运维配合
```

### 2.3 长期记忆（MEMORY.md）

路径：`MEMORY.md`

这是助手经过精心整理的长期知识库。它不是原始对话记录，而是提炼后的核心信息：你的偏好、重要决策、反复出现的模式、经验教训等。

**关键区别：** 日记是"今天发生了什么"，长期记忆是"什么是重要的"。长期记忆会定期从日记中提炼和更新。

**示例结构：**

```markdown
# 长期记忆

## 用户偏好
- 代码风格：ESLint Airbnb
- 语言：技术讨论用英文，日常沟通用中文
- 工作时间：上午 9 点到晚上 8 点，不要在深夜打扰

## 项目上下文
- 主项目使用 React + TypeScript
- 部署在 AWS，偏好 us-west-2 区域

## 重要决策记录
- 2026-03-15：决定从 REST 迁移到 GraphQL
```

## 三、Dreaming：主动记忆整理

这是 OpenClaw 记忆系统中最强大的功能之一。

**Dreaming（做梦）** 是指 AI 助手在空闲时间自动整理记忆的过程，就像人类在睡眠中巩固记忆一样。

### 3.1 工作原理

1. 助手进入空闲状态时，Dreaming 自动触发
2. 回顾最近的日记记录
3. 提取有价值的信息（新的偏好、重要决策、模式变化）
4. 更新 MEMORY.md
5. 生成记忆摘要供后续参考

### 3.2 启用 Dreaming

```bash
# 在 OpenClaw TUI 中执行
/dreaming on
```

Dreaming 启用后，助手会周期性地自动整理记忆，无需手动干预。

### 3.3 记忆回填（REM Backfill）

如果你有大量历史对话记录（比如从 ChatGPT 导出的数据），OpenClaw 支持记忆回填：

```bash
openclaw rem-harness --path /path/to/chatgpt-export
```

这个命令会把历史对话注入 Dreaming 流程，从中提取有价值的信息整合到长期记忆中。

## 四、Active Memory：实时记忆检索

OpenClaw v2026.4.10 引入了一个新功能：Active Memory 插件。

传统记忆系统的局限是"被动"的——助手只在需要时才搜索记忆。Active Memory 改变了这一点：

**每次对话开始时，Active Memory 会主动检索相关记忆并注入上下文。** 你不需要说"记得上次我们讨论的..."，助手就已经知道了。

### 4.1 配置 Active Memory

在 `openclaw.json` 中添加：

```json
{
  "plugins": {
    "slots": {
      "memory": "memory-active"
    }
  }
}
```

### 4.2 三种上下文模式

Active Memory 支持三种模式，适配不同使用场景：

**消息模式（message）：** 只根据当前消息检索记忆，轻量快速
**最近模式（recent）：** 检索最近 N 条消息的相关记忆，平衡精确度和上下文
**完整模式（full）：** 检索完整会话上下文的相关记忆，最全面但消耗更多 token

根据你的使用频率和 token 预算选择合适的模式。

## 五、记忆搜索：精准回忆

有时候你需要助手回忆一个具体的信息，比如"上个月我们讨论的那个数据库选型方案是什么？"

这时可以使用 `memory_search` 工具进行语义搜索：

```bash
# 在对话中直接提问
"搜索记忆中关于数据库选型的讨论"
```

助手会在所有记忆文件（MEMORY.md + memory/*.md）中进行语义搜索，找到最相关的内容返回。

## 六、实战：完整记忆配置模板

以下是一个推荐的完整配置：

### 6.1 目录结构

```
~/.openclaw/workspace/
├── AGENTS.md          # 助手行为准则
├── SOUL.md            # 助手人格设定
├── USER.md            # 用户信息
├── TOOLS.md           # 工具配置笔记
├── MEMORY.md          # 长期记忆
├── HEARTBEAT.md       # 心跳检查清单
└── memory/
    ├── 2026-04-13.md  # 昨天的日记
    ├── 2026-04-14.md  # 今天的日记
    └── heartbeat-state.json  # 心跳状态追踪
```

### 6.2 AGENTS.md 中的记忆指令

在 AGENTS.md 中明确告诉助手如何使用记忆：

```markdown
## 记忆规则

1. 每次会话开始，读取今天和昨天的 memory/*.md
2. 重要决策和偏好变更，立即写入当天日记
3. 不要在日记中记录敏感信息（密码、密钥等）
4. 每隔几天，从日记中提炼关键信息到 MEMORY.md
5. 对话中涉及的信息如果可能在未来有用，主动记录
```

### 6.3 隐私保护

记忆系统中可能包含敏感信息，需要注意：

**MEMORY.md 只在主会话加载。** 在群聊、共享会话等场景下，助手不会加载 MEMORY.md，防止个人信息泄露。

**敏感信息不入记忆。** 在 AGENTS.md 中明确禁止记录密码、API Key、私钥等。

**记忆文件在本地。** 所有记忆文件都存储在你的本地机器上，不会上传到云端。

## 七、常见问题与避坑

### 问题 1：记忆太多，Token 消耗增长

随着使用时间增长，记忆文件会越来越大。解决方法：

- 定期清理过时的日记文件（超过 30 天的可以归档）
- 在 Dreaming 中配置更严格的提炼规则
- 使用 Active Memory 的"消息模式"减少每次检索的 token 量

### 问题 2：助手"记得"了错误的信息

记忆不是万能的。如果助手记录了错误信息：

- 直接告诉助手"这个信息是错的，请更新记忆"
- 手动编辑 MEMORY.md 修正
- 让 Dreaming 下次运行时重新提炼

### 问题 3：多设备同步

OpenClaw 的记忆文件存储在工作区目录中。如果你使用 Git 管理工作区，记忆可以在多设备间同步：

```bash
cd ~/.openclaw/workspace
git add memory/ MEMORY.md
git commit -m "sync memory"
git push
```

但请注意：**不要把包含敏感信息的记忆文件推送到公共仓库。**

## 八、总结

OpenClaw 的记忆系统是一次从"无状态聊天"到"有状态 AI 伴侣"的质变：

- **工作记忆**处理当前对话的即时上下文
- **日记记忆**记录每日事件和决策
- **长期记忆**存储提炼后的核心知识
- **Dreaming** 自动整理和巩固记忆
- **Active Memory** 主动检索相关上下文

配置好这套系统后，你的 AI 助手将不再是一个每次都从零开始的工具，而是一个真正"认识你"的智能伙伴。

如果你还没有配置记忆系统，现在就开始吧——创建 `memory/` 目录，写一个 MEMORY.md，然后开启 Dreaming。你会发现，一个有记忆的 AI 助手和没有记忆的，完全是两个不同的物种。
